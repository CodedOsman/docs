**Student & Teacher Manual**

**1. Project Overview**

The altimeter and vertical speed indicator (VSI) are two of the six basic T-scan instruments in every aircraft cockpit. In this project, students install a BMP280 barometric pressure sensor on the Project 11 RC aircraft and log altitude and vertical speed to a MicroSD card during flight. After landing, they download the CSV file and graph the complete flight profile — seeing their takeoff, climb, cruise, descent, and landing phases as data for the first time. This mirrors exactly what flight data engineers do when they review an aircraft's on-board parameter recorder.

**Learning Objectives**

* Explain how a barometric pressure sensor measures altitude and why atmospheric pressure decreases with height
* Distinguish between indicated altitude (instrument reading) and density altitude (performance-relevant altitude)
* Wire a BMP280 sensor and MicroSD module to an ESP32 using I2C and SPI protocols respectively
* Write code that logs altitude, vertical speed, and temperature to a CSV file at 2 Hz
* Import the CSV into a spreadsheet; graph altitude and VSI against time; identify all flight phases
* Calculate density altitude from the logged temperature and pressure data

**5. Engineering Principles**

**How a Barometric Altimeter Works**

* Atmospheric pressure decreases with altitude at approximately 1 hPa per 8 metres at low altitude
* The BMP280 measures absolute pressure (hPa) and converts to altitude using the International Standard Atmosphere (ISA) model
* The ISA assumes sea-level pressure = 1013.25 hPa and temperature = 15°C — in Ghana, actual conditions often differ

|  |
| --- |
| **Density Altitude Formula**  Density altitude accounts for the actual effect of temperature and pressure on air density — this is what matters for aircraft performance.  Density Altitude (ft) = Pressure Altitude (ft) + 120 × (OAT\_°C – ISA\_Temp\_°C)  ISA Temperature at any altitude = 15°C – (2°C × altitude\_in\_thousands\_of\_feet)  Example: Accra elevation = 213 ft; OAT = 30°C; ISA Temp at 213 ft ≈ 15°C  Density Altitude = 213 + 120 × (30 – 15) = 213 + 1800 = 2013 ft  Practical implication: your aircraft performs as if it is at 2013 ft even though you are only 213 ft above sea level. |

**9. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Altitude drifts indoors** | Pressure changes from doors/windows/people moving | Normal — BMP280 is sensitive; calibrate outdoors in stable air before flight |
| **SD card not recognised** | Card not formatted FAT32 or wrong SPI pin | Format card as FAT32 in Windows/Mac; verify CS pin assignment in code |
| **Altitude noisy (jumps ±2 m)** | No IIR filter applied | Enable oversampling (OSS=4) and IIR filter (coeff=4) in BMP280 library config |
| **VSI always zero** | Time step not measured correctly | Use millis() to measure exact time between readings; divide altitude change by elapsed ms |
| **CSV file corrupted** | SD card removed before file closed | Add card.close() in the landing detection code; always power off before removing SD |

**10. Upgrade & Extension Ideas**

* Multi-Flight Log: modify code to create a new file (FLIGHT002.CSV, FLIGHT003.CSV etc.) on each power-up; compare multiple flights
* GPS Altitude Comparison: add the NEO-6M GPS from Project 9; log GPS altitude alongside barometric altitude; compare the two
* Temperature Lapse Rate: graph temperature vs. altitude from the log; calculate the actual lapse rate and compare to ISA standard (–2°C per 1000 ft)
* OLED Real-Time Display: combine with Project 9 OLED display; show live altitude and VSI during flight
* Density Altitude Alert: program an LED warning that activates when calculated density altitude exceeds a set threshold

**11. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* Format the SD cards as FAT32 before Lesson 1 — Windows formats cards > 32 GB as exFAT by default, which the SD library cannot read
* The density altitude calculation (Step 7) is a rich cross-curricular opportunity — link to the Physics class covering gas laws and atmospheric pressure
* Display a real aircraft flight data graph on the projector before students graph their own — they immediately recognise the climb, cruise, and descent phases

**Differentiation Strategies**

* Support – BMP280 wiring and Serial Monitor only; no SD logging; teacher provides pre-made CSV for graphing exercise
* Core – Full build; SD logging; flight; CSV download; graph; density altitude calculation
* Extension – Multi-flight comparison; GPS altitude overlay; temperature lapse rate analysis
