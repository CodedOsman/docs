**Student & Teacher Manual**

**1. Project Overview**

Every aircraft that flies at night — or in conditions of reduced visibility — is required by ICAO regulations to display navigation lights in specific colours, positions, and arcs. These lights allow other pilots and controllers to instantly identify the aircraft's position, heading, and orientation. In this project, students wire a complete LED navigation light system onto the Project 11 RC aircraft using an ESP32 microcontroller, implementing ICAO-standard colours and adding a programmable anti-collision strobe — exactly as fitted to every commercial aircraft operating at Kotoka International Airport.

**Learning Objectives**

* Identify the five ICAO-standard aircraft navigation lights: port, starboard, tail, anti-collision strobe, and landing light
* Explain the colour, position, and arc requirements for each light as defined by ICAO Annex 2
* Wire five LEDs with current-limiting resistors to an ESP32 microcontroller
* Program a flashing strobe at the correct ICAO rate (38–42 flashes per minute) using a delay loop
* Implement a channel-6 transmitter switch to control the landing light from the cockpit
* Demonstrate the orientation test: correctly identify the nose direction of the illuminated aircraft from 5 m in low light

**3. Build Steps & Assembly**

**Lesson 1 – ICAO Standards & Wiring Plan**

|  |  |
| --- | --- |
| **STEP 1** | **Study ICAO Lighting Standards** |
|  | * Review the ICAO Navigation Light Reference Table in Section 5 * Each student draws a top-view aircraft outline; marks the position, colour, and arc of each light * Identify: why red on the left (port) and green on the right (starboard)? * Answer: a pilot seeing red light is facing the aircraft's left side — it is coming from their right * Discuss: at Kotoka International Airport, how do ATC controllers identify aircraft orientation at night? |

|  |  |
| --- | --- |
| **STEP 2** | **Wiring Plan & Resistor Calculation** |
|  | * ESP32 GPIO outputs 3.3V. LED forward voltage (Vf) ≈ 2.0V (red/green), 3.0V (white) * Resistor value: R = (Vcc – Vf) / If = (3.3 – 2.0) / 0.01 = 130Ω minimum; use 470Ω for safe, bright operation * Draw the wiring diagram: 5 × LED + resistor chains from GPIOs 5, 18, 19, 21, 23 to GND * Plan wire routing: port LED wire runs left along wing spar; starboard right; tail along fuselage spine * Teacher approves wiring diagram before any soldering or connecting begins |

**Lesson 2 – Installation & Wiring**

|  |  |
| --- | --- |
| **STEP 3** | **Prepare LED Assemblies** |
|  | * Solder or twist a 470Ω resistor to the anode (long leg) of each LED * Cover the resistor-LED junction with heatshrink tubing; apply heat to shrink * Extend the anode wire to the required length: 30 cm for port/starboard (wingtip), 25 cm for tail * Label each assembly: P (port/red), S (starboard/green), T (tail/white), STR (strobe/white), LL (landing/white) |

|  |  |
| --- | --- |
| **STEP 4** | **Mount LEDs on Aircraft** |
|  | * Port LED: mount at the left wingtip, facing forward-left; secure with a drop of hot glue * Starboard LED: mount at the right wingtip, facing forward-right * Tail LED: mount at the rear of the fuselage, facing rearward * Strobe: mount on top of the fuselage at the CG point, facing upward * Landing light: mount on the nose, facing forward * Route all wires along the spar and fuselage; secure with masking tape every 5 cm |

|  |  |
| --- | --- |
| **STEP 5** | **Connect to ESP32** |
|  | * Connect each LED anode wire (via its resistor) to the correct GPIO as per the Wiring Reference Table * Connect all LED cathode wires to the common GND rail * Double-check: no bare wire touches the carbon frame or metal parts * Power on from USB: all steady LEDs should illuminate immediately |

**Lesson 3 – Code, Strobe & Test**

|  |  |
| --- | --- |
| **STEP 6** | **Upload Navigation Light Code** |
|  | * Open the nav\_lights.ino sketch from Kit 11 USB drive * Review the code structure: * Setup: all GPIO pins set to OUTPUT; all lights on * Loop: steady lights remain on; strobe GPIO toggles HIGH/LOW with 750 ms delay each * Landing light: reads RC channel 6 PWM value; >1500 μs = on; <1500 μs = off * Upload to ESP32; confirm strobe flashing and steady lights |

|  |  |
| --- | --- |
| **STEP 7** | **Verification Tests** |
|  | * Complete all rows in the Test Verification Table (Section 6) * Count strobe flashes in 60 seconds using a stopwatch — target 38–42 flashes * Night orientation test: darken the room; power on the aircraft; student stands 5 m away * Student calls out: 'The nose is pointing [direction]' based on light colours alone * Correct: red (port) on left of view = aircraft nose pointing to the right |

**4. Power & Safety Notes**

|  |
| --- |
| **⚠ Safety Notes**  Power: 3.3V/5V from ESP32 USB for ground tests. Aircraft LiPo only for flight — teacher connects.  LEDs and resistors: Never connect LEDs without resistors — unprotected LEDs will draw excessive current and burn out the ESP32 GPIO.  Heatshrink: Use a heat gun or lighter only with teacher present — ensure adequate ventilation.  Wiring: All wires must be routed away from propeller arcs. A wire caught in a propeller will cause immediate loss of control. |

**5. Engineering Principles**

**ICAO Navigation Light Standards**

| **Light** | **Colour** | **Position** | **Arc** | **Meaning** |
| --- | --- | --- | --- | --- |
| **Port navigation** | Red | Left wingtip | 110° forward arc | Aircraft's left side |
| **Starboard navigation** | Green | Right wingtip | 110° forward arc | Aircraft's right side |
| **Tail navigation** | White | Tail | 140° rear arc | Aircraft viewed from behind |
| **Anti-collision strobe** | White or Red | Fuselage top/bottom | 360° | Flashing; warns other aircraft |
| **Landing light** | White | Nose/wing leading edge | Forward | Illuminates runway on approach |

**Wiring Reference Table**

| **LED** | **GPIO** | **Resistor** | **Behaviour** | **ICAO Colour** |
| --- | --- | --- | --- | --- |
| **Port LED** | GPIO 5 | 470Ω | Steady on during flight | Red |
| **Starboard LED** | GPIO 18 | 470Ω | Steady on during flight | Green |
| **Tail LED** | GPIO 19 | 470Ω | Steady on during flight | White |
| **Anti-collision Strobe** | GPIO 23 | 470Ω | Flashing: 40 flashes/min | White |
| **Landing Light** | GPIO 21 | 470Ω | On demand via switch (CH6) | White |

|  |
| --- |
| **Why These Colours? – The Rules of the Air**  ICAO Annex 2 (Rules of the Air) mandates navigation light colours internationally.  Red on the left (port) and green on the right (starboard) are derived from maritime navigation rules — aircraft inherited the same convention.  A pilot seeing BOTH red and green simultaneously knows an aircraft is flying directly toward them — the most critical awareness possible.  A pilot seeing only WHITE is following another aircraft from behind — overtaking rules apply.  These colours are used at Kotoka International Airport, on every domestic Passion Air and Africa World Airlines flight, and on all aircraft worldwide. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **LED does not illuminate** | Anode/cathode reversed or GPIO wrong | Flip LED; confirm long leg to GPIO via resistor |
| **LED too dim** | Resistor value too high | Reduce to 330Ω; measure LED brightness in a darkened room |
| **Strobe rate wrong** | Delay value incorrect in code | Adjust delay\_ms value: 750 ms on + 750 ms off = 40 flashes/min |
| **Landing light always on** | Wrong logic in CH6 code | Check channel 6 PWM value threshold; reverse if/else condition |
| **LEDs draw too much current** | No resistors in circuit | Never connect LEDs directly to GPIO without resistors — GPIO will burn out |

**9. Upgrade & Extension Ideas**

* PWM Dimming: add a sunrise/sunset effect where all lights gradually dim and brighten using PWM analogWrite
* Taxi Light: add a lower-power white LED under the nose for ground movement illumination
* Formation Light: add a blue LED array under the fuselage for formation flying identification at night
* Bluetooth Control: use the ESP32 Bluetooth module to control individual lights from a phone app
* Real ICAO Flash Pattern: program the more complex ICAO-specified double-flash strobe pattern

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* Conduct the night orientation test at the end of Lesson 3 — even 30 seconds of dimmed lights in a curtained room is dramatically effective at demonstrating why ICAO colours matter
* Pre-solder the resistors onto the LED anodes before Lesson 2 for groups without soldering experience
* Show a short video of night approach at Kotoka International Airport — students can identify port and starboard lights on approaching aircraft

**Differentiation Strategies**

* Support – 3 steady LEDs only (port, starboard, tail); no strobe programming; wiring only
* Core – All 5 LEDs; strobe code; landing light channel 6 switch; night orientation test
* Extension – PWM dimming; Bluetooth control; real ICAO flash pattern
