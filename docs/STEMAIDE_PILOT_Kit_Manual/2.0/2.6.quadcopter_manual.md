**Student & Teacher Manual**

**1. Project Overview**

This project develops precision quadcopter piloting to professional standard. Students plan and fly a 4-waypoint square course, maintaining specific altitudes at each waypoint and executing a smooth return-and-land sequence. The structured scoring system rewards heading accuracy, altitude control, hover stability, and landing precision — the same performance metrics used by UAV operators in commercial survey, inspection, and delivery operations.

| Prerequisite | Project 10 (quadcopter built and 5 tethered hover flights completed) |

**Learning Objectives**

* Plan a 4-waypoint mission course including headings, distances, and target altitudes for each leg
* Fly a complete waypoint mission in sequence, hovering for 5 seconds at each waypoint
* Maintain target altitude within ±0.5 m during transit and ±0.3 m during hover
* Apply wind correction techniques to maintain intended heading over the ground
* Complete a full mission log with arrival times, altitudes, and distance observations
* Score a minimum of 35 out of 55 on the structured mission scoring table

**3. Mission Planning & Session Steps**

**Lesson 1 – Mission Planning & Course Layout**

|  |  |
| --- | --- |
| **STEP 1** | **Mission Planning Briefing** |
|  | * Introduce the concept of a waypoint mission: a pre-planned sequence of GPS or visual points * Review the Mission Profile table (Section 5): 4 waypoints in a square, each at a different altitude * Students copy the mission plan into their logbooks: waypoint, heading, distance, target altitude * Discuss wind conditions: how will wind affect each leg? Which legs will need correction? * Identify the most challenging leg before flying — this becomes the first practice focus |

|  |  |
| --- | --- |
| **STEP 2** | **Course Setup & Verification** |
|  | * Select an open outdoor area: minimum 25 m × 25 m, clear of obstacles and bystanders * Mark the Launch point with a cone; use a compass or phone GPS to establish North * Measure 10 m North from Launch and place WP 1 cone * From WP 1, measure 10 m East and place WP 2; 10 m South for WP 3; 10 m West for WP 4 * WP 4 should be 10 m from the Launch point (verify — the course is a square) * Place wind indicator upwind of the course; note wind direction and strength |

|  |  |
| --- | --- |
| **STEP 3** | **Pre-Flight Inspection & Sign-Off** |
|  | * Complete the full 10-point pre-flight checklist from Project 10 * Teacher countersigns; teacher connects LiPo battery * Pilot performs transmitter range check: walk 20 m away; confirm all controls still respond * Establish safety cordon: all observers remain at least 5 m behind the pilot * Brief the mission: pilot states each leg aloud before arming motors |

**Lesson 2 – Leg-by-Leg Practice**

|  |  |
| --- | --- |
| **STEP 4** | **Leg 1 Practice – Launch to WP 1** |
|  | * Arm motors; take off vertically to 1.5 m; hold hover for 5 s * Fly North toward WP 1 at 2.0 m altitude * Arrive at WP 1; hover for 5 s; land * Teacher and pilot assess: how close to WP 1 cone? Was altitude maintained? * Repeat Leg 1 until the pilot can consistently arrive within 1 m of the cone |

|  |  |
| --- | --- |
| **STEP 5** | **Legs 2–5 Individual Practice** |
|  | * Practice each remaining leg individually using the same method as Step 4 * For each leg: take off → fly the leg → hover at waypoint → land → debrief * Focus on the specific challenge for each leg (see Course Layout table) * Note: Leg 5 (WP 4 to Return) is diagonal — estimate the heading before flying |

**Lessons 3–4 – Full Mission Runs & Scoring**

|  |  |
| --- | --- |
| **STEP 6** | **First Full Mission Run (Practice)** |
|  | * Fly the complete mission sequence without stopping: Launch → WP1 → WP2 → WP3 → WP4 → Return → Land * Observer records observations in the waypoint log during the flight * No scoring on this run — focus on sequence completion * Debrief: which waypoint was hardest to reach? Which leg needed the most correction? |

|  |  |
| --- | --- |
| **STEP 7** | **Scored Mission Runs** |
|  | * Runs 2, 3, and 4 are scored using the Mission Scoring Table (Section 6) * Observer (teacher or assigned student) scores each category in real time * Complete the waypoint log for every run * Best score across all runs counts for the final competition * Pass mark: 35 / 55 points |

|  |  |
| --- | --- |
| **STEP 8** | **Time Challenge & Bonus Attempts** |
|  | * Students who achieve 35+ points attempt the Time Bonus: complete the mission in under 3 minutes * Students who have consistent altitude control attempt the Altitude Accuracy Bonus: all altitudes within ±0.3 m * Both bonuses can be attempted in the same run * Competition final: each pilot flies one scored run; top score wins |

|  |  |
| --- | --- |
| **STEP 9** | **Mission Debrief & Logbook Entry** |
|  | * Each student completes their waypoint log and scoring table * Group discussion: Which correction technique helped most? What would you change about the course layout? * Connect to real operations: commercial UAV operators file a mission plan before every flight and debrief against it afterwards * Teacher signs off completed logbooks |

| **⚠ CRITICAL SAFETY PROTOCOL**  LiPo battery: Teacher connects and disconnects only. Students must not handle the battery.  Propeller guards: MANDATORY for all flights in this project. No exceptions.  Flight zone: Minimum 25 m × 25 m; all bystanders outside the safety cordon at all times.  Emergency procedure: If drone becomes uncontrollable, reduce throttle to zero immediately and call 'ABORT'.  Altitude limit: Maximum 5 m altitude during this project. Do not exceed.  Battery: Land immediately if low-voltage warning activates. Do not attempt the next leg on low battery.  Weather: Do not fly in wind above 20 km/h or in rain. Postpone the lesson if conditions are unsuitable. |

**5. Mission Planning Reference**

**Course Layout**

| **Waypoint** | **Position (from Launch)** | **Hover Target** | **Altitude** | **Challenge Element** |
| --- | --- | --- | --- | --- |
| **Launch** | Start point – marked with cone | 5 s ground hold | 1.5 m | Takeoff precision |
| **WP 1** | 10 m North | 5 s hover | 2.0 m | Cardinal heading control |
| **WP 2** | 10 m East (of WP 1) | 5 s hover | 2.5 m | Altitude change in transit |
| **WP 3** | 10 m South (of WP 2) | 5 s hover | 2.0 m | Return leg orientation |
| **WP 4** | 10 m West (of WP 3) | 5 s hover | 1.5 m | Final approach altitude |
| **Return** | Back to launch point | 10 s hover | 1.0 m | Precision landing spot |

**Mission Profile – Leg by Leg**

| **Leg** | **From → To** | **Distance** | **Heading** | **Altitude Change** |
| --- | --- | --- | --- | --- |
| **1** | Launch → WP 1 | 10 m | North (360°) | Climb to 2.0 m |
| **2** | WP 1 → WP 2 | 10 m | East (090°) | Climb to 2.5 m |
| **3** | WP 2 → WP 3 | 10 m | South (180°) | Descend to 2.0 m |
| **4** | WP 3 → WP 4 | 10 m | West (270°) | Descend to 1.5 m |
| **5** | WP 4 → Return | 14 m (diagonal) | South-East (~135°) | Descend to 1.0 m; land |

Mission planning tools like DJI FlightHub, ArduPilot Mission Planner, or QGroundControl generate GPS waypoint sequences that the drone follows autonomously.  Manual waypoint flying (this project) develops the spatial awareness and vehicle control needed to supervise autonomous missions and intervene safely when automation fails.  Ghana's GCAA requires UAV operators to file a flight notification before every mission in controlled or congested airspace. |

**Wind Correction for Quadcopters**

* A quadcopter pushed by a crosswind will drift off its intended track unless the pilot corrects
* Correction: apply a small roll or pitch input into the wind — this creates a lateral force that counteracts the drift
* On a headwind leg (flying into wind): the drone will slow down — increase throttle slightly to maintain speed
* On a downwind leg (flying with wind): the drone will speed up and may overshoot the waypoint — reduce throttle
* Hover in wind: apply small, constant stick input into the wind to hold position

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Misses waypoints consistently** | Poor heading awareness; over-relying on visual | Use landmarks or heading indicator; practise single-leg flights before full mission |
| **Unstable hover at waypoint** | Overcorrecting stick inputs | Make smaller corrections; wait 0.5 s before correcting again; altitude hold mode if available |
| **Altitude fluctuates in transit** | Inconsistent throttle during movement | Set a base throttle for level flight; only adjust slightly for altitude corrections |
| **Drifts off leg heading** | Wind or IMU drift | Apply small roll/pitch correction into wind; re-check heading at each waypoint |
| **Hard landing** | Throttle cut too quickly on descent | Reduce throttle in small steps; final descent rate should be 0.5 m/s or less |
| **Battery depleted before mission ends** | High power consumption from corrections | Practice more; reduce unnecessary corrections; use a fully charged battery |

**9. Upgrade & Extension Ideas**

* Obstacle Course – Add vertical poles (bamboo canes) between waypoints; pilot must fly around or between them
* Payload Delivery – Attach a simple hook; pick up a small payload at WP 1; deliver it to WP 3
* Altitude Profile Mission – Each leg flown at a different altitude (e.g. 1 m, 2 m, 3 m, 2 m, 1 m) — tests vertical control
* Autonomous Mission – Connect a GPS module to the flight controller; program the 4 waypoints in ArduPilot Mission Planner; observe autonomous flight
* FPV Mission – Complete the full mission in first-person view using FPV goggles
* Team Relay – Two-pilot relay race: pilot 1 flies WP 1 and WP 2; pilot 2 takes over for WP 3, WP 4, and landing

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* Lesson 1 (mission planning) is important — pilots who brief the course before flying consistently perform better than those who jump straight to flight
* Use ground markers (chalk circles, not just cones) to show the 1 m accuracy target around each waypoint
* For the scored runs, have a second student act as observer and scorer — this also develops assessment skills
* Photograph or video each scored run from directly above (use a pole or second drone if available) for post-flight analysis
* Ensure the LiPo battery is fully charged between runs — a partially charged battery affects hover stability

**Differentiation Strategies**

* Support – Reduce course to 3 waypoints (triangle); accept 1.5 m accuracy; no altitude change between waypoints
* Core – Full 4-waypoint square; altitude changes; 5 s hovers; complete scoring table
* Extension – Obstacle course; payload delivery; autonomous GPS mission; FPV mission; team relay

**Assessment Suggestions**

* Mission score: 35 / 55 minimum (pass); 50+ (distinction)
* Waypoint log: All 6 rows completed with estimated or measured values
* Mission brief: Student states each leg heading and altitude before flying (verbal check)
* Debrief quality: Student identifies their weakest leg and explains the correction they would apply
