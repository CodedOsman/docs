**Student & Teacher Manual**

**1. Project Overview**

This project is the electronics capstone of the Advanced section. Students integrate four sensors — barometric pressure, gyroscope/accelerometer, compass, and GPS — into a single ESP32-powered cockpit display that renders five flight instruments simultaneously on a colour TFT screen. The result is a functional glass-cockpit panel that mirrors the Primary Flight Display (PFD) found in modern commercial and general aviation aircraft. Every design decision — from sensor placement to I2C address management to display layout — reflects real avionics engineering practice.

**Learning Objectives**

* Wire four sensors and a TFT display to an ESP32 using I2C, SPI, and UART protocols simultaneously
* Explain I2C address management and how to resolve address conflicts between multiple sensors on the same bus
* Write and integrate sensor-reading code for BMP280, MPU6050, QMC5883L, and NEO-6M GPS
* Design a cockpit instrument layout following the standard T-scan pattern used in real aircraft
* Implement a complementary filter to correct gyroscope drift using accelerometer data
* Mount all components on a faceplate and produce a complete labelled wiring diagram

**3. Build Steps & Assembly**

**Lesson 1 – Breadboard Setup & I2C Sensors**

|  |  |
| --- | --- |
| **STEP 1** | **Power Rail & ESP32 Setup** |
|  | * Place ESP32 across the centre gap of the full-size breadboard * Connect 3.3V pin → red power rail; GND → blue power rail * Connect a second GND from a different ESP32 GND pin for redundancy * Do not connect GPS or display power yet — add one sensor at a time |

|  |  |
| --- | --- |
| **STEP 2** | **Wire All Three I2C Sensors** |
|  | * BMP280: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * MPU6050: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * QMC5883L: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * All three share GPIO 21 and GPIO 22 — each has a unique I2C address * Run the I2C scanner sketch and confirm all three addresses are detected: 0x76/0x77, 0x68, 0x0D * If any address is missing, check that sensor's SDA and SCL connections |

|  |  |
| --- | --- |
| **STEP 3** | **GPS Module Wiring (UART)** |
|  | * GPS module uses UART — not I2C — so it uses different pins * GPS TX → ESP32 GPIO 2 (ESP32 receives GPS data) * GPS RX → ESP32 GPIO 3 (ESP32 can send commands to GPS) * GPS VCC → 3.3V (check module datasheet — some require 5V) * GPS GND → GND * Take the unit outdoors; open Serial Monitor at 9600 baud; confirm NMEA sentences appear within 90 seconds |

**Lesson 2 – TFT Display & Faceplate Prep**

|  |  |
| --- | --- |
| **STEP 4** | **TFT Display Wiring (SPI)** |
|  | * The ILI9341 uses SPI — a faster protocol than I2C, needed for screen refreshes * VCC → 3.3V; GND → GND * SCK → GPIO 18; MOSI → GPIO 23; CS → GPIO 5; DC → GPIO 4; RST → GPIO 2 * Note: GPIO 2 is shared with GPS TX — if conflict arises, reassign RST to GPIO 15 * Install the Adafruit ILI9341 and Adafruit GFX libraries * Run the display test sketch: screen should show colour bands and text |

|  |  |
| --- | --- |
| **STEP 5** | **Faceplate Design & Mounting Layout** |
|  | * Sketch the instrument layout on paper using the Panel Layout Table (Section 5) * Mark positions on the faceplate: attitude indicator (centre top, largest), altimeter (left), heading (right) * Cut display window in faceplate; dry-fit the TFT behind the opening * Mark mounting holes for ESP32, breadboard, and GPS antenna window * Do not glue anything yet — confirm all wiring reaches before fixing components |

**Lessons 3–4 – Software Integration**

|  |  |
| --- | --- |
| **STEP 6** | **Install Libraries & Sensor Sketches** |
|  | * Install via Arduino Library Manager: Adafruit\_BMP280, MPU6050\_light, QMC5883LCompass, TinyGPS++, Adafruit\_ILI9341 * Test each sensor with its individual example sketch before combining * Confirm: BMP280 reads altitude in metres; MPU6050 returns pitch and roll; QMC5883L returns 0–360° heading; GPS returns lat/lon |

|  |  |
| --- | --- |
| **STEP 7** | **Upload the Integrated Cockpit Code** |
|  | * Open the cockpit\_panel.ino sketch from Kit 7 USB drive * Review the code structure: sensor reading loop → data processing → TFT draw function * Identify the complementary filter section: it combines gyroscope and accelerometer to prevent attitude drift * Upload to ESP32; confirm all 5 instrument values appear on the TFT display * If any reading is missing, check that the corresponding library is installed and the correct I2C address is set |

|  |  |
| --- | --- |
| **STEP 8** | **Compass Calibration** |
|  | * Open Serial Monitor; enter calibration mode as described in the kit sketch * Wave the unit in a slow figure-8 pattern for 30 seconds to capture the magnetic field range * Save calibration offsets; verify heading reads within ±15° of a reference compass * Keep compass module at least 5 cm from the ESP32 and any other electronics |

**Lessons 5–6 – Faceplate Assembly & Testing**

|  |  |
| --- | --- |
| **STEP 9** | **Mount Components on Faceplate** |
|  | * Secure TFT display behind the faceplate window using M2 screws or foam tape * Mount ESP32 and breadboard on the rear of the faceplate using double-sided tape * Route and bundle all wiring; secure with zip ties * GPS antenna must be on the top surface of the panel and face upward for best signal * Label all wire bundles with a marker: I2C, SPI, UART, Power |

|  |  |
| --- | --- |
| **STEP 10** | **Full System Test** |
|  | * Power on the panel; verify all 5 instruments display simultaneously * Complete the Test Verification Table (Section 6) * Take the panel outdoors for 90 seconds — confirm GPS acquires a fix and position is displayed * Tilt the panel: confirm attitude indicator updates; rotate: confirm heading updates * Each student produces a labelled wiring diagram of the complete circuit |

**5. Engineering Principles**

**Instrument Panel Functions**

| **Instrument** | **Sensor** | **Display Value** | **Real Aircraft Equivalent** |
| --- | --- | --- | --- |
| **Altimeter** | BMP280 Barometric Pressure | Altitude in metres (m) | Barometric altimeter (QNH/QFE) |
| **Attitude Indicator** | MPU6050 Gyro/Accel | Pitch (°) and Roll (°) | Artificial Horizon (ADI) |
| **Heading Indicator** | QMC5883L Compass | Magnetic heading (0–360°) | Directional Gyro / HSI |
| **Position** | NEO-6M GPS | Latitude, Longitude, Speed | GPS/GNSS navigation display |
| **Variometer** | BMP280 (rate of change) | Climb/Descent rate (m/s) | Vertical Speed Indicator (VSI) |

**Communication Protocols in Use**

* I2C (GPIO 21/22): Used for BMP280, MPU6050, and QMC5883L — suitable for slow sensor data; up to 127 devices on two wires
* SPI (GPIO 18/23/5/4): Used for the TFT display — fast protocol needed for screen pixel updates at 60+ Hz
* UART (GPIO 2/3): Used for GPS — streams NMEA sentence strings at 9600 baud; parsed by TinyGPS++ library

|  |
| --- |
| **The T-Scan Instrument Layout**  Real aircraft cockpits follow the Basic T layout: Attitude Indicator centre top, Airspeed left, Altimeter right, Heading Indicator below centre.  Pilots are trained to scan in a T pattern — primary instruments are always in the same position so they can be found instantly under stress.  Our panel uses the same principle: attitude indicator is the largest and most central instrument; altitude and heading flank it.  Consistent layout reduces pilot workload and is mandated by ICAO standards for certified aircraft. |

**Panel Layout Design**

| **Panel Zone** | **Instrument(s)** | **Position on Faceplate** | **Priority** |
| --- | --- | --- | --- |
| **Top Centre** | Attitude Indicator | Largest element; primary scan | 1 – Critical |
| **Top Left** | Altimeter | Upper left quadrant | 2 – Primary |
| **Top Right** | Heading Indicator | Upper right quadrant | 2 – Primary |
| **Bottom Left** | Variometer | Lower left quadrant | 3 – Secondary |
| **Bottom Centre** | GPS Position | Lower centre strip | 3 – Secondary |
| **Bottom Right** | System Status (GPS fix, battery) | Lower right corner | 4 – Advisory |

|  |
| --- |
| **Complementary Filter – Fixing Gyroscope Drift**  A gyroscope measures angular rate (how fast you are turning) — integrating this over time gives angle.  Problem: small errors accumulate over time (drift) — after 60 seconds, the attitude reading is wrong.  Solution: the accelerometer measures gravity direction and always knows 'down' — accurate but noisy.  Complementary filter: angle = 0.98 × (gyro\_angle) + 0.02 × (accel\_angle)  This blends 98% gyroscope (smooth, short-term accurate) with 2% accelerometer (long-term stable). |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **GPS no fix indoors** | Satellite signal blocked | Take unit outdoors; allow 60–90 s for cold start fix |
| **Compass heading incorrect** | Magnetic interference from nearby electronics | Calibrate with figure-8 waving motion; keep compass 5 cm from other modules |
| **TFT display blank** | Wrong SPI pin assignment or incorrect CS/DC wiring | Re-check SPI pins against wiring table; confirm CS and DC are not swapped |
| **I2C sensor not detected** | Address conflict or wiring error | Run I2C scanner; verify SDA/SCL; check for address jumper on BMP280 |
| **Altitude drifts with temperature** | BMP280 requires temperature compensation | Enable oversampling and IIR filter in BMP280 library settings |
| **Attitude drifts over time** | Gyroscope accumulates error (gyro drift) | Implement complementary filter: fuse gyroscope with accelerometer data |
| **GPS speed always 0** | No movement or fix not 3D | Move the unit outdoors; wait for 3D fix (needs 4+ satellites) |

**9. Upgrade & Extension Ideas**

* SD Card Data Logger – Add a MicroSD module; log all instrument readings with GPS timestamp to a CSV file for post-flight review
* Audio Alerts – Add a buzzer or speaker; trigger alerts when altitude drops below a threshold or attitude exceeds limits
* Graphical Attitude Ball – Draw a filled circle on the TFT that rolls and pitches as a physical artificial horizon indicator
* Moving Map Display – Use GPS coordinates to draw a simple dot-on-grid position map on a section of the TFT
* Wireless Telemetry – Use the ESP32 Wi-Fi to broadcast instrument data to a browser on a local network
* Battery Voltage Monitor – Add a voltage divider circuit to display remaining battery level on the panel

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* Pre-install all Arduino libraries on classroom computers before Lesson 3 — six libraries need to be installed and combined library conflicts can occur
* Run the I2C scanner sketch as the first software step in Lesson 1 — it immediately confirms all hardware is wired correctly
* The GPS cold start can take 2–5 minutes outdoors — plan Lesson 6 to include at least 10 minutes of outdoor testing time
* Build a completed working panel before the project begins — students need a concrete visual target

**Differentiation Strategies**

* Support – Pre-wired sensor board; students focus on software upload, calibration, testing, and wiring diagram only
* Core – Full wiring from scratch; library installation; compass calibration; faceplate mounting; complete test verification
* Extension – Graphical attitude ball; moving map display; SD card logger; wireless telemetry

**Assessment Suggestions**

* Wiring diagram: Complete, colour-coded, all pins and addresses labelled; signed off by teacher
* Test verification table: All 8 tests completed and results recorded
* Protocol explanation: Student correctly identifies which sensor uses I2C, SPI, or UART and explains why
* Compass calibration: Heading reads within ±15° of reference — measurable, objective outcome
