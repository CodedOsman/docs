**Student & Teacher Manual**

**1. Project Overview**

Weather is the single biggest external factor affecting aircraft performance, safety, and operations. Every pilot, dispatcher, and controller must understand temperature, pressure, humidity, wind speed, and wind direction — and how each variable affects the aircraft. In this project, students build a functional aviation weather station that measures all five parameters, displays them live on an OLED screen, and logs one week of data. They then calculate daily density altitude and decode a real METAR weather report from Kotoka International Airport.

**Learning Objectives**

* Identify the five weather parameters measured at an aviation weather station and name the correct instrument for each
* Wire a DHT22 (temperature/humidity), BMP280 (pressure), and cup anemometer to an ESP32
* Display all five parameters simultaneously on an OLED screen, updating every 30 seconds
* Log one week of weather data and graph temperature, pressure, and wind speed vs. time
* Calculate daily density altitude from logged temperature and pressure data
* Decode a real METAR weather report from Kotoka International Airport

**3. Build Steps & Assembly**

**Lesson 1 – Sensor Wiring**

|  |  |
| --- | --- |
| **STEP 1** | **Wire BMP280 & OLED (I2C)** |
|  | * BMP280: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * OLED: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * Run I2C scanner; confirm both detected (BMP280 at 0x76; OLED at 0x3C) * Upload BMP280 example; confirm pressure and temperature in Serial Monitor |

|  |  |
| --- | --- |
| **STEP 2** | **Wire DHT22** |
|  | * DHT22 VCC → 3.3V; GND → GND * DHT22 data pin → GPIO 4 * Add a 10kΩ resistor between the data pin and VCC (pull-up resistor — required) * Install DHT library; upload DHT22 example; confirm temperature and humidity readings |

|  |  |
| --- | --- |
| **STEP 3** | **Wire Anemometer** |
|  | * Anemometer Hall effect sensor: VCC → 3.3V; GND → GND; Signal → GPIO 34 (input-only pin) * Each cup revolution generates one pulse on the signal pin * Test: spin the anemometer by hand; Serial Monitor should show increasing pulse count * Wind speed calculation: pulses per second × cup circumference = wind speed in m/s |

**Lesson 2 – Software & Display**

|  |  |
| --- | --- |
| **STEP 4** | **Upload Weather Station Code** |
|  | * Open weather\_station.ino from Kit 9 USB drive * Review the display layout: Temperature (°C), Humidity (%), Pressure (hPa), Wind Speed (km/h) * Review the data logging section: every 30 minutes, values are printed to Serial (or SD card) * Upload; confirm all 4 values appear correctly on the OLED display |

|  |  |
| --- | --- |
| **STEP 5** | **Outdoor Installation** |
|  | * Mount the station in a shaded, ventilated location outdoors * Anemometer must be unobstructed — minimum 1 m above any surface * BMP280 and DHT22 must be shielded from direct sunlight (a small cup over the sensors works) * Connect a 5V power bank for extended outdoor logging |

**Lessons 3–4 – Data Collection & Analysis**

|  |  |
| --- | --- |
| **STEP 6** | **One Week of Data** |
|  | * Record readings 3 times per day: 08:00, 12:00, 16:00 * Enter readings into the data table (Section 6) and spreadsheet * After 5 days: graph temperature, pressure, and wind speed vs. time * Identify trends: does pressure drop before rain? Does temperature peak at 12:00? |

|  |  |
| --- | --- |
| **STEP 7** | **Density Altitude Calculation** |
|  | * For each 12:00 reading: calculate density altitude using the formula from Project 30 * Graph density altitude vs. date * Discussion: on the hottest, most humid day, what was the density altitude? * What would this mean for an aircraft taking off from Kotoka International Airport? |

|  |  |
| --- | --- |
| **STEP 8** | **METAR Decoding** |
|  | * Obtain a current METAR for DGAA (Kotoka International Airport) from aviationweather.gov * Example: METAR DGAA 121200Z 20010KT 9999 FEW018 28/22 Q1011 * Decode each element: station code, time (UTC), wind (200° at 10 knots), visibility (9999 m), cloud (few at 1800 ft), temperature (28°C), dew point (22°C), QNH (1011 hPa) * Compare METAR values to your station readings at the same time |

**5. Engineering Principles**

**Weather Parameters & Aviation Effects**

| **Parameter** | **Sensor** | **Unit** | **How It Affects Aircraft Performance** |
| --- | --- | --- | --- |
| **Temperature** | DHT22 / BMP280 | °C | Higher temp = lower air density = reduced lift and engine power |
| **Relative Humidity** | DHT22 | % | High humidity reduces effective air density (water vapour is lighter than dry air) |
| **Barometric Pressure** | BMP280 | hPa | Lower pressure = lower air density = higher density altitude |
| **Wind Speed** | Anemometer cup | km/h | Headwind helps takeoff; tailwind requires longer runway |
| **Wind Direction** | Wind vane / compass | ° | Crosswind component determines usable runway |

|  |
| --- |
| **METAR – The Global Aviation Weather Code**  METAR (Meteorological Aerodrome Report) is the international standard weather observation format.  Issued every 30 minutes at major airports; every hour at smaller stations.  Used by pilots for pre-flight weather briefing; by ATC for runway selection; by dispatchers for fuel calculation.  Every operational airport in Ghana, including Kotoka (DGAA), Kumasi (DGSI), and Tamale (DGLE), issues METAR reports.  Understanding a METAR is a core competency for all aviation personnel — pilots, controllers, engineers, and dispatchers. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **DHT22 reads 0 or -999** | Wrong GPIO or 10kΩ pull-up resistor missing | Add 10kΩ resistor between data pin and VCC; confirm correct GPIO |
| **Pressure drifts after power-on** | BMP280 needs warm-up time | Wait 2 minutes after power-on before recording; allow thermal stabilisation |
| **Anemometer reads 0 in wind** | Hall effect sensor wiring issue | Confirm magnet passes the sensor; check VCC and GND; test with multimeter |
| **Data disappears on power-off** | No SD card or logging not enabled | Add MicroSD module; log to file; alternatively use Serial to PC |
| **Readings vary wildly** | Station in direct sunlight | Mount station in shade or use a radiation shield over temperature sensor |

**9. Upgrade & Extension Ideas**

* Wind Direction: add a compass module (QMC5883L); display 8-point compass direction alongside wind speed
* Rain Gauge: add a tipping-bucket rain gauge; log rainfall in mm per day
* Solar Irradiance: add a BH1750 light intensity sensor; log irradiance in W/m² (connects directly to Project 35 solar aircraft power budget)
* ATIS Broadcast: write a simple code that generates an automatic ATIS (Automatic Terminal Information Service) report from the sensor data — students read the ATIS aloud

**10. Teacher Notes & Differentiation**

* The METAR decoding step (Step 8) can be done in class without any hardware — use a real current METAR from aviationweather.gov as a class decoding exercise
* For the density altitude calculation, prepare a worked example on the board first using that day's actual readings
* Support – OLED display only (no logging); 3 parameters only (temp, pressure, wind); teacher-guided METAR decoding
* Core – Full 5-sensor station; one week logging; graphs; density altitude; METAR decoding
* Extension – Wind direction; rain gauge; solar irradiance; ATIS broadcast
