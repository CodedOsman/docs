**Student & Teacher Manual**

**1. Project Overview**

Modern aircraft are full of gyroscopes and accelerometers — from the attitude indicator in the cockpit to the stability augmentation system in the flight control computer. This project wires an MPU6050 6-axis sensor to an ESP32 and displays pitch, roll, and yaw rate as live graphs on the Arduino Serial Plotter. Students directly observe how the sensor responds to aircraft-like motions, apply a complementary filter to smooth the output, and map each measurement to its real cockpit instrument equivalent.

**Learning Objectives**

* Wire an MPU6050 to an ESP32 and read all 6 axes: accelerometer X/Y/Z and gyroscope X/Y/Z
* Display pitch, roll, and yaw rate as live graphs on the Arduino Serial Plotter
* Apply a complementary filter to combine gyroscope and accelerometer data to remove noise and drift
* Complete a 7-motion data table by applying specific tilts and rotations and recording the readings
* Map all five cockpit attitude instruments to the corresponding MPU6050 output
* Explain why a gyroscope alone (without an accelerometer) produces a drifting attitude reference

**3. Session Steps**

**Lesson 1 – Wiring & Basic Readings**

|  |  |
| --- | --- |
| **STEP 1** | **Wire MPU6050 to ESP32** |
|  | * MPU6050 VCC → ESP32 3.3V * MPU6050 GND → ESP32 GND * MPU6050 SDA → ESP32 GPIO 21 * MPU6050 SCL → ESP32 GPIO 22 * Run I2C scanner sketch: confirm MPU6050 detected at address 0x68 * Install MPU6050\_light library via Arduino Library Manager |

|  |  |
| --- | --- |
| **STEP 2** | **Upload Basic Reading Sketch** |
|  | * Open the gyro\_visualiser.ino sketch from Kit 4 USB drive * This sketch reads and prints: AccX, AccY, AccZ, GyroX, GyroY, GyroZ at 20 Hz * Upload; open Serial Monitor at 115200 baud * Tilt the board in various directions; observe which values change * Key: AccZ ≈ 1.0 g when flat (gravity pulling down); GyroX/Y/Z ≈ 0 when stationary |

**Lesson 2 – Pitch, Roll & Complementary Filter**

|  |  |
| --- | --- |
| **STEP 3** | **Calculate Pitch & Roll from Accelerometer** |
|  | * Pitch = atan2(AccX, sqrt(AccY² + AccZ²)) × 180/π * Roll = atan2(AccY, sqrt(AccX² + AccZ²)) × 180/π * Add these calculations to the sketch; print Pitch and Roll on Serial * Open Serial Plotter (Tools → Serial Plotter): live pitch and roll graphs appear * Tilt the board: verify pitch increases when nose tilts up; roll when banking right |

|  |  |
| --- | --- |
| **STEP 4** | **Demonstrate Gyroscope Drift** |
|  | * Integrate the GyroX value over time to estimate pitch from gyro only: * gyro\_pitch += GyroX × dt (where dt = time step in seconds) * Hold board still for 30 seconds: observe gyro\_pitch slowly drifting away from zero * This demonstrates why a gyro alone cannot provide a long-term attitude reference * Now apply the complementary filter: pitch = 0.98 × (pitch + GyroX×dt) + 0.02 × accel\_pitch * Observe: the filtered pitch is smooth AND drift-free — the best of both sensors |

**Lesson 3 – Data Table & Instrument Mapping**

|  |  |
| --- | --- |
| **STEP 5** | **Complete the Motion Data Table** |
|  | * Place the MPU6050 flat on the table; confirm all readings near zero; record in Row 1 * Apply each motion from the data table (Section 6) one at a time; record all three values * Hold each position for 3 seconds before reading to allow the filter to settle * Compare expected vs. actual: pitch should read approximately +30° when nose tilted 30° up |

|  |  |
| --- | --- |
| **STEP 6** | **Cockpit Instrument Mapping** |
|  | * Using the Instrument Reference Table (Section 5), complete the mapping exercise: * 'The attitude indicator shows pitch and roll — which MPU6050 outputs does it use?' * 'The turn coordinator shows yaw rate — which gyro axis is this?' * 'The inclinometer (ball) shows if you are in a coordinated turn — which accelerometer axis?' * Students produce a hand-drawn diagram of the T-scan cockpit, labelling each instrument with its sensor source |

**5. Engineering Principles**

**Instrument to Sensor Mapping**

| **Real Aircraft Instrument** | **Axis Measured** | **Sensor in This Project** | **Display Output** |
| --- | --- | --- | --- |
| **Attitude Indicator (AI)** | Pitch + Roll | MPU6050 Accel + Gyro | Bar graph: pitch –90° to +90°; roll –180° to +180° |
| **Turn Coordinator** | Yaw rate + Roll | MPU6050 Gyro Z + Y | Rate needle: –3°/s (left) to +3°/s (right) |
| **Inclinometer (Ball)** | Lateral acceleration | MPU6050 Accel X | Ball position: left/right of centre |
| **Rate-of-Turn** | Yaw rate | MPU6050 Gyro Z | Numeric display: °/s |
| **Heading Indicator** | Yaw (relative) | MPU6050 Gyro Z integrated | Compass rose: 0–360° |

|  |
| --- |
| **Why Two Sensors Are Better Than One**  A gyroscope measures angular RATE (°/s) — integrating rate over time gives angle, but errors accumulate (drift).  An accelerometer measures acceleration in all three axes — when stationary, it measures gravity direction accurately, but it is noisy during motion.  The complementary filter combines both: gyro for fast, smooth changes; accelerometer for long-term correction.  This is identical to the sensor fusion algorithm inside the MPU6050 used in the Project 10 quadcopter's flight controller.  Advanced systems (Kalman Filter) do the same thing mathematically — the complementary filter is the simplified, intuitive version. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **All axes read 0** | I2C address wrong or wiring fault | Run I2C scanner; confirm 0x68 detected; check SDA GPIO 21 and SCL GPIO 22 |
| **Pitch/roll very noisy** | No filter applied | Apply complementary filter: angle = 0.98×(gyro) + 0.02×(accel) |
| **Gyro drifts after 30 seconds** | Normal gyroscope integration error | This demonstrates WHY the complementary filter is needed — show before and after |
| **Heading Indicator drifts to wrong value** | No magnetometer; gyro integration only | Explain: a real heading indicator uses a gyro + compass; this demo uses gyro only — drift is expected |
| **Serial Plotter shows flat line** | Wrong variable sent to Serial.println | Ensure pitch, roll, and yaw\_rate are on separate Serial.print lines separated by commas |

**9. Upgrade & Extension Ideas**

* Graphical Attitude Indicator: draw a pitch-and-roll display on the SSD1306 OLED from Project 9 — a moving horizon line
* Accelerometer-only stall detector: when AccZ drops below 0.7g during motion, trigger a buzzer alert — this simulates a basic g-force stall warning
* Vibration Analysis: mount the MPU6050 on the Project 10 drone motor; plot AccZ at different throttle levels — identify the resonance frequency of the frame
* Kalman Filter: implement a Kalman filter instead of the complementary filter; compare noise and drift performance

**10. Teacher Notes & Differentiation**

* The Serial Plotter (not Serial Monitor) is the visual tool that makes this project compelling — demonstrate it at the front of the room with a large screen before students try individually
* The gyroscope drift demonstration (Step 4) is the most important conceptual moment — let students observe the drift for 60 seconds before revealing the filter solution
* Support – Wiring and Serial Monitor only; teacher provides pitch/roll formulas pre-coded; students read data and fill data table
* Core – Full wiring; pitch/roll calculation; complementary filter; data table; instrument mapping diagram
* Extension – OLED attitude display; stall detector; vibration analysis; Kalman filter
