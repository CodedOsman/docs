**Student & Teacher Manual**

**1. Project Overview**

This project introduces students to avionics — the electronic systems that form the nervous system of a modern aircraft. Using an ESP32 microcontroller, an ultrasonic sensor (simulating an altimeter), a gyroscope/accelerometer (simulating an attitude indicator), and an OLED display, students build a functional instrument panel that mirrors the core flight instruments found in every cockpit. This is the first project in the Intermediate section.

**Learning Objectives**

* Identify the three primary flight instruments simulated: altimeter, attitude indicator, and master caution system
* Wire an ESP32 breadboard circuit correctly using the pin connection table
* Explain the I2C communication protocol and why the OLED and gyroscope share two wires
* Upload code to the ESP32 that reads sensor data and displays it on the OLED
* Configure alert thresholds and verify warning LEDs and buzzer trigger correctly
* Produce a labelled wiring diagram of the complete avionics bay circuit

**3. Build Steps & Assembly**

**Lesson 1 – Breadboard Setup & Wiring**

|  |  |
| --- | --- |
| **STEP 1** | **Breadboard Power Rail Setup** |
|  | * Place ESP32 across the centre gap of the breadboard (pins on both sides) * Connect ESP32 3V3 pin → breadboard positive rail (red) * Connect ESP32 5V pin → a separate breadboard row (for ultrasonic sensor only) * Connect ESP32 GND pin → breadboard negative rail (blue) * Add a second GND jumper from the negative rail for redundancy |

|  |  |
| --- | --- |
| **STEP 2** | **Ultrasonic Sensor Wiring** |
|  | * Insert HC-SR04 into the breadboard * VCC → 5V row (HC-SR04 requires 5V – not 3.3V) * GND → GND rail * TRIG → GPIO 2 on ESP32 * ECHO → GPIO 4 on ESP32 * Verify against the Wiring Reference Table in Section 5 |

|  |  |
| --- | --- |
| **STEP 3** | **MPU6050 & OLED Display Wiring (I2C)** |
|  | * Both MPU6050 and OLED use I2C — they share the same SDA and SCL lines * MPU6050: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * OLED: VCC → 3.3V; GND → GND; SDA → GPIO 21; SCL → GPIO 22 * Both devices connect to the SAME two wires — this is how I2C works * Each device has a unique address so the ESP32 knows which to query |

|  |  |
| --- | --- |
| **STEP 4** | **LED & Buzzer Wiring** |
|  | * Insert one 330Ω resistor per LED into the breadboard * Red LED: resistor → GPIO 5; LED anode (long leg) to resistor; cathode → GND * Green LED: resistor → GPIO 18; anode; cathode → GND * Yellow LED: resistor → GPIO 19; anode; cathode → GND * Buzzer: positive leg → GPIO 23; negative leg → GND * Test: all three LEDs should light when GPIOs set HIGH in a test sketch |

**Lesson 2 – Software & Code Upload**

|  |  |
| --- | --- |
| **STEP 5** | **Install Arduino IDE & Libraries** |
|  | * Install Arduino IDE 2.x on classroom computer * Add ESP32 board support via Board Manager URL: https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package\_esp32\_index.json * Install libraries via Library Manager: Adafruit\_SSD1306, Adafruit\_GFX, MPU6050\_light * Select board: ESP32 Dev Module; select correct COM port |

|  |  |

| **STEP 6** | **Upload & Test the Code** |
|  | * Open avionics\_bay.ino from Kit 4 USB drive * Review code with students: identify sensor reading, alert logic, and display sections * Upload to ESP32; open Serial Monitor at 115200 baud to verify sensor readings * OLED should show: ALT: XXcm | PITCH: X° | ROLL: X° * If OLED is blank: run I2C scanner sketch to find the correct display address |

**Lessons 3–4 – Calibration, Testing & Wiring Diagram**

|  |  |
| --- | --- |
| **STEP 7** | **Sensor Calibration** |
|  | * Place ESP32 assembly flat on a level surface * Check Serial Monitor: pitch and roll should read near 0° * If offset exists, note values and adjust pitch\_offset and roll\_offset variables in code * Hold hand 100 cm above ultrasonic sensor; reading should show ~100 cm * Test at 30 cm and 15 cm; confirm alert triggers at < 30 cm |

|  |  |
| --- | --- |
| **STEP 8** | **Alert System Verification** |
|  | * Slowly move hand toward sensor; confirm Red LED + buzzer activate at 30 cm * Tilt sensor board to 30°; confirm Yellow LED + buzzer activate * Return to flat and all-clear: Green LED illuminates; buzzer off * Record each test result in the Test Verification Table (Section 6) |

|  |  |
| --- | --- |
| **STEP 9** | **Produce the Wiring Diagram** |
|  | * Each student draws a complete, labelled wiring diagram on A4 paper * Colour coding: red = 3.3V, orange = 5V, black = GND, other colours = signal wires * Label every component and every GPIO number * Compare with partner to check accuracy; teacher signs off completed diagram |

**5. Engineering Principles**

**Avionics Concepts**

* Altimeter: Measures altitude above ground level (AGL). Real aircraft use barometric pressure sensors. We simulate with ultrasonic distance to a surface below
* Attitude Indicator (AI): Shows pitch and roll relative to the horizon. The MPU6050 gyroscope measures angular acceleration in all three axes
* Master Caution System: In real cockpits, a Master Caution light alerts the pilot when a system exceeds a limit. Our LED and buzzer system mirrors this

|  |
| --- |
| **I2C Protocol – How Two Devices Share Two Wires**  I2C (Inter-Integrated Circuit) allows multiple devices on just two wires: SDA (data) and SCL (clock).  Each device has a unique address (e.g. OLED = 0x3C, MPU6050 = 0x68).  The ESP32 (master) broadcasts an address; only the matching device (slave) responds.  This is why OLED and MPU6050 can share GPIO 21 and GPIO 22 without conflict. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **OLED blank after power-on** | Wrong I2C address | Run I2C scanner sketch; use found address in code |
| **Altitude always reads 0** | TRIG/ECHO pins swapped | Swap GPIO 2 and GPIO 4 connections |
| **Gyroscope shows no data** | SDA/SCL not connected | Check GPIO 21 (SDA) and 22 (SCL) |
| **LEDs do not light** | Anode/cathode reversed | Flip LED; long leg (anode) to GPIO pin |
| **Buzzer silent** | Wrong GPIO or passive buzzer | Confirm GPIO 23; must use active buzzer |
| **Code will not upload** | USB driver missing | Install CP2102 or CH340 driver; restart IDE |

**9. Upgrade & Extension Ideas**

* Temperature Display – Add a DHT11 sensor; display temperature and humidity alongside altitude and attitude
* Heading Indicator – Add an HMC5883L compass module (also I2C); display compass heading on the OLED
* Data Logger – Add a MicroSD card module; log all sensor readings to a CSV file for post-flight analysis
* Bluetooth Telemetry – Send live data to a smartphone via the ESP32's built-in Bluetooth module
* Physical Cockpit Panel – Mount all components in a cardboard or 3D-printed instrument panel fascia

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* Pre-install Arduino IDE and all libraries on classroom computers before Lesson 2 — library downloads can fail on slow connections
* Prepare a pre-wired demonstration circuit — students see the finished product at the start of Lesson 1
* Colour-coded jumper wires are essential — display a wire colour guide on the whiteboard
* Pairs work best for wiring; individuals for coding and wiring diagram

**Differentiation Strategies**

* Support – Pre-wired kit; students focus on code upload, calibration, and testing only
* Core – Full wiring, code upload, calibration, alert verification, wiring diagram
* Extension – Additional sensors; data logging; Bluetooth telemetry; physical cockpit panel

**Assessment Suggestions**

* Wiring diagram: Complete, labelled, colour-coded; all GPIO numbers correct
* Test verification table: All 7 tests completed and results recorded
* I2C explanation: Student explains protocol and alert logic in own words
* Code comprehension: Student can identify and modify the alert threshold values
