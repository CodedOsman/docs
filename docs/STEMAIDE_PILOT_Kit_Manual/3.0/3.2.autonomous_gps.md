**Student & Teacher Manual**

**1. Project Overview**

This project marks the transition from manual piloting to autonomous operations — one of the most significant developments in modern aviation. Students install a GPS module on the Project 10 quadcopter, plan a 4-waypoint mission using ArduPilot Mission Planner software, upload the mission to the flight controller, and observe the drone fly the mission hands-free. Safety failsafes (Return to Launch, GeoFence, battery protection) are configured and verified before any autonomous flight takes place.

**Learning Objectives**

* Install and orient a GPS module on the quadcopter flight controller
* Install ArduPilot Mission Planner and connect the quadcopter via USB telemetry
* Plan a 4-waypoint GPS mission with altitude assignments using Mission Planner's map interface
* Upload the mission to the flight controller and verify it on the ground before flight
* Configure and verify three failsafe types: Return to Launch, GeoFence, and low-battery RTL
* Execute a complete autonomous mission and review the flight log in Mission Planner

**3. Build Steps & Assembly**

**Lesson 1 – GPS Hardware Installation**

|  |  |
| --- | --- |
| **STEP 1** | **GPS Module Mounting** |
|  | * Mount the GPS module on the GPS mast; position the mast at the rear of the drone frame * GPS antenna must face upward; ensure nothing obstructs the sky view from the antenna * Keep the GPS module at least 5 cm from ESCs, motors, and the VTX (if fitted) to minimise magnetic interference * Route GPS cable through the frame; secure with zip ties; leave enough slack for vibration |

|  |  |
| --- | --- |
| **STEP 2** | **GPS Connection to Flight Controller** |
|  | * Connect GPS UART TX → FC GPS RX pad * Connect GPS UART RX → FC GPS TX pad * Connect compass SDA → FC I2C SDA; compass SCL → FC I2C SCL * Connect GPS VCC → 5V pad; GPS GND → GND pad * In ArduPilot / Betaflight: enable GPS UART on the correct port; set baud rate to 115200 |

**Lesson 2 – Mission Planner Setup**

|  |  |
| --- | --- |
| **STEP 3** | **Install & Connect Mission Planner** |
|  | * Install ArduPilot Mission Planner from ardupilot.org/planner on the classroom computer * Flash ArduPilot Copter firmware to the flight controller using Mission Planner's Install Firmware page * Connect drone to computer via USB; select correct COM port in MP (top right); connect at 115200 baud * Verify: HUD (Heads-Up Display) shows live attitude data; GPS Status shows satellite count |

|  |  |
| --- | --- |
| **STEP 4** | **Mandatory Calibrations in Mission Planner** |
|  | * Accelerometer calibration: Initial Setup → Mandatory Hardware → Accel Calibration; follow on-screen instructions * Compass calibration: Mandatory Hardware → Compass; perform onboard calibration (rotate drone in figure-8 pattern outdoors) * Radio calibration: Mandatory Hardware → Radio Calibration; move all sticks to their extremes * ESC calibration: Mandatory Hardware → ESC Calibration (if not done previously) * Take drone outdoors; confirm GPS 3D Fix achieved (green GPS icon in Mission Planner) |

**Lesson 3 – Mission Planning**

|  |  |
| --- | --- |
| **STEP 5** | **Plan the Waypoint Mission** |
|  | * Open Flight Plan tab in Mission Planner; the satellite map shows current GPS location * Right-click on the map at the desired WP 1 position; select 'Add Way Point' * Set altitude for WP 1: 10 m above home point * Repeat for WP 2 (15 m), WP 3 (10 m), WP 4 (15 m) * Right-click at home point and select 'Add Return to Launch' as the final command * Review the mission path shown on the map — all waypoints should form a logical route |

|  |  |
| --- | --- |
| **STEP 6** | **Configure Failsafes** |
|  | * In Mission Planner Config tab → Failsafe: * RC Failsafe: set to RTL; trigger after 1 second of signal loss * Battery Failsafe: set voltage threshold to 3.5V per cell; action = RTL * In GeoFence tab: enable circular fence; radius = 50 m from home; breach action = RTL * Test RC failsafe: arm drone; hover at 1 m; teacher briefly powers off transmitter; confirm drone initiates RTL |

**Lessons 4–5 – Mission Upload & Autonomous Flight**

|  |  |
| --- | --- |
| **STEP 7** | **Upload Mission to Flight Controller** |
|  | * In Flight Plan tab: click Write WPs button; Mission Planner confirms successful upload * Click Read WPs to verify the stored mission matches the planned mission * Switch to Auto mode on the transmitter mode switch * Verify in Mission Planner: current mode shows AUTO; mission waypoints highlighted |

|  |  |
| --- | --- |
| **STEP 8** | **Execute the Autonomous Mission** |
|  | * Teacher connects LiPo battery; drone in AUTO mode * Students and observers stand at least 10 m behind the pilot position * Pilot advances throttle past mid-stick — drone takes off automatically * Students observe: drone should fly to each waypoint, pause briefly, then continue * Do NOT touch sticks during autonomous flight unless the drone behaves unexpectedly * To abort: flip mode switch to Stabilize; pilot takes manual control immediately |

|  |  |
| --- | --- |
| **STEP 9** | **Post-Mission Log Review** |
|  | * Connect drone to computer after landing; open Flight Data tab in Mission Planner * Load the onboard log file (BIN format); review GPS track on the map * Identify: how close to each waypoint did the drone actually fly? * Compare intended vs actual track; discuss causes of any deviation * Complete the Mission Log Table (Section 6) |

**4. Power & Safety Notes**

|  |
| --- |
| **⚠ CRITICAL – AUTONOMOUS FLIGHT SAFETY**  LiPo battery: Teacher connects and disconnects only. Autonomous drones can move unexpectedly.  Arm only outdoors with GPS 3D fix (8+ satellites). Never arm in AUTO mode without a GPS fix.  Manual override: Mode switch must be accessible at all times. Practice switching to Stabilize before any auto flight.  GeoFence: MUST be configured and verified before the first autonomous flight.  Flight zone: Minimum 50 m × 50 m clear area. All observers behind pilot and outside fence boundary.  Never step in front of an autonomously flying drone — it will not see you and will not deviate. |

**5. Engineering Principles**

**How GPS Navigation Works**

* The GPS module receives signals from 4+ satellites to calculate its position to within 1–3 metres
* The flight controller compares current GPS position to the next waypoint's GPS coordinates
* It calculates the required heading and distance, then applies throttle, pitch, and roll to move toward the waypoint
* This loop runs approximately 400 times per second — far faster than any human pilot

|  |
| --- |
| **Coordinate Systems in UAV Navigation**  GPS uses WGS84 coordinates: Latitude (north/south, -90° to +90°) and Longitude (east/west, -180° to +180°).  Altitude in ArduPilot is measured in metres above the home point (takeoff location), not sea level.  Mission Planner displays coordinates in decimal degrees: Accra is approximately 5.5600° N, -0.2057° E.  GPS accuracy of 1–3 m is sufficient for waypoint navigation but NOT for precision landing — that requires additional sensors. |

**Failsafe Systems**

| **Failsafe Type** | **Trigger Condition** | **Drone Action** | **Setting Location** |
| --- | --- | --- | --- |
| **Return to Launch (RTL)** | RC signal lost > 1 s | Climbs to RTL altitude; flies home; lands | Betaflight Failsafe tab |
| **Low Battery RTL** | Battery voltage < 3.5V/cell | Initiates RTL automatically | ArduPilot Battery Monitor |
| **GeoFence Breach** | Drone exits defined boundary | Returns to home point | ArduPilot GeoFence settings |
| **GPS Loss** | GPS fix lost in auto mode | Switches to manual; pilot takes control | ArduPilot GPS Glitch |
| **RC Override** | Pilot flips mode switch | Immediately exits auto; pilot controls | Mode switch on transmitter |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **GPS no 3D fix** | Insufficient satellites or indoors | Move outdoors; wait 2–3 minutes; ensure GPS antenna faces up |
| **Drone won't arm in Auto mode** | GPS not ready or compass not calibrated | Wait for 3D fix; complete compass calibration in MP |
| **Mission upload fails** | USB connection unstable | Use a data USB cable; try a different port; restart Mission Planner |
| **Drone drifts from waypoints** | Compass interference or calibration error | Redo compass calibration away from motors and electronics |
| **Drone won't return to exact home** | GPS accuracy is ±1–3 m | This is normal GPS accuracy; expected landing within 2 m of home |
| **Mission runs wrong waypoint sequence** | Waypoints uploaded out of order | Delete all waypoints; re-enter in correct order; re-upload |
| **Failsafe not triggering** | Failsafe not configured | Set failsafe in Betaflight Failsafe tab; test with RC off |

**9. Upgrade & Extension Ideas**

* Survey Grid Mission – Plan a lawnmower-pattern grid mission over a 50 m × 50 m area; simulate aerial survey
* Follow Me Mode – Enable Follow Me in Mission Planner; drone tracks pilot's phone GPS position
* Telemetry Radio – Add a 433 MHz or 915 MHz telemetry radio; view live GPS track in Mission Planner during flight
* Payload Drop at Waypoint – Add a DO\_SET\_SERVO command at WP 3 to trigger a servo release at a specific waypoint
* Flight Log Analysis – Import BIN log into UAV Log Viewer; graph altitude, speed, and GPS accuracy over the mission

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* Install Mission Planner and flash ArduPilot firmware on the FC before Lesson 2 — firmware flashing can take 20 minutes and requires a good USB connection
* The compass calibration must be done outdoors, away from buildings and vehicles — allow time for this in Lesson 2
* Practice the manual override (mode switch to Stabilize) with each pilot before the autonomous mission — they must be able to do it without looking at the transmitter
* View the flight log together as a class after the first autonomous mission — the map view showing actual vs planned track is always surprising and engaging

**Differentiation Strategies**

* Support – Teacher pre-plans the mission; students focus on upload, failsafe verification, and flight observation; manual override drill only
* Core – Full mission planning, upload, failsafe config, autonomous mission, log review
* Extension – Survey grid; Follow Me mode; telemetry radio; payload drop command; log analysis
