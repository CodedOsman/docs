**Student & Teacher Manual**

**1. Project Overview**

In real aviation, an aircraft that has not been properly maintained cannot legally fly. Every inspection, every repair, and every parts replacement must be recorded in a maintenance log — and the log must be signed by a licensed engineer. This project creates a complete maintenance system for either the Project 10 quadcopter or the Project 11 RC aircraft: a 12-month maintenance schedule with 5 maintenance types, a set of maintenance tasks performed and signed off, and a completed maintenance log that mirrors the logbooks used by Ghana's licensed Aircraft Maintenance Engineers.

**Learning Objectives**

* Classify the five types of aircraft maintenance: pre-flight, post-flight, periodic, annual, and on-condition
* Create a 12-month maintenance schedule for the Project 10 or 11 aircraft, assigning each task to the correct maintenance type
* Perform and sign off 8 specific maintenance tasks in the correct sequence using the task reference table
* Complete a maintenance log with the required fields: date, task, parts replaced, hours added, and signatures
* Explain the role of a CAMO (Continuing Airworthiness Management Organisation) in managing aircraft maintenance
* Identify the GCAA regulation and ICAO Annex that govern UAV maintenance in Ghana

**3. Session Steps**

**Lesson 1 – Maintenance Theory & Schedule Creation**

|  |  |
| --- | --- |
| **STEP 1** | **Maintenance Types Classification** |
|  | * Review the Maintenance Types Reference Table (Section 5) * Students classify 10 maintenance scenarios as pre-flight, post-flight, periodic, annual, or on-condition: * Scenario 1: 'The aircraft just landed after a hard touchdown' → On-condition * Scenario 2: 'The aircraft has accumulated 50 flying hours' → Periodic * Scenario 3: 'The pilot is about to fly for the first time today' → Pre-flight * Scenario 4: 'The aircraft has been in service for one year' → Annual |

|  |  |
| --- | --- |
| **STEP 2** | **Create the Maintenance Schedule** |
|  | * Using the Task Reference Table (Section 5), assign each task to its maintenance type * Create a 12-month calendar: mark pre-flight tasks (every flight), post-flight (every flight), 50-hr periodic (estimated at every 2 months for 25 hr/month), and annual * Add on-condition triggers: hard landing, crash, any unusual vibration, water exposure * Teacher reviews the completed schedule before maintenance tasks begin |

**Lesson 2 – Maintenance Tasks & Log Entries**

|  |  |
| --- | --- |
| **STEP 3** | **Perform Tasks T-01 to T-04** |
|  | * T-01: Inspect all frame arms for cracks; record findings in the log * T-02: Check all motor mount screws with the M3 hex driver; tighten any that are loose; record * T-03: Clean motor windings with compressed air; photograph before and after; record * T-04: Inspect all ESC solder joints under magnification; check for cold joints or cracking; record * Each task: student signs 'Technician' column; teacher countersigns 'Approved By' column before proceeding to next task |

|  |  |
| --- | --- |
| **STEP 4** | **Perform Tasks T-05 to T-08** |
|  | * T-05: Load-test the LiPo battery using the watt meter; record voltage under 10A load vs. open circuit * T-06: Calibrate the compass using Betaflight / ArduPilot compass calibration wizard; record new offsets * T-07: Inspect and replace any damaged propellers; record propeller part number and reason for replacement * T-08: Measure elevator, aileron, and rudder deflection angles using the protractor; record measured vs. designed values |

**Lesson 3 – CAMO Framework & Regulations**

|  |  |
| --- | --- |
| **STEP 5** | **Complete the Maintenance Log** |
|  | * Review all 8 log entries: every entry must have date, task reference, description, parts (or 'none'), hours added, and two signatures * Calculate total maintenance hours accumulated; update the running total * Identify any defect that could affect airworthiness: record as 'OPEN DEFECT' until rectified * Teacher performs a final log review; signs the maintenance log cover page |

|  |  |
| --- | --- |
| **STEP 6** | **CAMO Concept & GCAA Regulations** |
|  | * Definition: A CAMO (Continuing Airworthiness Management Organisation) plans and manages all maintenance for an aircraft or fleet * In commercial aviation, no aircraft flies without a current CAMO maintenance release * For UAVs in Ghana: GCAA Civil Aviation Regulations Part 33 (Remotely Piloted Aircraft) requires maintenance records for commercial UAV operations * ICAO Annex 6 (Aircraft Operations) Part I requires maintenance release documentation before every commercial flight * Students write a 1-page summary: 'What is a CAMO, why is maintenance logging important, and what GCAA regulation applies to our aircraft?' |

**4. Power & Safety Notes**

|  |
| --- |
| **⚠ Maintenance Safety**  Power: LiPo battery must be disconnected before any maintenance task. Teacher reconnects only for the post-maintenance functional test.  Propellers: REMOVE all propellers before any electrical or mechanical maintenance.  Solvents: If using electrical contact cleaner for motor cleaning, ensure adequate ventilation.  Log accuracy: A maintenance log with false or missing entries is a serious safety violation in real aviation — treat this logbook with the same integrity expected of a professional engineer. |

**5. Engineering Principles**

**Maintenance Types Reference**

| **Type** | **Interval** | **Description** | **Example Tasks** |
| --- | --- | --- | --- |
| **Pre-flight** | Before every flight | Visual and functional inspection | 10-point checklist from Project 5; control surface check |
| **Post-flight** | After every flight | Damage check; log unusual events | Inspect for cracks, loose joints, prop damage; update log |
| **Periodic (50 hr)** | Every 50 flying hours | Deeper inspection of key systems | Re-tension all screws; check ESC solder joints; clean motors |
| **Annual** | Every 12 months | Full structural and systems inspection | Measure all deflections; check battery health; replace worn parts |
| **On-condition** | When triggered by event | Inspect after hard landing, crash, or unusual behaviour | Post-crash structural check; motor bearing test; frame inspection |

**Task Reference Table**

| **Task Ref** | **Task Description** | **Category** | **Special Tools** | **Est. Time** |
| --- | --- | --- | --- | --- |
| **T-01** | Inspect all frame arms for cracks and deformation | Pre-flight | Naked eye | 2 min |
| **T-02** | Check all motor screws for tightness | Post-flight / 50 hr | M3 hex driver | 3 min |
| **T-03** | Clean motor windings with compressed air | 50 hr | Air duster | 5 min |
| **T-04** | Check ESC solder joints under magnification | 50 hr | Magnifier, multimeter | 10 min |
| **T-05** | Load-test LiPo battery under 10A draw | Annual | Watt meter | 5 min |
| **T-06** | Calibrate compass after any crash | On-condition | Flight controller software | 5 min |
| **T-07** | Replace propellers | On-condition | Prop wrench | 3 min |
| **T-08** | Measure and record control surface deflection angles | Annual | Protractor | 10 min |

|  |
| --- |
| **Why Maintenance Logging Matters**  The 1988 Aloha Airlines Flight 243 accident — where a large section of fuselage separated in flight — was partly attributed to inadequate maintenance record-keeping and inspection procedures.  The accident directly led to ICAO strengthening Annex 6 maintenance documentation requirements.  Every aircraft with a valid Certificate of Airworthiness has a complete, unbroken maintenance log dating from its first flight.  Ghana's AMEs (Aircraft Maintenance Engineers) are required to sign maintenance entries under GCAA Part 66 — their licence is at risk if entries are inaccurate. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Log entries vague** | No specific task description | Use the exact task reference (T-01 etc.) and describe findings: 'No cracks found; all joints secure' |
| **Maintenance type misclassified** | Confusion between on-condition and periodic | On-condition is triggered by an EVENT (crash, warning); periodic is triggered by TIME or HOURS |
| **No defects found** | Task performed too quickly | Spend the full allocated time; use a magnifier for ESC joints; defects are usually there if you look carefully |
| **Signature missing** | Sign-off step overlooked | Every task must be signed by the technician AND countersigned by teacher before the log is accepted |
| **Hours not tracked** | Flight log not maintained | Start a flight log for the Project 10 or 11 aircraft immediately; record duration of every flight |

**9. Upgrade & Extension Ideas**

* Digital Maintenance Log: recreate the logbook as a Google Sheet or simple database; practice digital record-keeping
* Time Between Overhaul (TBO): research what TBO means for aircraft engines; calculate the TBO for a Lycoming O-320 engine vs. the ESC in this kit
* Maintenance Schedule Software: use a free maintenance management app (e.g. Flightdocs free tier) to schedule and track the 8 tasks
* Real AME Interview: invite a licensed GCAA AME to class; ask about a real maintenance decision they had to make

**10. Teacher Notes & Differentiation**

* The signature requirement is non-negotiable and must be enforced — it teaches students that a maintenance entry without a signature is worthless in real aviation
* Show the Aloha Airlines 1988 video footage before Lesson 1 — it makes the maintenance logging requirement viscerally real
* Support – 4 tasks only (T-01, T-02, T-03, T-07); pre-printed log with first two rows completed as examples
* Core – All 8 tasks; complete log; CAMO summary; schedule creation
* Extension – Digital log; TBO research; maintenance management software; AME interview
