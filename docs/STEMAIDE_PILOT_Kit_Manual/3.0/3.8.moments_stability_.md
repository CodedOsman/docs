**Student & Teacher Manual**

**1. Project Overview**

Flight stability and control are governed by moments — the product of forces and their distances from the aircraft's axes of rotation. This project brings together the physics of moments from the classroom into a direct aviation context: students calculate pitching, rolling, and yawing moments for a real aircraft scenario, verify the calculations on the Project 8 balance beam, and demonstrate all three control axes live using the Project 27 control surface demonstrator. The final deliverable is a correctly annotated three-view drawing showing all control surfaces and their axes.

**Learning Objectives**

* Define the three axes of an aircraft: longitudinal (roll), lateral (pitch), and normal (yaw)
* Identify the control surface that governs rotation about each axis: ailerons/roll, elevator/pitch, rudder/yaw
* Calculate pitching moments using Moment = Force × Perpendicular Distance from the axis
* Verify that the sum of pitching moments equals zero for a trimmed (level) aircraft
* Demonstrate all three axes of rotation using the Project 27 control surface demonstrator
* Produce a correct three-view orthographic drawing of an aircraft, labelling all control surfaces and axes

**3. Session Steps**

**Lesson 1 – Three Axes & Moment Theory**

|  |  |
| --- | --- |
| **STEP 1** | **Three Axes of Flight – Physical Demonstration** |
|  | * Set up the Project 27 demonstrator on the desk * Teacher moves the aileron servo: class observes the roll axis (wing tips move up/down) * Teacher moves the elevator servo: class observes the pitch axis (nose moves up/down) * Teacher moves the rudder servo: class observes the yaw axis (nose moves left/right) * Students draw a front-view and side-view of the aircraft; label all three axes with arrows * Verify: roll axis passes through the aircraft nose-to-tail; pitch axis passes left-right through the wings; yaw axis passes vertically through the fuselage |

|  |  |
| --- | --- |
| **STEP 2** | **Moment Calculation Theory** |
|  | * Review the lever principle from physics: Moment (N·m) = Force (N) × Perpendicular Distance (m) * For an aircraft: the centre of gravity (CG) is the pivot point for all moment calculations * A force ahead of the CG creates a nose-down pitching moment; behind the CG = nose-up * For level, trimmed flight: Sum of all pitching moments = 0 (clockwise = counter-clockwise) * Complete the Moment Calculation Worksheet (Section 6) using the four scenarios in the table |

|  |  |
| --- | --- |
| **STEP 3** | **Balance Beam Verification** |
|  | * Use the Project 8 balance beam to model a pitching moment physically * Place a 50 g weight at 10 cm from the pivot: moment = 50 × 10 = 500 g·cm * Balance it with a 25 g weight at the other side: what distance is needed? (20 cm) * Verify: 25 × 20 = 500 g·cm — moments balance * Translate: this models a tailplane downforce balancing the lift-generated pitching moment |

**Lessons 2–3 – Three-View Drawing**

|  |  |
| --- | --- |
| **STEP 4** | **Three-View Orthographic Drawing** |
|  | * A three-view drawing shows: FRONT view, SIDE view (from the right), and TOP view * All three views are aligned: side view top edge aligns with front view top edge; side view length aligns with top view length * Start with the side view: fuselage profile, wing position, tail position * Project lines left for the front view: shows wingspan, dihedral angle, fuselage cross-section * Project lines down for the top view: shows wing planform, tail planform, fuselage width |

|  |  |
| --- | --- |
| **STEP 5** | **Label All Control Surfaces & Axes** |
|  | * On the completed three-view drawing, label every control surface with its name and axis * Ailerons: label on the top view; draw a dashed line for the roll axis (nose to tail) * Elevator: label on the side view; draw a dashed line for the pitch axis (left to right) * Rudder: label on the front view; draw a dashed line for the yaw axis (top to bottom) * Add arrows showing the direction of rotation each surface produces |

**5. Engineering Principles**

**The Three Axes of Flight**

| **Axis** | **Name** | **Motion** | **Control Surface** | **Positive Direction** |
| --- | --- | --- | --- | --- |
| **Longitudinal** | Roll axis | Rolling left or right | Ailerons | Right wing down |
| **Lateral** | Pitch axis | Nose up or nose down | Elevator | Nose up |
| **Normal** | Yaw axis | Nose left or nose right | Rudder | Nose right |

|  |
| --- |
| **Moment Equilibrium in a Trimmed Aircraft**  A trimmed aircraft flies with no tendency to pitch, roll, or yaw without control input.  This requires all moments about each axis to sum to zero.  Pitching: the tail's downforce × tail moment arm = wing lift offset × CG-to-AC distance.  If CG moves forward: nose-down moment increases; pilot must add back pressure to maintain level flight.  If CG moves aft: nose-up moment increases; aircraft becomes less stable and may pitch up uncontrollably.  This is why the load-and-trim sheet from Project 8 is checked before every commercial flight. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Pitching moment calculated wrong** | Force and moment arm not perpendicular | Draw a diagram first; confirm the arm is measured perpendicular to the force vector |
| **Confused roll and yaw** | Names learned without physical reference | Point to each axis on the Project 27 demonstrator while naming it; physical beats verbal |
| **Balance beam does not match calculation** | Different scale of forces | This is expected — verify the ratio, not the absolute value |
| **Three-view drawing inaccurate** | Orthographic projection not understood | Show a worked example of a box in three views before attempting the aircraft |

**9. Upgrade & Extension Ideas**

* Rolling Moment Calculation: calculate the rolling moment produced by differential aileron deflection on the Project 11 aircraft
* Stability Margin: measure the CG position and aerodynamic centre position on the Project 11 aircraft; calculate the static margin as a percentage of mean aerodynamic chord
* Dutch Roll Demonstration: in the RC simulator, set the CG to a very aft position; observe the Dutch roll oscillation; explain it using yaw and roll coupling

**10. Teacher Notes & Differentiation**

* The physical demonstration with the Project 27 demonstrator (Step 1) must come before any calculation — students who understand the physical motion first learn the maths much more easily
* For the three-view drawing, show a completed example of a simple box in three views first — orthographic projection is unfamiliar to many students
* Support – Three-axis identification and labelling only; one moment calculation scenario; no three-view drawing
* Core – All three axes; four moment scenarios; balance beam verification; complete three-view drawing
* Extension – Rolling moment calculation; stability margin measurement; Dutch roll demonstration
