**Student & Teacher Manual**

**1. Project Overview**

This project adds a servo-operated cargo release mechanism to the Project 11 RC plane, transforming it into a precision delivery aircraft. Students must design a reliable release mechanism, account for how the payload changes the aircraft's centre of gravity, and develop an accurate mental model of ballistics — the physics of how a released payload travels after leaving the aircraft. The challenge is structurally identical to real aerial delivery operations used for humanitarian aid, agricultural inputs, and medical supply delivery across Ghana and across Africa.

**Learning Objectives**

* Design and build a servo-operated cargo release mechanism that triggers reliably from the transmitter
* Calculate the effect of payload mass and position on the aircraft's centre of gravity
* Explain the ballistic trajectory of a released payload using forward velocity and gravity
* Develop and apply a consistent approach procedure (altitude, speed, release timing) for accurate drops
* Record 5 drop attempts with approach data; analyse the pattern of errors to improve technique
* Achieve at least one drop within 3 m of the target

**3. Build Steps & Assembly**

**Lesson 1 – Mechanism Design & Build**

|  |  |
| --- | --- |
| **STEP 1** | **Design the Release Mechanism** |
|  | * Sketch two or three different release mechanism concepts in your logbook * Consider: How will the servo arm move to release the hook? How will the hook re-engage for the next drop? * Recommended design: a U-shaped hook mounted under the fuselage; servo arm rotates to push hook open * Key requirement: mechanism must hold the payload securely under vibration AND release cleanly on command * Have teacher approve your design sketch before building |

|  |  |
| --- | --- |
| **STEP 2** | **Build & Install the Mechanism** |
|  | * Bend the release hook from stiff wire (or use the printed plastic hook in Kit 6) * Mount the hook under the fuselage at the CG point — this minimises the effect on aircraft balance * Mount the micro servo beside the hook, oriented so its arm pushes the hook open at full travel * Connect servo to receiver channel 5 (or the designated spare channel) * Assign a two-position switch on the transmitter to channel 5 * Bench test: switch to release position; confirm hook opens fully; switch back; confirm hook closes |

|  |  |
| --- | --- |
| **STEP 3** | **Payload Preparation & CG Check** |
|  | * Fill the payload bag to 50–75 g; seal securely; add a strip of bright tape for visibility on the ground * Hang payload from the release hook; measure new CG position with payload installed * CG must remain within 30–35% chord from the leading edge — re-balance with ballast if needed * Test: fly hands-off for 3 seconds in the simulator with the shifted CG in mind; is the trim still acceptable? * Fly a short circuit WITHOUT releasing the payload to confirm the aircraft handles correctly under load |

**Lesson 2 – Ground Release Tests**

|  |  |
| --- | --- |
| **STEP 4** | **Ground Release Test** |
|  | * Suspend the aircraft at 1 m height (hold at fuselage or hang from a beam) * With payload attached, trigger the release switch 5 times * Criteria: all 5 releases must be clean (no sticking, no partial releases) * After each release, re-engage the hook and re-attach the payload; confirm it holds securely * If any release fails, adjust servo travel limits or reduce friction on the hook pivot |

**Lesson 3 – First Drop Flights**

|  |  |
| --- | --- |
| **STEP 5** | **Approach & Drop Procedure** |
|  | * Establish a consistent approach: 20 m/s airspeed, 15 m altitude, flying toward the target * Release point: fly until the target disappears under the aircraft nose, THEN release * This offset accounts for the payload's forward travel after release * Record every drop in the Drop Log: approach speed, altitude, release timing, distance to target |

|  |  |
| --- | --- |
| **STEP 6** | **Analyse & Adjust** |
|  | * After 3 drops, review the Drop Log: are you consistently long, short, or accurate? * If consistently long (too far): release later, fly lower, or slow the approach * If consistently short (not far enough): release earlier or fly slightly higher * Adjust one variable at a time — changing multiple variables makes it impossible to identify the cause of improvement |

|  |  |
| --- | --- |
| **STEP 7** | **Scored Competition Drops** |
|  | * Set up the 2 m target circle in an open area * Each pilot gets 5 scored drops; record result in the scoring table * Best score across all 5 drops is the competition score * Award: highest total score, most improved (comparing Drop 1 vs Drop 5) |

**5. Engineering Principles**

**Ballistics – Predicting Where the Payload Lands**

When the payload is released, it immediately has two components of velocity: forward (equal to the aircraft speed) and zero vertical (no initial downward velocity). Gravity then accelerates it downward at 9.8 m/s². The payload follows a parabolic path — identical to a thrown ball.

|  |
| --- |
| **Drop Distance Formula (Simplified)**  Time to fall from altitude h: t = √(2h / g) where g = 9.8 m/s²  Forward travel during fall: d = v × t where v = aircraft speed in m/s  Example: flying at 15 m altitude at 20 m/s → t = √(2×15/9.8) = 1.75 s → d = 20 × 1.75 = 35 m ahead  Therefore: release the payload approximately 35 m before the target when flying at 20 m/s at 15 m altitude.  In practice: this is refined through repeated drops and adjustment. |

**Variables Affecting Drop Accuracy**

| **Variable** | **Effect on Drop Point** | **Pilot Control** |
| --- | --- | --- |
| **Aircraft altitude** | Higher altitude = longer forward travel | Fly lower for shorter drop distance |
| **Aircraft speed** | Faster = payload travels further forward | Reduce speed before drop zone |
| **Release point** | Earlier = payload lands further ahead | Release slightly before flying over target |
| **Payload aerodynamics** | Streamlined = less drag = more forward travel | Use compact, symmetrical payload shape |
| **Wind (headwind)** | Payload drifts less forward | Release slightly later than in calm conditions |
| **Wind (tailwind)** | Payload drifts further forward | Release slightly earlier than in calm conditions |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Payload will not release** | Mechanism binding or servo not reaching full travel | Check servo arm reaches release angle; reduce friction in latch |
| **Payload releases too early** | Vibration triggering latch | Add a spring-loaded catch; ensure latch engages positively |
| **Aircraft trims badly with payload** | Payload too far from CG | Mount payload within 2 cm of the CG point; re-balance after mounting |
| **Always drops long (too far forward)** | Releasing too early; altitude too high | Release later; fly lower approach; reduce speed |
| **Always drops short** | Releasing too late; altitude too low | Release earlier; increase approach altitude slightly |
| **Servo connection fails in flight** | Connector worked loose from vibration | Lock servo connector with a zip tie; apply vibration-damping tape |

**9. Upgrade & Extension Ideas**

* Multiple Payloads – Add two hooks and two servos; practice releasing different payloads at different waypoints
* Parachute Delivery – Attach a small parachute to the payload; observe how it changes the ballistic path
* GPS-Triggered Release – Use a flight controller with waypoint capability; trigger servo release automatically at a GPS coordinate
* Camera-Guided Drop – Fit an FPV camera; release payload while viewing the target from the FPV feed
* Heavy Lift Challenge – Strengthen the airframe; increase payload to 200 g; test the limits of the Project 11 design

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* The ground release test (Step 4) is essential — do not skip it; an unreliable mechanism will fail at the worst moment in flight
* Mark the release switch position with brightly coloured tape so pilots can find it without looking down at the transmitter
* For the first drop flight, fly 50% higher than the planned altitude — this gives more time to react and the drop will be longer, teaching the students to release earlier
* After the competition, discuss real-world applications: Zipline Medical Drone (Rwanda/Ghana) delivers blood products using the same ballistic release principle

**Differentiation Strategies**

* Support – Pre-built release mechanism; larger target circle (5 m); no approach data recording; 3 drops only
* Core – Full mechanism build; CG analysis; 5 scored drops; Drop Log completed
* Extension – Drop distance formula calculations; parachute delivery; multiple payload release
