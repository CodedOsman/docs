**Student & Teacher Manual**

**1. Project Overview**

This is the engineering capstone of the Expert section. Students apply the complete engineering design process — define, research, ideate, decide, prototype, test, iterate, present — to design and build an original quadcopter from loose components. Unlike Projects 10–15 where students followed instructions, this project requires students to make and justify every design decision: frame size, motor selection, battery capacity, propeller pairing, and mission configuration. The result must hover, fly a basic mission, and be presented to a review panel.

**Learning Objectives**

* Define a mission with specific payload, flight time, range, and operational requirements
* Generate at least two design concepts with sketches; evaluate them using a weighted decision matrix
* Fabricate a custom frame from plywood, acrylic, or composite material using appropriate tools
* Select and pair motors, ESCs, propellers, and a battery using thrust and efficiency calculations
* Install and configure a flight controller, GPS module, and receiver from scratch
* Conduct structured ground tests and a tethered first flight; identify and fix issues
* Measure at least 3 performance metrics; compare to targets; implement one design iteration
* Deliver a 10-minute design review presentation to a panel, justifying every major decision

**3. Build Phases & Steps**

**Phase 1: Design (Lessons 1–3)**

|  |  |
| --- | --- |
| **STEP 1** | **Mission Definition** |
|  | * Write a one-paragraph mission brief: what will this drone do? For whom? Where? * Define 5 quantified requirements: payload capacity (g), flight time (min), range (m), maximum speed (m/s), maximum all-up weight (g) * Example: 'Survey cocoa farm crop health; carry a camera payload of 300 g; fly for 15 min; cover 500 m radius; weigh < 1.5 kg' * Teacher reviews and approves mission brief before concept sketching begins |

|  |  |
| --- | --- |
| **STEP 2** | **Concept Sketches** |
|  | * Each student produces a minimum of 2 concept sketches on A4 paper * Each sketch must show: top-view frame layout, arm length and angle, motor positions, battery location, payload mount position * Label frame material, estimated arm length (mm), and motor size for each concept * Write a 2-sentence description of each concept's key advantage |

|  |  |
| --- | --- |
| **STEP 3** | **Decision Matrix** |
|  | * Use the Decision Matrix template (Section 6) to evaluate both concepts * Assign scores for each criterion; multiply by weight; sum the weighted scores * The concept with the highest total score is the recommended design * Write a 3-sentence justification: why does the selected concept best meet the mission requirements? * Teacher approves decision matrix and selected concept before building begins |

**Phase 2: Prototype (Lessons 4–8)**

|  |  |
| --- | --- |
| **STEP 4** | **Frame Fabrication** |
|  | * Transfer the selected concept sketch to the frame material using pencil and ruler * Cut the central plate and four arms using a jigsaw, scroll saw, or craft knife (depending on material) * Sand all edges smooth; remove all burrs that could cut wires * Drill motor mount holes using a template; check alignment with the motor pattern * Assemble the frame dry (no glue/bolts yet); check arm symmetry: all diagonal measurements must be equal |

|  |  |
| --- | --- |
| **STEP 5** | **Motor Selection & Mounting** |
|  | * Use the motor KV calculator: target hovering thrust = 2× total take-off weight; select motors that provide this at 50% throttle * Mount CW motors on Front-Left and Rear-Right; CCW on Front-Right and Rear-Left * Apply thread-lock to all motor mount screws; torque firmly * Confirm each motor spins freely by hand with no rubbing or wobble |

|  |  |
| --- | --- |
| **STEP 6** | **Electronics Installation & Soldering** |
|  | * Solder ESC power leads to the PDB: positive (red) and negative (black); confirm polarity before soldering * Solder ESC motor pads to motor leads (3 wires per motor — direction confirmed later) * Mount PDB and FC on central plate using nylon standoffs for vibration isolation * Connect all ESC signal wires to FC motor outputs in correct order * Mount GPS module on a mast at least 5 cm above the FC; connect to FC GPS UART port |

|  |  |
| --- | --- |
| **STEP 7** | **Configuration & Calibration** |
|  | * Flash Betaflight or ArduPilot firmware to the flight controller * Configure motor mapping, ESC protocol, receiver type, and failsafes * Complete accelerometer, compass, and radio calibrations * Spin each motor individually in Betaflight Motor tab to confirm correct direction * Correct any wrong-direction motors by swapping two of the three ESC-to-motor wires |

**Phase 3: Testing (Lessons 9–10)**

|  |  |
| --- | --- |
| **STEP 8** | **Ground Tests** |
|  | * Complete the full pre-flight checklist from Project 10 * Confirm: all 4 motors spin correct direction; all controls respond; GPS fix acquired; battery voltage correct * Teacher connects battery for first motor-spin test; students observe from 3 m distance * Record ground test results in the performance table (Section 6) |

|  |  |
| --- | --- |
| **STEP 9** | **Tethered First Flight** |
|  | * Attach tether to the frame centre; anchor to ground * Teacher connects battery; arms the drone; gradually applies throttle * Observe: does the drone lift off level? Does it drift? Does it oscillate? * Record all issues observed; document in the design portfolio |

**Phase 4: Iteration (Lessons 11–12)**

|  |  |
| --- | --- |
| **STEP 10** | **Design Review & Iteration** |
|  | * List all issues identified during testing; rank by severity * Select the most critical issue; propose and implement a design change * Re-test after the change; confirm the issue is resolved * Document: what was changed, why, and what happened * Measure and record at least 3 performance metrics (Section 6) |

|  |  |
| --- | --- |
| **STEP 11** | **Final Flight & Presentation Preparation** |
|  | * Fly the full performance test mission: take off; hover for 60 s; fly a 20 m circuit; land * Record final performance metrics; compare to original targets from Mission Definition * Prepare a 10-minute design review presentation covering: mission brief, concepts, decision matrix, build process, test results, iteration, final performance * Each group member must present at least one section |

**5. Engineering Principles**

**Design Trade-Offs**

| **Design Factor** | **Trade-Off** | **How to Decide** |
| --- | --- | --- |
| **Frame size (larger)** | More stable; longer flight time; less agile; harder to transport | Match to mission: survey needs stability; racing needs agility |
| **Motor KV (higher)** | Faster; more responsive; shorter flight time; more battery drain | Higher KV for racing; lower KV for long-endurance or heavy-lift |
| **Battery capacity (larger)** | Longer flight time; heavier; reduces agility and payload | Calculate: target flight time × average current draw = mAh needed |
| **Frame material (carbon)** | Lighter; stronger; expensive; hard to fabricate without tools | Use if budget allows; otherwise plywood with carbon reinforcement |
| **Propeller size (larger)** | More efficient; more thrust; slower response; risk of ground strikes | Match to motor KV; larger prop needs lower KV motor |
| **Number of motors (6 vs 4)** | More thrust and redundancy; heavier; more complex; more expensive | Use hexacopter for payload > 500g or when one motor failure must not cause crash |

|  |
| --- |
| **The Engineering Design Process**  Define → clearly state the problem and constraints before generating solutions.  Ideate → generate multiple concepts without judging them; quantity before quality.  Select → use evidence (decision matrix) to choose the best concept; do not rely on gut feeling alone.  Prototype → build the simplest version that can be tested; don't optimise prematurely.  Test → measure against the original requirements; use quantified criteria.  Iterate → make one change at a time; test after each change to identify its effect.  Present → engineers must communicate their decisions as clearly as they make them. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Frame too flexible** | Material too thin or joints not reinforced | Add gussets at arm joints; use thicker plate; consider carbon strip reinforcement |
| **Motors mismatched to propellers** | Wrong KV for selected prop size | Use motor KV calculator: KV × battery voltage = unloaded RPM; check prop chart |
| **Battery too heavy for frame** | Capacity selected without weight check | Weigh all components first; battery should be < 30% of total take-off weight |
| **FC does not arm** | Compass not calibrated or incorrect motor mapping | Redo compass calibration outdoors; verify motor order in Betaflight motor tab |
| **Excessive vibration** | Motors or props unbalanced; frame resonance | Balance propellers with a prop balancer; add dampers under FC |
| **Short flight time** | Battery too small or motors drawing too much current | Measure actual current draw with a watt meter; select battery accordingly |

**9. Upgrade & Extension Ideas**

* Hexacopter – Design a 6-motor configuration; calculate thrust redundancy; fly a motor-failure simulation
* Foldable Frame – Design arms that fold for transport; calculate the additional weight of the hinge mechanism
* Long-Endurance Design – Optimise for maximum flight time; target 20+ minutes; graph battery discharge curve
* Heavy-Lift Design – Optimise for maximum payload; target 500 g; document structural reinforcement choices
* Waterproof Enclosure – Design sealed motor nacelles and ESC enclosures; test in light rain

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* The decision matrix (Step 3) is the most important intellectual step — allocate a full lesson to it; challenge students to justify every score with evidence
* Pre-cut a sample frame in plywood to show students what is achievable; this prevents overly ambitious designs that cannot be built in time
* Invite a local engineer, GCAA inspector, or drone company representative to attend the final presentation — it raises the stakes and the quality of student work
* For the first tethered flight, teacher operates the transmitter for the first 20 seconds; hands over control only when stability is confirmed

**Differentiation Strategies**

* Support – Pre-cut frame components; teacher-selected motor and battery; student focuses on assembly, configuration, and testing from Phase 2
* Core – Full design process; custom frame fabrication; all phases completed; 10-minute presentation
* Extension – Hexacopter; long-endurance optimisation; commercial pitch presentation; competition entry
