**Student & Teacher Manual**

**1. Project Overview**

Formation flight is one of aviation's most technically demanding disciplines — requiring precise coordination, reliable communication, and fast control loops. In this project, two GPS-equipped quadcopters fly in formation autonomously: a leader drone follows a pre-planned waypoint circuit while a wingman drone continuously calculates the required offset position and flies to it. This mirrors the multi-agent drone systems used for survey coverage, entertainment light shows, and military operations.

**Learning Objectives**

* Explain the leader-follower formation control architecture and the role of GPS offsets
* Install and configure inter-drone communication modules on both drones
* Programme the leader drone with a 4-waypoint autonomous circuit in Mission Planner
* Write or configure the wingman offset controller to maintain a 5 m formation offset
* Verify the communication link: leader position received by wingman within 100 ms
* Fly a complete V-formation circuit and record offset error at 30-second intervals

**3. Build Phases & Steps**

**Phase 1: Setup & Communication (Lessons 1–2)**

|  |  |
| --- | --- |
| **STEP 1** | **Individual Drone Verification** |
|  | * Fly each drone individually to confirm GPS, hover stability, and return-to-launch * Both drones must pass the Project 10 pre-flight checklist independently * Assign roles: Drone 1 = Leader (callsign L); Drone 2 = Wingman (callsign W) |

|  |  |
| --- | --- |
| **STEP 2** | **Communication Module Installation** |
|  | * Install a LoRa or WiFi module on each drone — typically connected to a UART port * Configure the module baud rate to match the flight controller telemetry port * Test on the ground: power both drones; open a terminal on each laptop * Send a test GPS position string from L to W; confirm W receives it within 100 ms |

|  |  |
| --- | --- |
| **STEP 3** | **Ground Station Setup** |
|  | * Connect both drones to Mission Planner simultaneously using the telemetry radios * In Mission Planner: switch to multi-vehicle view; confirm both HUDs are active * Verify GPS position accuracy: both drones should show within 2 m of their actual position |

**Phase 2: Programming (Lessons 3–5)**

|  |  |
| --- | --- |
| **STEP 4** | **Leader Mission Planning** |
|  | * Plan a 4-waypoint rectangular mission for the Leader drone in Mission Planner * Waypoints: 20 m × 20 m square; altitude 10 m; loiter 3 s at each waypoint * Upload mission to Leader; read back to verify * Set Leader failsafe: RTL if communication lost |

|  |  |
| --- | --- |
| **STEP 5** | **Wingman Offset Controller** |
|  | * The Wingman controller receives Leader GPS position and adds the formation offset * V-formation offset: Wingman = Leader position + (–5 m North, +5 m East) * The Wingman then flies to that calculated target GPS position in GUIDED mode * Loop runs at 2 Hz: new Leader position → calculate offset target → send GUIDED command * Test on ground: move Leader drone by hand; Wingman should update its target position |

**Phase 3: Flight Testing (Lessons 6–8)**

|  |  |
| --- | --- |
| **STEP 6** | **Individual Auto Flight Tests** |
|  | * Fly the Leader drone solo through the complete 4-waypoint mission * Fly the Wingman drone solo in GUIDED mode with a simulated offset target (stationary) * Both must complete individual tests successfully before formation flight attempt |

|  |  |
| --- | --- |
| **STEP 7** | **Formation Flight** |
|  | * Teacher connects both batteries; both drones arm simultaneously * Leader arises first to 10 m; Wingman arms and rises to 10 m * Start the Leader autonomous mission; Wingman begins tracking * Record offset error at 30-second intervals in the data table * Emergency: if either drone behaves unexpectedly, teacher aborts both immediately |

**4. Power & Safety Notes**

|  |
| --- |
| **⚠ CRITICAL – TWO DRONE SAFETY**  TWO drones in the air simultaneously doubles the risk. Both drones must be individually proven before formation flight.  Minimum separation during formation: 4 m. Never attempt a formation tighter than this with GPS-guided control.  Two designated spotters: one per drone; both call 'ABORT' if their drone behaves unexpectedly.  Teacher controls battery connections for BOTH drones.  Flight zone: minimum 50 m × 50 m; all observers at least 10 m outside the formation path.  Both drones land before any battery change or adjustment. |

**5. Engineering Principles**

**Formation Types**

| **Formation** | **Layout** | **Use Case** | **Offset (Leader = 0,0)** |
| --- | --- | --- | --- |
| **Line Astern** | Wingman directly behind leader | Simple; easy to implement | 0, –5 m |
| **V-Formation** | Leader at front; wingman each side | Survey; area coverage | –5 m left / +5 m right |
| **Echelon Right** | Wingman to right and behind | Camera covering left | –5 m, –5 m |
| **Box** | 2×2 grid | Maximum area coverage | ±5 m, ±5 m |

**Communication Data Fields**

| **Data Field** | **Update Rate** | **Format** | **Example** |
| --- | --- | --- | --- |
| **GPS Latitude** | 2 Hz | Decimal degrees | 5.560012 |
| **GPS Longitude** | 2 Hz | Decimal degrees | -0.205741 |
| **Altitude (m)** | 2 Hz | Metres above home | 12.5 |
| **Heading (°)** | 5 Hz | 0–360 magnetic | 274 |
| **Speed (m/s)** | 5 Hz | Ground speed | 8.2 |
| **Status flag** | 1 Hz | Integer (0=OK, 1=WARN) | 0 |

|  |
| --- |
| **Leader-Follower Control Loop**  Step 1: Leader broadcasts its GPS position, altitude, heading, and speed at 2 Hz.  Step 2: Wingman receives the data and applies the formation offset in the North-East frame.  Step 3: Wingman sends a GUIDED waypoint command to its own flight controller with the offset target.  Step 4: Wingman FC flies to the offset target using its own GPS.  Step 5: Loop repeats every 500 ms.  Key constraint: GPS accuracy is ±1–3 m, so the minimum achievable offset error is also ±1–3 m. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Communication lost** | Range exceeded or RF interference | Reduce inter-drone distance; switch to a less crowded frequency |
| **Wingman position offset too large** | GPS accuracy limit; latency | Reduce flight speed; increase waypoint tolerance in ArduPilot |
| **Wingman oscillates around leader** | Control loop gain too high | Reduce P gain in the offset-following controller |
| **Leader and wingman collide** | Offset calculated in wrong reference frame | Verify offset is in GPS (North-East) frame, not body frame |
| **One drone loses GPS** | GPS antenna blocked or cold start | Fly only with confirmed 3D fix; pre-warm GPS for 3 min before arming |

**9. Upgrade & Extension Ideas**

* Three-Drone Formation – Add a third drone; programme both wingmen from a single leader
* Dynamic Formation Change – Mid-flight formation change from V to Line Astern; programme the offset switch
* Obstacle Avoidance – Add ultrasonic sensors to each drone; programme collision avoidance between formation members
* Light Show Choreography – Programme a sequence of formation changes to create a coordinated light display

**10. Teacher Notes & Differentiation**

* This project requires two confident pilots — assign your most experienced students; the formation flight step is not for beginners
* The communication test (Step 2) is the critical gate — do not proceed to flight until ground comms are confirmed < 100 ms
* Support – Simulate formation with one real drone and one ground vehicle broadcasting position; student programs the offset controller without flying
* Core – Full two-drone formation; V-formation circuit; offset error data collected
* Extension – Three drones; dynamic formation change; light show choreography
