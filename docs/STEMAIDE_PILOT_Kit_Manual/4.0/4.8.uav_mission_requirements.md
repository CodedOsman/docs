**Student & Teacher Manual**

**1. Project Overview**

Before any engineering product is built, its requirements must be defined — and before any concept is selected, multiple alternatives must be generated and evaluated. This project applies the front-end engineering design process to a real Ghanaian UAV mission: students choose one of four mission briefs, quantify the requirements, generate three distinct UAV concepts, and evaluate them using a weighted decision matrix. The output is a professional Mission Requirements & Concept Report that forms the foundation of the Project 46 sizing calculations and Project 19 build.

**Learning Objectives**

* Select a real Ghanaian UAV mission brief and define at least 5 quantified performance requirements
* Generate three genuinely different UAV concept designs (e.g. quadcopter, fixed-wing, hybrid) with sketches and key features
* Complete a weighted decision matrix evaluating all three concepts against 7 criteria
* Identify the highest-scoring concept and write a 2-paragraph justification of the selection
* Produce a 1-page Mission Requirements & Concept Report suitable for presenting to a client
* Receive and respond to peer review feedback on the requirements report

**3. Session Steps**

**Lesson 1 – Mission Selection & Requirements**

|  |  |
| --- | --- |
| **STEP 1** | **Mission Brief Selection** |
|  | * Each group selects one of the four mission briefs (A–D) * Read the brief carefully: what is the problem being solved? Who is the customer? * Use the Mission Requirements Reference Table to identify the baseline requirements for the chosen mission * Add two additional requirements specific to the Ghana context (e.g. 'must withstand 30°C ambient temperature'; 'must comply with GCAA Part 33') |

|  |  |
| --- | --- |
| **STEP 2** | **Requirements Quantification** |
|  | * Convert all requirements to specific numbers * Not acceptable: 'long flight time' * Acceptable: 'minimum 30 minutes flight time at cruise speed with full payload' * Add a constraint: 'maximum all-up weight of 1.5 kg' — this forces design decisions * Add a GCAA compliance requirement: 'must include Return to Launch failsafe; must be registerable under GCAA Part 33' |

**Lesson 2 – Concept Generation**

|  |  |
| --- | --- |
| **STEP 3** | **Generate Three Concepts** |
|  | * Concept 1: Multi-rotor (quadcopter or hexacopter) — stable hover; vertical takeoff; limited endurance * Concept 2: Fixed-wing — longer endurance; higher speed; needs launch/landing infrastructure * Concept 3: Hybrid / VTOL — best of both; more complex; heavier * For each concept: produce a top-view sketch; list 3 key features; estimate payload capacity and flight time * Do not pre-select the winner — each concept must be genuinely explored |

|  |  |
| --- | --- |
| **STEP 4** | **Complete the Decision Matrix** |
|  | * Use the Decision Matrix Template (Section 6) * Score each concept against each criterion using 1–5 (1=poor; 5=excellent) * Multiply each score by the criterion weight (%); sum all weighted scores for each concept * The concept with the highest total score is the recommended design * Write a 2-paragraph justification: why does this concept best serve the mission requirements? |

**Lesson 3 – Report Writing & Peer Review**

|  |  |
| --- | --- |
| **STEP 5** | **Write the Requirements Report** |
|  | * One-page report structure: * Section 1 (150 words): Mission brief — what problem does this UAV solve, for whom, in Ghana? * Section 2 (100 words): Requirements table — all requirements with specific values * Section 3: Three concept sketches with brief descriptions * Section 4: Completed decision matrix * Section 5 (150 words): Recommendation — which concept was selected and why? |

|  |  |
| --- | --- |
| **STEP 6** | **Peer Review** |
|  | * Exchange reports with another group * Reviewing group asks: Are all requirements specific and measurable? Are three genuinely different concepts shown? Is the matrix scoring justified? * Write 2 strengths and 1 improvement on a sticky note; attach to the report * Author group reads feedback; revises the report if needed before submission |

**4. Safety Notes**

|  |
| --- |
| **Design Safety Requirements**  All UAV concepts must include as a minimum: Return to Launch (RTL) failsafe on signal loss; maximum operating altitude of 120 m above ground level (GCAA regulation); registration provision per GCAA Part 33.  Any concept intended for flight over people must include propeller guards, a geofence, and insurance provision.  These are not optional design features — they are regulatory requirements in Ghana. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Requirements too vague** | Mission not defined with numbers | Every requirement must have a specific number: 'heavy' → '300 g payload' |
| **Only one concept generated** | First idea accepted too quickly | Force three genuinely different concepts: quad, fixed-wing, and hybrid are a good starting set |
| **Matrix scores inconsistent** | Scoring done by feel rather than evidence | Score based on specific evidence: 'Concept A scores 4 for payload because it can carry 350 g which exceeds the 300 g requirement' |
| **Regulatory compliance ignored** | Students unfamiliar with GCAA UAV rules | Reference Project 43: GCAA Part 33 applies; all concepts must include RTL failsafe as a minimum |
| **All concepts score nearly equal** | Criteria not weighted meaningfully | Increase the weighting gap between the most important criterion (endurance: 20%) and least (sourcing: 10%) |

**9. Upgrade & Extension Ideas**

* Real Client Brief: approach a local farm, clinic, or government agency; present your requirements report as a real proposal; ask for their feedback
* Cost Estimate: add a cost column to the matrix; research component prices on local electronics markets in Accra; calculate total build cost for each concept
* Trade Study: perform a parametric trade study — vary payload from 100 g to 500 g in 100 g steps; graph the effect on required wing area and battery capacity
* Regulatory Risk Assessment: add a risk column — which concept has the highest regulatory risk in Ghana? Which is most likely to receive GCAA approval quickly?

**10. Teacher Notes & Differentiation**

* The key skill in this project is translating vague requirements ('it should fly a long time') into specific, measurable values ('minimum 30 minutes at 10 m/s with 300 g payload') — spend significant time on this in Lesson 1
* Encourage students to look at real Ghanaian UAV companies (e.g. Zipline operations in Ghana) when defining the mission brief
* Support – Teacher provides pre-filled requirements table; students generate 2 concepts and complete a simplified 5-criterion matrix
* Core – Full requirements quantification; 3 concepts; full 7-criterion matrix; report; peer review
* Extension – Real client brief; cost estimate; trade study; regulatory risk assessment
