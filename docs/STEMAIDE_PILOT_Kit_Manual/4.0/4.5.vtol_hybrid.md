**Student & Teacher Manual**

**1. Project Overview**

VTOL (Vertical Takeoff and Landing) hybrid aircraft represent the frontier of modern aviation engineering. Combining the convenience of a helicopter with the efficiency of a fixed-wing aircraft, VTOL platforms are used by the world's most advanced military, commercial, and delivery operators. This project challenges students to design, build, configure, and fly a VTOL hybrid — either a QuadPlane (separate lift and cruise motors), a tilt-rotor, or a tilt-wing. Every design decision must be justified, and the aircraft must successfully complete a hover, a transition, and a forward flight leg to pass.

**Learning Objectives**

* Compare the three VTOL configurations and select one with a written justification
* Construct a hybrid airframe combining fixed-wing and multirotor structural requirements
* Install and configure a VTOL-capable flight controller (ArduPilot QuadPlane or Tiltrotor firmware)
* Tune the hover PID for stable vertical flight before attempting any transition
* Execute at least 4 mode transitions: 2 hover → forward, 2 forward → hover
* Fly a complete VTOL mission: vertical takeoff, transition, forward cruise leg, transition back, vertical landing

**3. Build Phases & Steps**

**Phase 1: Design (Lessons 1–2)**

|  |  |
| --- | --- |
| **STEP 1** | **Configuration Selection & Design Sketch** |
|  | * Review the three VTOL configurations in the table below * Each student sketches their preferred configuration; groups vote on which to build * Produce a top-view and side-view sketch showing: wing, fuselage, motor positions, battery location * Teacher approves design before any building begins |

|  |  |
| --- | --- |
| **STEP 2** | **Weight Budget** |
|  | * Weigh every component before assembly: frame, motors, ESCs, FC, GPS, battery, servos * Sum the weights; the total must be below 1.5 kg for safe VTOL flight on the recommended battery * If over budget: identify the heaviest non-essential item and find a lighter alternative |

**Phase 2: Build (Lessons 3–6)**

|  |  |
| --- | --- |
| **STEP 3** | **Airframe Construction** |
|  | * Build the fixed-wing airframe from foam board (or use the Project 11 aircraft as the base) * Add VTOL motor mounts at the four arm positions: reinforce the foam with plywood backing plates * Motor mounts must withstand the full thrust of all 4 lift motors pointing vertically |

|  |  |
| --- | --- |
| **STEP 4** | **Tilt Mechanism (Tilt-Rotor or Tilt-Wing only)** |
|  | * Build the pivot arm from 3 mm aluminium strip or strong plastic * Mount servo beside the pivot; connect with a short pushrod * Test manually: servo at 0° = motors vertical; servo at 90° = motors horizontal * Both motors must reach the same angle simultaneously — adjust linkage until symmetric |

|  |  |
| --- | --- |
| **STEP 5** | **Electronics & Configuration** |
|  | * Flash ArduPilot QuadPlane or Tiltrotor firmware to the flight controller * Configure motor layout (lift motors + cruise motor) in Mission Planner * Calibrate accelerometer, compass, and radio * Configure transition parameters: transition airspeed (12–15 m/s recommended); back-transition threshold |

**Phase 3: Testing (Lessons 7–10)**

|  |  |
| --- | --- |
| **STEP 6** | **Hover Test – Tethered** |
|  | * Tether the aircraft to the ground; teacher connects battery * Arm in QLOITER mode; slowly increase throttle to hover * Target: stable hover at 1.5 m for 30 seconds without excessive oscillation * If oscillating: reduce P gain in QuadPlane hover PID; retest |

|  |  |
| --- | --- |
| **STEP 7** | **First Transition** |
|  | * Take off in hover mode to 15 m altitude * Switch to AUTO or FBWA mode; aircraft should accelerate forward and tilt motors * Observe: wing should begin generating lift as speed builds; lift motors reduce power * If transition fails: increase transition airspeed parameter; retry |

|  |  |
| --- | --- |
| **STEP 8** | **Full VTOL Mission** |
|  | * Take off vertically; climb to 20 m; transition to forward flight * Fly a 50 m straight leg; back-transition to hover; descend and land vertically * Complete the Transition Log for every mode change * Teacher and pilot debrief after each mission attempt |

**5. Engineering Principles**

**VTOL Configuration Comparison**

| **Configuration** | **Description** | **Pros** | **Cons** |
| --- | --- | --- | --- |
| **Quad-Plane** | Separate vertical lift motors + fixed-wing cruise motor | Simpler; well-supported in ArduPilot | More motors, more weight, more drag in cruise |
| **Tilt-Rotor** | Forward motors tilt from vertical to horizontal | Efficient in both modes; fewer motors | Complex mechanism; risk of tilt servo failure |
| **Tilt-Wing** | Entire wing section tilts with motors | Most aerodynamically efficient | Heaviest mechanism; most complex build |

**Flight Mode Descriptions**

| **Mode** | **Motor Orientation** | **Flight Behaviour** | **Control Source** |
| --- | --- | --- | --- |
| **Hover (VTOL)** | Vertical — lift motors up | Quadcopter-like; differential thrust for attitude | FC multirotor PID loop |
| **Transition** | Tilting / mixed | Both lift and wing lift acting; critical phase | FC transitions between control loops |
| **Forward Flight** | Horizontal — cruise motor forward | Fixed-wing; ailerons, elevator, rudder active | FC fixed-wing PID loop |

|  |
| --- |
| **The Transition Phase – Most Critical Design Challenge**  During transition, the aircraft must accelerate from hover speed (0 m/s) to minimum wing flying speed (typically 12–18 m/s).  The lift motors are gradually reducing power as wing lift builds — there is a critical phase where neither system is fully effective.  Too slow a transition: the aircraft loses altitude and may crash.  Too fast a transition: the aircraft pitches violently and may enter an unrecoverable attitude.  The transition airspeed parameter in ArduPilot controls exactly where this crossover happens. |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Unstable hover** | FC PID not tuned for VTOL hover mode | Tune hover PID in QGroundControl or Mission Planner; reduce I gain first |
| **Transition failure — nose pitches** | Transition speed too low; wing not generating enough lift | Increase forward speed before tilting; adjust transition airspeed parameter |
| **FC won't switch modes** | VTOL configuration error in firmware | Re-flash VTOL firmware; follow ArduPilot Tiltrotor/QuadPlane setup guide |
| **Tilt mechanism binds** | Servo torque insufficient or pivot not smooth | Use higher-torque servo; lubricate pivot; reduce motor mount weight |
| **Motor tilt asymmetric** | Servo linkage not equal on both sides | Adjust servo arm length; verify both motors tilt to same angle simultaneously |

**9. Upgrade & Extension Ideas**

* Autonomous VTOL Mission – Programme a complete Mission Planner auto-mission: VTOL takeoff → waypoints → VTOL landing
* Payload Integration – Add the Project 17 cargo release mechanism; deliver a payload during the forward cruise leg
* FPV Integration – Add the Project 15 FPV system; fly the transition in first-person view
* Tilt-Wing Upgrade – If successful with QuadPlane, attempt the more efficient tilt-wing configuration

**10. Teacher Notes & Differentiation**

* Recommend the QuadPlane configuration for first-time VTOL builds — it has the best ArduPilot support and most online documentation
* The weight budget (Step 2) is the most important planning step — VTOL aircraft that are overweight hover poorly and transition dangerously
* Support – Pre-built airframe with motor mounts; students focus on configuration, PID tuning, and hover test
* Core – Full build; configuration; tethered hover; 4 transitions; VTOL mission
* Extension – Autonomous VTOL mission; payload delivery; FPV transition
