**Student & Teacher Manual**

**1. Project Overview**

This is the final project in the Aviation and Aerospace Education Kit. Students bring together every skill, concept, and habit developed across Projects 1–19 to plan, execute, and present a real-world aviation mission solving a genuine Ghanaian challenge. Students choose one of four mission briefs — crop survey, medical delivery, flood mapping, or search and rescue — and complete the full engineering cycle from mission planning through flight execution to data analysis and professional presentation to a panel of judges.

**Learning Objectives**

* Select a real Ghanaian problem and define quantified mission requirements for an aviation solution
* Plan a complete flight mission: flight path, altitude, payload, data collection method, and safety assessment
* Execute the mission using the aircraft from Project 11 or Project 19, collecting meaningful data
* Analyse mission data against pre-defined success criteria and draw evidence-based conclusions
* Produce a professional 8-section design portfolio documenting the complete project
* Deliver a 10-minute presentation to a panel of judges, justifying all decisions and answering questions

**2. Mission Briefs**

| **Mission Option** | **Scenario** | **Aircraft** | **Payload** | **Success Metric** |

**3. Project Phases & Steps**

**Phase 1: Mission Planning (Lessons 1–3)**

|  |  |
| --- | --- |
| **STEP 1** | **Select Mission Brief & Define Requirements** |
|  | * Each group chooses one of the four mission briefs (Options A–D) * Write a mission statement: one paragraph explaining the real Ghanaian problem being addressed * Define 5 quantified success criteria specific to the chosen mission * Identify all safety risks and mitigation measures in a written Safety Assessment * Teacher approves mission brief and safety assessment before any build work begins |

|  |  |
| --- | --- |
| **STEP 2** | **Aircraft Selection & Modification Plan** |
|  | * Choose either the Project 11 fixed-wing or Project 19 custom quadcopter — justify the choice * Identify any modifications needed: payload mount, camera bracket, additional sensor * Plan the modifications; confirm CG will remain within limits after modifications * Document the aircraft configuration in Section 2 of the portfolio |

|  |  |
| --- | --- |
| **STEP 3** | **Mission Flight Plan** |
|  | * Draw the planned flight path on a scaled map or field diagram * Mark: takeoff, each waypoint, payload release or data collection point, landing * Assign altitude to each phase; confirm altitude is appropriate for the mission * Calculate estimated flight time; confirm battery capacity is sufficient with 20% reserve * Assign crew roles: pilot, payload operator, spotter, data recorder, safety officer |

**Phase 2: Build & Integration (Lessons 4–6)**

|  |  |
| --- | --- |
| **STEP 4** | **Build & Install Mission Systems** |
|  | * Fabricate and install any new components: payload mount, camera bracket, release mechanism * Re-check CG with all mission equipment installed; adjust if required * Test all mission-specific systems on the ground: camera records correctly; payload releases; GPS logs data * Complete a full 10-point pre-flight checklist including mission payload |

|  |  |
| --- | --- |
| **STEP 5** | **Practice Flights** |
|  | * Fly at least 2 complete practice missions before Mission Day * Practice flight 1: fly the planned route without collecting data; identify problems * Practice flight 2: fly with full mission payload and data collection systems active * Document all issues found and changes made in the Build & Integration Log (Section 4 of portfolio) |

**Phase 3: Execute Mission (Lessons 7–9)**

|  |  |
| --- | --- |
| **STEP 6** | **Mission Day Preparation** |
|  | * Complete a full pre-flight inspection; teacher signs the checklist * Brief all crew on their roles; agree on abort criteria (e.g. 'if battery drops below 3.6V per cell, abort immediately') * Set up mission area; confirm safety cordon; check wind conditions * Teacher connects battery; crew takes positions; pilot confirms readiness |

|  |  |
| --- | --- |
| **STEP 7** | **Execute the Mission** |
|  | * Pilot flies the planned route; crew members perform their assigned roles * Data recorder logs all waypoint arrivals, altitudes, times, and observations in real time * If an abort is required: pilot reduces throttle to zero; teacher takes manual control if needed * After safe landing: teacher disconnects battery; crew reviews preliminary data before leaving the field |

**Phase 4: Analysis & Presentation (Lessons 10–15)**

|  |  |
| --- | --- |
| **STEP 8** | **Data Analysis** |
|  | * Review all collected data: photos, GPS logs, waypoint records * Compare results to the success criteria defined in Step 1 * For Mission A (crop survey): annotate the map with healthy/stressed zones observed * For Mission B (medical delivery): calculate distance from target for each delivery * For Mission C/D (mapping/rescue): overlay flight path on the field map; assess coverage * Write a 1-page analysis: what the data shows; whether the mission was successful; why or why not |

|  |  |
| --- | --- |
| **STEP 9** | **Design Portfolio Completion** |
|  | * Complete all 8 sections of the portfolio using the template in Section 6 * All sections must be typed or neatly handwritten; photos must be labelled * Teacher reviews a draft portfolio before final submission * Submit the portfolio on the agreed date |

|  |  |
| --- | --- |
| **STEP 10** | **Presentation to Panel** |
|  | * Deliver a 10-minute presentation to a panel (teacher + invited guests if available) * Structure: mission brief (1 min) → aircraft and payload (2 min) → flight execution (2 min) → data and results (3 min) → what we would do differently (2 min) * Each group member must speak for at least 90 seconds * Panel asks 2–3 questions; group answers together * Demonstrate the aircraft (indoors or outdoors) after the verbal presentation if time permits |

**5. Engineering Principles**

**The Full Engineering Design Cycle – Applied**

* Define: the mission brief states the problem; the requirements state what success looks like
* Research: understanding existing solutions (Zipline, NADMO UAVs) informs better design decisions
* Ideate: the aircraft selection and modification plan explores options before committing
* Select: every major decision should be justified with evidence — not personal preference
* Prototype: practice flights test the system in the real environment before Mission Day
* Test: the mission execution generates real performance data against real success criteria
* Evaluate: the data analysis section compares actual vs planned performance honestly
* Iterate: the reflection section identifies what would be changed in a second mission cycle
* Present: the portfolio and presentation communicate the engineering process to others

**6. Portfolio Template & Rubric**

**Portfolio Structure**

| **Section** | **Content Required** | **Min Pages** | **Assessment Weight** |
| --- | --- | --- | --- |
| **1 – Mission Definition** | Selected mission brief; requirements; success criteria | 1 | 10% |
| **2 – Aircraft Design** | Aircraft type selection with justification; any modifications made | 1 | 10% |
| **3 – Mission Plan** | Flight path diagram; payload plan; safety assessment | 2 | 15% |
| **4 – Build & Integration Log** | Annotated photos of build; wiring diagram; component list | 2 | 15% |
| **5 – Flight Data** | Waypoint log; photos or video stills; sensor data if available | 2 | 20% |
| **6 – Data Analysis** | Results vs success criteria; interpretation; what the data shows | 1 | 15% |
| **7 – Reflection** | What worked; what failed; what you would change; learning summary | 1 | 10% |
| **8 – References** | Sources used; acknowledgements | 0.5 | 5% |

**Assessment Rubric**

| **Criterion** | **Weight** | **Excellent (4)** | **Good (3)** | **Developing (2)** | **Beginning (1)** |
| --- | --- | --- | --- | --- | --- |
| **Mission Completed** | 30% | All objectives met; data complete | Most objectives met | Partial completion | Mission not completed |
| **Data Quality** | 20% | Meaningful, accurate, analysed | Good data with minor gaps | Data collected but not analysed | Insufficient data |
| **Technical Quality** | 20% | Aircraft performed reliably throughout | Minor technical issues | Significant issues; partial flight | Aircraft did not fly |
| **Presentation** | 15% | Clear, confident; all questions answered | Good delivery; most questions answered | Acceptable; some gaps | Unclear or incomplete |
| **Innovation** | 15% | Novel Ghana-specific solution; creative approach | Good local relevance | Some local context | No local connection |

**7. Expected Output & Success Criteria**

| **Outcome** | **Success Criteria** |
| --- | --- |
| Mission completed | All objectives from the chosen mission brief achieved |
| Aircraft functional | Flown successfully for the full mission duration without critical failure |
| Data collected | Meaningful data (photos, GPS log, waypoint log) gathered during the mission |
| Portfolio submitted | All 8 sections complete; professionally presented |
| Presentation delivered | 10-minute panel presentation; all group members speak; questions answered |
| Rubric score | Total weighted score of 60% or higher across all five criteria |

**8. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Mission not completed on flight day** | Insufficient ground testing or planning | Allocate at least 2 full test flights before the mission day; use a pre-mission checklist |
| **Technical failure mid-mission** | Component not tested adequately | Ground test every system for 10 minutes before mission day; have spare components |
| **Data unusable (blurry photos, no GPS lock)** | Wrong sensor settings or no GPS fix | Test all sensors 30 minutes before mission; fly only with confirmed GPS fix |
| **Portfolio incomplete** | Poor time management | Use the portfolio template from Lesson 1; complete one section per lesson |
| **Presentation unclear** | Jargon without explanation; no structure | Practice out loud at least once; use the portfolio as the presentation structure |
| **No Ghana context** | Mission not connected to local need | Research real Ghanaian case: Zipline, GIS mapping, cocoa industry UAV use |

**9. Upgrade & Extension Ideas**

* Commercial Pitch – Present the mission as a business plan to a simulated investor panel; include cost analysis and a revenue model
* National Competition – Enter the project in a national STEM or aviation competition (Ghana Science & Maths Quiz; GCAA youth programme)
* Community Partnership – Partner with a real cocoa farm, clinic, or local authority; fly the mission for an actual stakeholder
* Advanced Payload – Add a multispectral camera (if available) for real plant health data; compare to visual assessment
* Published Report – Write up the project as a formal technical report in the style of a GCAA Airspace Management publication

**10. Teacher Notes & Differentiation**

**Lesson Planning Tips**

* The mission brief selection in Lesson 1 is a major decision — help groups think about what is actually achievable with their aircraft and in the time available
* Invite a real stakeholder to Mission Day if possible (a local farmer, a clinic nurse, a NADMO officer) — it transforms the exercise into a genuine service
* Require a draft portfolio review at Lesson 10 — groups that submit a draft on time consistently produce better final portfolios
* For the presentation panel, invite another teacher, a parent, or a community member — external audiences raise the quality of student communication

**Differentiation Strategies**

* Support – Mission B only (simplest payload and route); 5-section portfolio; 5-minute presentation; teacher-assisted flight planning
* Core – Any mission brief; full 8-section portfolio; 10-minute presentation; independent flight planning
* Extension – Commercial pitch; community partnership; multispectral payload; published technical report

**Assessment Suggestions**

* Portfolio: submitted on time; all 8 sections complete; evidence-based analysis in Section 6
* Presentation: each member speaks; questions answered with evidence from the portfolio
* Rubric score: 60% minimum across all 5 criteria for a pass; 80%+ for distinction
* Mission completion: teacher-observed on Mission Day; signed off before portfolio submission
