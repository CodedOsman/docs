**Student & Teacher Manual**

**1. Project Overview**

Engineering without numbers is guesswork. This project translates the Mission Requirements from Project 45 into a complete set of UAV sizing calculations: all-up weight, required hover thrust, motor power draw, battery capacity in mAh, and — for fixed-wing concepts — wing area. Every formula is derived from first principles and applied to the student's own mission. The output is a Sizing Calculation Report that gives the student the exact specifications needed to source components for a real build.

**Learning Objectives**

* Calculate the total all-up weight (AUW) from a component mass breakdown
* Apply the thrust equation to calculate required hover thrust with a 1.5× safety factor
* Calculate motor power draw at cruise speed using thrust, velocity, and motor efficiency
* Size the battery in mAh from total system power draw and required flight endurance
* Select a motor and propeller combination matched to the calculated thrust and power requirements
* Calculate wing area for a fixed-wing concept using the lift equation
* Produce a complete Sizing Calculation Report with all formulas, working, and results tabulated

**3. Session Steps**

**Lesson 1 – Mass Budget & Thrust**

|  |  |
| --- | --- |
| **STEP 1** | **Component Mass Budget** |
|  | * List every component of the chosen UAV concept; estimate the mass of each in grams * Categories: Airframe/frame; Motors × N; ESCs × N; Flight controller; Receiver; Battery; Payload; Wiring/connectors * Sum all component masses to get Total All-Up Weight (AUW) in kg * If AUW exceeds the maximum from Project 45, identify the heaviest non-essential item and find a lighter alternative * Rule of thumb: battery should be 20–30% of AUW for most multirotor applications |

|  |  |
| --- | --- |
| **STEP 2** | **Hover Thrust Calculation** |
|  | * Total thrust required for hover = AUW × g = AUW × 9.81 N * Apply safety factor: T\_required = AUW × 9.81 × 1.5 (50% thrust margin for manoeuvre and wind) * Number of motors: typically 4 for quad; 6 for hex; 8 for octo * Thrust per motor: T\_motor = T\_required ÷ N\_motors * Record in the Sizing Calculation Table (Section 5) |

**Lesson 2 – Power, Battery & Motor Selection**

|  |  |
| --- | --- |
| **STEP 3** | **Power Calculation** |
|  | * Motor power at cruise (50% throttle): P\_motor = T\_motor × v\_exit / η\_motor * v\_exit (propeller exit velocity) is approximately 10–15 m/s for typical UAV propellers * η\_motor = 0.80 (use this value unless motor datasheet gives a better figure) * Total electrical power: P\_total = (P\_motor × N\_motors) + P\_avionics * P\_avionics ≈ 5–10 W (FC, receiver, GPS, LEDs) |

|  |  |
| --- | --- |
| **STEP 4** | **Battery Sizing** |
|  | * Energy required: E = P\_total × t\_flight (in hours) * Battery capacity in Wh: E\_Wh = E * Convert to mAh: C\_mAh = (E\_Wh / V\_battery) × 1000 * Example: P\_total = 150 W; t\_flight = 0.5 h; V\_battery = 11.1V → C = (150 × 0.5 / 11.1) × 1000 = 6757 mAh → choose 7000 mAh LiPo * Add 20% reserve: never discharge below 80% capacity → final C\_selected = C\_mAh / 0.8 |

|  |  |
| --- | --- |
| **STEP 5** | **Motor & Propeller Selection** |
|  | * Motor KV selection: lower KV for large props (< 900 KV for 8-inch+); higher KV for small props * KV × battery voltage = unloaded RPM (e.g. 900 KV × 11.1V = 9990 RPM at no load) * Verify on motor datasheet: at the calculated thrust, is the current within ESC and motor ratings? * ESC must be rated at least 30% above peak motor current * Record final selections: motor model/KV, propeller diameter/pitch, ESC ampere rating |

**Lesson 3 – Wing Area (Fixed-Wing) & Report**

|  |  |
| --- | --- |
| **STEP 6** | **Wing Area Calculation (Fixed-Wing Concepts Only)** |
|  | * Lift equation: L = 0.5 × ρ × V² × S × CL * For level flight: L = AUW × g * Rearranging: S = (AUW × g) / (0.5 × ρ × V² × CL) * Use: ρ = 1.2 kg/m³ (air density at sea level, 30°C); CL = 1.2 (typical trainer airfoil at cruise) * Example: AUW = 1 kg; V = 12 m/s → S = (1 × 9.81) / (0.5 × 1.2 × 144 × 1.2) = 0.095 m² = 950 cm² * From wing area and aspect ratio (AR): Span = √(S × AR); Chord = Span / AR |

|  |  |
| --- | --- |
| **STEP 7** | **Produce the Sizing Report** |
|  | * Complete the Sizing Calculation Table with all 8 parameters * Show all working: write each formula, substitute values, show the result with units * Compare to Project 45 requirements: does the calculated design meet all mission requirements? * If not: identify which parameter needs adjustment and recalculate * Final page: Bill of Materials (BOM) — list every component with estimated mass, quantity, and approximate Ghana market price |

**4. Safety Notes**

|  |
| --- |
| **Calculation Integrity**  Always include units in every calculation — a number without units is meaningless.  Double-check: thrust per motor must be at least 2× the motor's share of AUW weight at hover throttle.  If battery capacity exceeds 10,000 mAh: double-check all inputs — this is unusually large for a 1 kg UAV.  Battery safety: LiPo batteries sized at > 5,000 mAh require careful handling — reference Project 10 LiPo safety protocols. |

**5. Sizing Calculation Table**

| **Parameter** | **Formula** | **Example (Mission A: Cocoa Survey)** | **Your Calculation** |
| --- | --- | --- | --- |
| **Total All-Up Weight (AUW)** | AUW = Structural + Battery + Payload | 0.4 + 0.3 + 0.3 = 1.0 kg |  |
| **Hover Thrust Required** | T = AUW × g × 1.5 (safety margin) | 1.0 × 9.8 × 1.5 = 14.7 N |  |
| **Thrust per Motor** | T\_motor = T\_total / N\_motors | 14.7 / 4 = 3.7 N per motor |  |
| **Motor Power (cruise)** | P = T × v\_cruise / η\_motor | 3.7 × 10 / 0.8 = 46 W per motor |  |
| **Total Power Draw** | P\_total = P\_motor × N + P\_avionics | 46 × 4 + 5 = 189 W |  |
| **Battery Capacity** | C = P\_total × t\_flight / V\_battery | 189 × 0.5h / 11.1V = 8.5 Ah = 8500 mAh |  |
| **Wing Area (if fixed-wing)** | S = AUW / (0.5 × ρ × V² × CL) | 1.0 / (0.5 × 1.2 × 100 × 1.2) = 0.014 m² |  |
| **Propeller Selection** | P = T × v\_tip / η\_prop → choose size | 3.7 N at 0.8 eff → 8-inch prop at 5000 RPM |  |

**7. Common Errors & Fixes**

| **Error** | **Likely Cause** | **Fix** |
| --- | --- | --- |
| **Battery capacity seems impossibly large** | Motor efficiency too low or flight time too long | Check η\_motor = 0.75–0.85 for good motors; verify flight time is realistic |
| **Thrust per motor seems very high** | AUW overestimated | Weigh actual components; a typical 1 kg quadcopter needs only 3–4 N per motor |
| **Wing area result is tiny** | Wrong units — m vs cm | Confirm all inputs in SI units: AUW in kg, velocity in m/s, area in m² |
| **Propeller selection unclear** | KV × voltage = RPM relationship not applied | Use: RPM = KV × battery voltage (e.g. 900 KV × 11.1V = 9990 RPM); select prop to match |
| **Different groups get very different answers** | Different assumptions | Standardise: all groups use ρ = 1.2 kg/m³, η\_motor = 0.8, CL = 1.2, safety factor = 1.5 |

**9. Upgrade & Extension Ideas**

* eCalc Cross-Check: enter your calculated values into ecalc.ch (free online UAV calculator) and compare results to your manual calculations
* Parametric Study: vary the flight time requirement from 15 to 60 minutes in 15-minute steps; graph battery capacity vs. flight time — this is a classic design trade-off
* Endurance vs. Payload Trade: keep AUW constant; graph how payload capacity changes as battery mass increases (endurance increases but payload decreases)
* Full Bill of Materials: research actual prices for every component at Accra electronics markets (Circle area) or Jumia Ghana; calculate total project cost

**10. Teacher Notes & Differentiation**

* Work through one complete example calculation on the board before students start — the unit conversion from W to mAh is consistently the most confusing step
* The eCalc cross-check (extension) is very valuable for catching errors — even core students benefit from comparing their manual result to the online calculator
* Support – Provide a partially completed table with formulas pre-printed; students substitute their own mission values and solve for the missing quantities
* Core – Full 8-parameter calculation; all working shown; requirements verification; Bill of Materials
* Extension – eCalc cross-check; parametric trade study; endurance vs. payload graph; full costed BOM
