## Overview

Build a simple water level indicator with three LEDs.

This project demonstrates reading an analog sensor and comparing it to thresholds.

The final result is one LED that shows low, medium, or high water level.

## Full Project Code

```python
from machine import Pin, ADC
import time

red = Pin(0, Pin.OUT)
yellow = Pin(1, Pin.OUT)
green = Pin(2, Pin.OUT)
sensor = ADC(Pin(26))

print('Water level indicator ready')

while True:
    raw = sensor.read_u16()
    level = int((raw / 65535) * 100)
    print('Level:', level, '%')

    if level < 30:
        red.on()
        yellow.off()
        green.off()
    elif level < 70:
        red.off()
        yellow.on()
        green.off()
    else:
        red.off()
        yellow.off()
        green.on()

    time.sleep(0.5)
```

## Troubleshooting

| Problem                  | Possible Cause                         | Solution                                     |
| ------------------------ | -------------------------------------- | -------------------------------------------- |
| LEDs never change        | Sensor AOUT not connected to GPIO 26   | Reconnect the analog output to GPIO 26       |
| Readings look stuck      | Sensor dry, dirty, or not powered      | Check VCC, GND, and clean the probe          |
| Wrong LED threshold behavior | Different sensor calibration        | Adjust the threshold values in the code      |

- ## Beginner Note
- The water sensor gives an analog value, not just ON or OFF. As more of the sensor touches water, the reading usually changes. For classroom projects, test the sensor with dry, slightly wet, and wet conditions so students can decide which LED should represent low, medium, and high water level.
- Keep the Raspberry Pi Pico 2W, breadboard, jumper wires, and USB cable away from water. Only the sensing part of the water sensor should touch water. Also, avoid leaving resistive water sensors powered in water for long periods, because continuous current through wet exposed traces can corrode the sensor faster.
