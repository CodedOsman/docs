## Overview

Build a button logger that shows recent button presses on a web page.

This project demonstrates edge detection, event history, and simple time tracking since boot.

The final result should show the last 10 button press times on a local web page.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
import network
import socket
import time
from machine import Pin

SSID = 'YOUR_WIFI_NAME'
PASSWORD = 'YOUR_WIFI_PASSWORD'

button = Pin(15, Pin.IN, Pin.PULL_UP)
presses = []
MAX_PRESSES = 10
last_button_state = 1
start_ms = time.ticks_ms()

wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(SSID, PASSWORD)

print('Connecting to Wi-Fi...')
for _ in range(15):
    if wlan.isconnected():
        break
    time.sleep(1)

if not wlan.isconnected():
    raise RuntimeError('Wi-Fi connection failed')

ip_address = wlan.ifconfig()[0]
print('Connected. Open http://{} in your browser'.format(ip_address))


def elapsed_timestamp():
    total_seconds = time.ticks_diff(time.ticks_ms(), start_ms) // 1000
    minutes = total_seconds // 60
    seconds = total_seconds % 60
    return '{:02d}:{:02d} since boot'.format(minutes, seconds)


def web_page(log_entries):
    rows = ''
    for index, text in enumerate(reversed(log_entries), 1):
        rows += '<tr><td>{}</td><td>{}</td></tr>'.format(index, text)

    if not rows:
        rows = "<tr><td colspan='2'>No button presses recorded yet</td></tr>"

    return '''<!DOCTYPE html>
<html>
<head>
    <meta name='viewport' content='width=device-width, initial-scale=1'>
    <meta http-equiv='refresh' content='2'>
    <title>Wi-Fi Button Logger</title>
</head>
<body style='font-family:Arial;text-align:center;padding:30px'>
    <h1>Wi-Fi Button Logger</h1>
    <p>Press the physical button to create a log entry.</p>
    <table border='1' cellpadding='8' cellspacing='0' style='margin:0 auto'>
        <tr><th>#</th><th>Time</th></tr>
        LOG_ROWS
    </table>
    <p>Showing the last MAX_TEXT entries</p>
    <a href='/clear'><button style='margin-top:20px;padding:10px 20px'>Clear Log</button></a>
</body>
</html>'''.replace('LOG_ROWS', rows).replace('MAX_TEXT', str(MAX_PRESSES))


address = socket.getaddrinfo('0.0.0.0', 80)[0][-1]
server = socket.socket()
server.bind(address)
server.listen(1)
server.settimeout(0.2)

while True:
    current_state = button.value()

    if current_state == 0 and last_button_state == 1:
        stamp = elapsed_timestamp()
        presses.append(stamp)

        if len(presses) > MAX_PRESSES:
            presses.pop(0)

        print('Button pressed at', stamp)
        time.sleep(0.15)

    last_button_state = current_state

    try:
        client, client_address = server.accept()
    except OSError:
        continue

    print('Client connected from', client_address)
    request = client.recv(1024).decode()

    if 'GET /clear' in request:
        presses = []
        print('Log cleared')

    response = web_page(presses)
    client.send('HTTP/1.1 200 OK\r\nContent-Type: text/html\r\nConnection: close\r\n\r\n'.encode())
    client.sendall(response.encode())
    client.close()
```

## How the Code Works

| Code Section        | What It Does                                               | Why It Matters                                                           |
| ------------------- | ---------------------------------------------------------- | ------------------------------------------------------------------------ |
| Pin.PULL_UP         | Keeps the button input high when the button is not pressed | No external pull-up resistor is needed                                   |
| Edge detection      | Logs only when the button changes from released to pressed | This prevents many log entries from one long press                       |
| elapsed_timestamp() | Creates a simple time label since the Pico started         | This avoids pretending the Pico knows real clock time when no RTC is set |
| presses list        | Stores only the most recent log entries                    | The web page stays easy to read                                          |

## Troubleshooting

| Problem                            | Possible Cause                                   | Solution                                                            |
| ---------------------------------- | ------------------------------------------------ | ------------------------------------------------------------------- |
| No log entries appear              | Button wiring is wrong or page is not refreshing | Check GPIO 15, GND, and the auto-refresh behavior                   |
| Many entries appear from one press | Button bounce is causing extra edges             | Keep the short debounce delay and press more slowly while testing   |
| Timestamps look strange            | They are relative to boot, not real clock time   | This is expected unless you add a real-time clock or NTP sync later |
