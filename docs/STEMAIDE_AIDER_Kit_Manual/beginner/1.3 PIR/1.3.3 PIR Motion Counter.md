## Overview

Build a motion event counter using a PIR sensor and an OLED display.

This project demonstrates digital input counting, I2C display communication, and data persistence.

The final result counts and displays each motion event detected by the PIR sensor.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import Pin, I2C
import sh1106
import time

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=400000)
display = sh1106.SH1106_I2C(128, 64, i2c)
pir = Pin(1, Pin.IN)

motion_count = 0
last_motion = 0

print('Motion counter ready')
print('Wait 10 seconds for PIR warm-up')
time.sleep(10)

while True:
    motion = pir.value()

    if motion == 1 and last_motion == 0:
        motion_count += 1
        print('Motion detected! Count:', motion_count)

    display.fill(0)
    display.text('Motion Counter', 10, 8, 1)
    display.text('Count: {}'.format(motion_count), 18, 30, 1)
    display.text('MOTION!' if motion else 'Waiting...', 18, 50, 1)
    display.show()

    last_motion = motion
    time.sleep(0.2)
```

## How the Code Works

| Code Section   | What It Does                                    | Why It Matters                           |
| -------------- | ----------------------------------------------- | ---------------------------------------- |
| PIR input      | Configures GPIO 1 as input for motion detection | Reads motion events from the sensor      |
| motion_count   | Stores the total count of motion events         | Persists across loop iterations          |
| Edge detection | Detects rising edge from the PIR sensor         | Counts only one event per motion trigger |
| OLED update    | Shows the current count on the display          | Provides visual feedback of the count    |

After waiting for the PIR warm-up time, each motion event in front of the sensor increments the counter. The OLED display shows the running total, and the Shell prints each detection.

## Troubleshooting

| Problem                       | Possible Cause                  | Solution                                                  |
| ----------------------------- | ------------------------------- | --------------------------------------------------------- |
| Count does not change         | PIR not warmed up or wrong pins | Wait 10 seconds and check GPIO 1 wiring                   |
| OLED shows nothing            | Wrong I2C pins or no library    | Confirm SDA/GPIO 8 and SCL/GPIO 9, install sh1106 library |
| Multiple counts per motion    | No debounce logic               | Add delay after counting to avoid multiple triggers       |
| OLED shows strange characters | Wrong I2C address               | Run I2C scanner to find correct address                   |
