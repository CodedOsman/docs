# Overview

Build a web-controlled LED using the Pico 2 W's Wi-Fi feature.

This project demonstrates how a tiny web server can control hardware from a phone or computer on the same network.

The final result should open a web page where **ON** and **OFF** buttons control the LED remotely.

# Full Project Code

```python
import network
import socket
import time
from machine import Pin

SSID = 'YOUR_WIFI_NAME'
PASSWORD = 'YOUR_WIFI_PASSWORD'

led = Pin(0, Pin.OUT)

wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(SSID, PASSWORD)

print('Connecting to Wi-Fi...')

for _ in range(15):
    if wlan.isconnected():
        break

    time.sleep(1)

if not wlan.isconnected():
    raise RuntimeError('Wi-Fi connection failed')

ip_address = wlan.ifconfig()[0]

print('Connected. Open http://{} in your browser'.format(ip_address))


def web_page():
    state = 'ON' if led.value() else 'OFF'

    html = """<!DOCTYPE html>
<html>
<head>
<meta name='viewport' content='width=device-width, initial-scale=1'>
<title>Wi-Fi LED Control</title>
<style>
body{font-family:Arial;text-align:center;padding:40px}
.btn{padding:16px 28px;font-size:20px;margin:10px}
</style>
</head>
<body>
<h1>Wi-Fi LED Control</h1>
<p>LED state: STATE_TEXT</p>
<a href='/on'><button class='btn'>ON</button></a>
<a href='/off'><button class='btn'>OFF</button></a>
</body>
</html>"""

    return html.replace('STATE_TEXT', state)


address = socket.getaddrinfo('0.0.0.0', 80)[0][-1]

server = socket.socket()
server.bind(address)
server.listen(1)

while True:

    client, client_address = server.accept()

    print('Client connected from', client_address)

    request = client.recv(1024).decode()

    if 'GET /on' in request:
        led.on()

    elif 'GET /off' in request:
        led.off()

    response = web_page()

    client.send(
        'HTTP/1.1 200 OK\r\n'
        'Content-Type: text/html\r\n'
        'Connection: close\r\n\r\n'
    )

    client.sendall(response)

    client.close()
```

# How the Code Works

| Code Section     | What It Does                                         | Why It Matters                                         |
| ---------------- | ---------------------------------------------------- | ------------------------------------------------------ |
| Wi-Fi setup      | Connects the Pico 2 W to your local wireless network | The web page cannot work until the board is online     |
| IP address print | Shows the address to open in your browser            | Students need this to reach the Pico web page          |
| `web_page()`     | Builds the HTML page with the current LED state      | The browser needs a page to display buttons and status |
| Request check    | Looks for `/on` or `/off` in the browser request     | This is how the LED is controlled from the web page    |

# Troubleshooting

| Problem                                | Possible Cause                            | Solution                                                         |
| -------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------- |
| No Wi-Fi connection                    | Wrong SSID or password                    | Recheck the Wi-Fi name and password carefully                    |
| IP address does not appear             | Board is not connected to Wi-Fi           | Run the Wi-Fi connection test first                              |
| Web page loads but LED does not change | Wrong GPIO pin or LED wiring              | Check that the LED is connected to GPIO 0 through the resistor   |
| Browser cannot open the page           | Phone/computer is not on the same network | Connect the browser device to the same Wi-Fi network as the Pico |
