# Overview

Build a temperature alert monitor with a BME280 sensor, an LED warning, and a browser page where the alert limits can be changed.

This project demonstrates threshold-based monitoring and simple browser input handling.

The final result should light the LED and show an alert on the web page when the temperature goes above the high limit or below the low limit.

# Full Project Code

Upload and run this code after the individual tests work correctly.

```python
import network
import socket
import time
from machine import I2C, Pin
import BME280

SSID = 'YOUR_WIFI_NAME'
PASSWORD = 'YOUR_WIFI_PASSWORD'

led = Pin(0, Pin.OUT)
high_temp = 30.0
low_temp = 18.0

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=400000)
try:
    bme = BME280.BME280(i2c=i2c, address=0x76)
except OSError:
    bme = BME280.BME280(i2c=i2c, address=0x77)


def current_temp_c():
    temp_text = str(bme.temperature).replace('C', '')
    return float(temp_text)


def parse_param(request, name, default):
    if name + '=' not in request:
        return default
    start = request.find(name + '=') + len(name) + 1
    end = request.find(' ', start)
    value_text = request[start:end].split('&')[0]
    try:
        return float(value_text)
    except ValueError:
        return default


def web_page(temp_value, high_value, low_value, alert_active):
    status = 'ALERT!' if alert_active else 'Normal'
    return '''<!DOCTYPE html>
<html>
<head>
    <meta name='viewport' content='width=device-width, initial-scale=1'>
    <meta http-equiv='refresh' content='3'>
    <title>Temperature Alert</title>
</head>
<body style='font-family:Arial;text-align:center;padding:30px'>
    <h1>Wi-Fi Temperature Alert</h1>
    <h2>{:.1f} C</h2>
    <p>Status: {}</p>
    <form>
        High: <input type='number' name='high' value='{}' step='0.5'>
        Low: <input type='number' name='low' value='{}' step='0.5'>
        <button type='submit'>Update</button>
    </form>
</body>
</html>'''.format(temp_value, status, high_value, low_value)


wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(SSID, PASSWORD)

print('Connecting to Wi-Fi...')
for _ in range(15):
    if wlan.isconnected():
        break
    time.sleep(1)

if not wlan.isconnected():
    raise RuntimeError('Wi-Fi connection failed')

ip_address = wlan.ifconfig()[0]
print('Connected. Open http://{} in your browser'.format(ip_address))

address = socket.getaddrinfo('0.0.0.0', 80)[0][-1]
server = socket.socket()
server.bind(address)
server.listen(1)

while True:
    client, client_address = server.accept()
    request = client.recv(1024).decode()

    high_temp = parse_param(request, 'high', high_temp)
    low_temp = parse_param(request, 'low', low_temp)

    temp_value = current_temp_c()
    alert = temp_value > high_temp or temp_value < low_temp
    led.value(1 if alert else 0)

    response = web_page(temp_value, high_temp, low_temp, alert)
    client.send('HTTP/1.1 200 OK\r\nContent-Type: text/html\r\nConnection: close\r\n\r\n'.encode())
    client.sendall(response.encode())
    client.close()
```

# How the Code Works

| Code Section     | What It Does                                            | Why It Matters                                                      |
| ---------------- | ------------------------------------------------------- | ------------------------------------------------------------------- |
| current_temp_c() | Reads the BME280 temperature and converts it to a float | The code needs a number to compare with thresholds                  |
| parse_param()    | Reads the high and low values from the browser request  | This lets the user change the limits without editing the code       |
| Threshold check  | Decides whether temperature is outside the safe range   | This is the alert logic for the project                             |
| LED alert        | Turns the LED on when the temperature is outside limits | The project gives a quick local warning without opening the browser |

# Troubleshooting

| Problem                        | Possible Cause                                                  | Solution                                                                   |
| ------------------------------ | --------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Temperature never changes      | Sensor is not responding or the text is not converted correctly | Run the BME280 temperature test first and recheck `current_temp_c()`       |
| Threshold update does not work | The browser values are not being parsed properly                | Watch the request handling and try simple values like `25` and `20`        |
| LED never lights               | Alert condition is not reached or LED wiring is wrong           | Use easy test values like a very low high limit and recheck the LED wiring |
