# Full Project Code

```python
import network
import socket
import time
from machine import Pin

SSID = 'YOUR_WIFI_NAME'
PASSWORD = 'YOUR_WIFI_PASSWORD'

relay = Pin(0, Pin.OUT)
relay.value(1)

wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(SSID, PASSWORD)

print('Connecting to Wi-Fi...')

for _ in range(15):
    if wlan.isconnected():
        break
    time.sleep(1)

if not wlan.isconnected():
    raise RuntimeError('Wi-Fi connection failed')

ip_address = wlan.ifconfig()[0]
print('Connected. Open http://{} in your browser'.format(ip_address))


def web_page():
    state = 'ON' if relay.value() == 0 else 'OFF'

    html = """<!DOCTYPE html>
<html>
<head>
<meta name='viewport' content='width=device-width, initial-scale=1'>
<title>Wi-Fi Relay</title>
</head>
<body style='font-family:Arial;text-align:center;padding:40px'>
<h1>Wi-Fi Relay Switch</h1>
<p>Relay state: STATE_TEXT</p>
<a href='/on'><button style='padding:16px 28px;font-size:20px'>ON</button></a>
<a href='/off'><button style='padding:16px 28px;font-size:20px'>OFF</button></a>
</body>
</html>"""

    return html.replace('STATE_TEXT', state)


address = socket.getaddrinfo('0.0.0.0', 80)[0][-1]

server = socket.socket()
server.bind(address)
server.listen(1)

while True:

    client, client_address = server.accept()
    print('Client connected from', client_address)

    request = client.recv(1024).decode()

    if 'GET /on' in request:
        relay.value(0)

    elif 'GET /off' in request:
        relay.value(1)

    response = web_page()

    client.send(
        'HTTP/1.1 200 OK\r\n'
        'Content-Type: text/html\r\n'
        'Connection: close\r\n\r\n'
    )

    client.sendall(response)
    client.close()
```

# How the Code Works

| Code Section              | What It Does                                                  | Why It Matters                                   |
| ------------------------- | ------------------------------------------------------------- | ------------------------------------------------ |
| `relay.value(1)`          | Starts the relay in the OFF state for most active-low modules | Prevents unexpected switching at startup         |
| Wi-Fi setup               | Connects the board to the network                             | The browser page needs the board to be online    |
| `web_page()`              | Creates the HTML page and shows relay status                  | Lets the user see whether the relay is ON or OFF |
| `/on` and `/off` requests | Changes the relay state from browser commands                 | Provides remote relay control                    |

# Troubleshooting

| Problem                               | Possible Cause                             | Solution                                            |
| ------------------------------------- | ------------------------------------------ | --------------------------------------------------- |
| Relay does not click                  | Wrong power wiring or relay type           | Check VCC, GND, and whether the relay is active-low |
| Web page loads but relay does nothing | Wrong GPIO pin or relay input wiring       | Check that the relay IN pin is connected to GPIO 0  |
| Wi-Fi connection fails                | Wrong SSID/password or unsupported network | Use a 2.4 GHz network and recheck the credentials   |
