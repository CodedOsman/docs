# Safety Alert Buzzer

# Overview

Build a simple safety alert system using a flame sensor, buzzer, LED, OLED, and browser status page.

This project demonstrates latching an alert so it stays active until you reset it.

The final result should sound and display an alert when the flame sensor is triggered, and keep that warning latched until the alarm is reset after the hazard is gone.

# Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap.
Keep the USB port facing outward so you can easily connect it to your computer.

![Circuit 2](../../assets/beginner/SAFETY ALERT BUZZER/circuit_2.png)

### Step 2: Place the Flame sensor, Buzzer, LED, and OLED

Place the Flame sensor module on the breadboard or beside it where it can sense safely.

Place the active buzzer on the breadboard and identify its positive (+) and negative (-) pins.

Place the LED with its two legs in different breadboard rows.

Place the SH1106 OLED display module on the breadboard.

Identify VCC, GND, and DOUT on the sensor before wiring.

![Circuit 3](../../assets/beginner/SAFETY ALERT BUZZER/circuit_3.png)

### Step 3: Connect the Flame sensor

Connect Flame sensor VCC to 3.3V.

Connect Flame sensor GND to GND.

Connect Flame sensor DOUT to GPIO 14.

![Circuit 4](../../assets/beginner/SAFETY ALERT BUZZER/circuit_4.png)

### Step 4: Connect the Buzzer

Connect the buzzer positive (+) pin to GPIO 0.

Connect the buzzer negative (-) pin to GND.

![Circuit 5](../../assets/beginner/SAFETY ALERT BUZZER/circuit_5.png)

### Step 5: Connect the LED

Connect the LED long leg to one end of a 220Ω resistor.

Connect the other end of the resistor to GPIO 2.

Connect the LED short leg to GND.

![Circuit 6](../../assets/beginner/SAFETY ALERT BUZZER/circuit_6.png)

### Step 6: Connect OLED Power and I2C

Connect OLED VCC to 3.3V.

Connect OLED GND to GND.

Connect OLED SDA to GPIO 8.

Connect OLED SCL to GPIO 9.

![Circuit 7](../../assets/beginner/SAFETY ALERT BUZZER/circuit_7.png)

# Full Project Code

Upload and run this code after the individual tests work correctly.

| import network
import socket
import time
from machine import I2C, Pin
import sh1106

SSID = 'YOUR_WIFI_NAME'
PASSWORD = 'YOUR_WIFI_PASSWORD'

flame = Pin(14, Pin.IN)
buzzer = Pin(0, Pin.OUT)
led = Pin(2, Pin.OUT)
alert_latched = False

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=400000)
oled = sh1106.SH1106_I2C(128, 64, i2c)


def web_page(detected, latched):
    status = 'FIRE DETECTED!' if detected or latched else 'SAFE'
    return '''<!DOCTYPE html>
<html>
<head>
    <meta name='viewport' content='width=device-width, initial-scale=1'>
    <meta http-equiv='refresh' content='2'>
    <title>Safety Alert</title>
</head>
<body style='font-family:Arial;text-align:center;padding:30px'>
    <h1>Safety Alert Buzzer</h1>
    <h2>{}</h2>
    <p>Alert latch: {}</p>
    <p><a href='/reset'><button>RESET ALARM</button></a></p>
    <p>Educational demo only. Do not use for real safety protection.</p>
</body>
</html>'''.format(status, 'ON' if latched else 'OFF')


wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(SSID, PASSWORD)

print('Connecting to Wi-Fi...')
for _ in range(15):
    if wlan.isconnected():
        break
    time.sleep(1)

if not wlan.isconnected():
    raise RuntimeError('Wi-Fi connection failed')

ip_address = wlan.ifconfig()[0]
print('Connected. Open http://{} in your browser'.format(ip_address))

address = socket.getaddrinfo('0.0.0.0', 80)[0][-1]
server = socket.socket()
server.bind(address)
server.listen(1)
server.settimeout(0.2)

while True:
    fire_detected = flame.value() == 1
    if fire_detected:
        alert_latched = True

    buzzer.value(1 if alert_latched else 0)
    led.value(1 if alert_latched else 0)

    oled.fill(0)
    oled.text('Safety Alert', 12, 8, 1)
    oled.text('FIRE!' if alert_latched else 'SAFE', 34, 30, 1)
    oled.text('RESET VIA WEB', 8, 52, 1)
    oled.show()

    try:
        client, client_address = server.accept()
    except OSError:
        continue

    request = client.recv(1024).decode()
    if 'GET /reset' in request and not fire_detected:
        alert_latched = False
        print('Alarm reset')

    response = web_page(fire_detected, alert_latched)
    client.send('HTTP/1.1 200 OK\r\nContent-Type: text/html\r\nConnection: close\r\n\r\n'.encode())
    client.sendall(response.encode())
    client.close() |

# How the Code Works

| Code Section | What It Does | Why It Matters |
| --- | --- | --- |
| Flame sensor input | Reads whether the sensor currently detects a flame source | This is the main safety trigger in the project |
| alert_latched | Keeps the alarm active after detection | This helps make sure the alert is not missed |
| RESET via web | Allows the alert to be cleared after the hazard is gone | The project needs a clear way to leave alarm mode |
| OLED and web status | Show the same current safety condition in two places | Students can monitor locally and remotely |

# Expected Result

During supervised testing, if the flame sensor detects a flame source, the buzzer and LED should turn on and stay latched. The OLED and web page should show the alert. After the flame source is removed, using the RESET button on the web page should clear the latched alarm.

# Troubleshooting

| Problem | Possible Cause | Solution |
| --- | --- | --- |
| False triggers happen often | Sensor is too sensitive or sees other infrared sources | Adjust the sensor if possible and keep bright heat sources away |
| No detection | Flame is too far away or sensor is not aimed correctly | Use a safe supervised test distance and point the sensor toward the flame source |
| RESET does not clear | Flame is still present so the latch re-triggers immediately | Remove the flame source fully before resetting |
