## Overview

Build a Bluetooth environmental monitor that sends temperature and humidity data from a BME280 sensor to a phone app.

This project demonstrates BLE monitoring with a real I2C sensor.

The final result should let a phone request the latest temperature and humidity values from the Pico.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import I2C, Pin
import bluetooth
import time
from ble_uart import BLEUART
import BME280

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=400000)

try:
    bme = BME280.BME280(i2c=i2c, address=0x76)
except OSError:
    bme = BME280.BME280(i2c=i2c, address=0x77)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Weather')


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    if command in ('read', 'status', 'env'):
        uart.write(('Temperature: {}\n'.format(str(bme.temperature))).encode())
        uart.write(('Humidity: {}\n'.format(str(bme.humidity))).encode())
    elif command == 'help':
        uart.write(b'Commands: read, status, env, help\n')
    else:
        uart.write(b'Unknown command. Send help.\n')


uart.on_rx(on_rx)

print('Bluetooth environmental monitor ready')
print('Connect with a BLE app and send read, status, env, or help')

while True:
    time.sleep(0.1)
```

## How the Code Works

| Code Section     | What It Does                                               | Why It Matters                                                 |
| ---------------- | ---------------------------------------------------------- | -------------------------------------------------------------- |
| BME280 setup     | Creates the I2C sensor object and checks common addresses  | The project needs real environmental data to send by Bluetooth |
| BLEUART service  | Creates the Bluetooth link to the phone app                | This is the communication channel for the readings             |
| Command handling | Lets the phone request the latest temperature and humidity | The app receives sensor data only when asked                   |
| help response    | Lists the supported commands                               | Beginners can recover quickly if they forget the command names |

## Troubleshooting

| Problem                        | Possible Cause                                                         | Solution                                               |
| ------------------------------ | ---------------------------------------------------------------------- | ------------------------------------------------------ |
| No BME280 readings             | Wrong I2C pins or wrong address                                        | Use GPIO 8/9 and test `0x76` or `0x77`                 |
| Phone cannot find Pico-Weather | BLE libraries are missing or Bluetooth is not active                   | Check the library files and rerun the advertising test |
| Only one value appears         | The app may only show one line at a time depending on the BLE terminal | Scroll in the app or resend the command                |
