This project demonstrates using the same BLE communication idea from the LED project but with sound output.

The final result should let a phone turn the buzzer on, turn it off, or play short beep patterns.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import Pin
import bluetooth
import time
from ble_uart import BLEUART

buzzer = Pin(0, Pin.OUT)
buzzer.off()

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Buzzer')


def beep(count=1):
    for _ in range(count):
        buzzer.on()
        time.sleep(0.15)
        buzzer.off()
        time.sleep(0.15)


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    if command in ('1', 'on'):
        buzzer.on()
        uart.write(b'Buzzer ON\n')
    elif command in ('0', 'off'):
        buzzer.off()
        uart.write(b'Buzzer OFF\n')
    elif command == 'beep':
        beep(1)
        uart.write(b'Beep played\n')
    elif command == 'double':
        beep(2)
        uart.write(b'Double beep played\n')
    elif command == 'help':
        uart.write(b'Commands: on, off, beep, double, help\n')
    else:
        uart.write(b'Unknown command. Send help.\n')


uart.on_rx(on_rx)

print('Bluetooth buzzer controller ready')
print('Connect with a BLE app and send commands: on, off, beep, double, help')

while True:
    time.sleep(0.1)
```

## How the Code Works

| Code Section     | What It Does                                          | Why It Matters                                        |
| ---------------- | ----------------------------------------------------- | ----------------------------------------------------- |
| Buzzer setup     | Creates the active buzzer output on GPIO 0            | The Bluetooth commands need a sound device to control |
| beep() function  | Creates one or more short buzzer sounds               | A function keeps repeated sound code easy to reuse    |
| BLEUART service  | Creates the Bluetooth communication link to the phone | This is how the phone sends control messages          |
| on_rx() callback | Processes incoming text commands                      | This is where the buzzer actions are chosen           |

## Troubleshooting

| Problem                       | Possible Cause                                       | Solution                                                  |
| ----------------------------- | ---------------------------------------------------- | --------------------------------------------------------- |
| No sound                      | Buzzer wiring or buzzer type is wrong                | Check polarity and confirm you are using an active buzzer |
| Phone cannot find Pico-Buzzer | BLE libraries are missing or Bluetooth is not active | Check the library files and rerun the advertising test    |
| Commands do nothing           | The app is sending different text than expected      | Use simple commands like on, off, beep, or double         |
