# Project 1.12.25: Bluetooth Sensor Data Logger

**Beginner Embedded Systems Project Using Raspberry Pi Pico 2 W and MicroPython**


# Overview

Build a Bluetooth sensor data logger that sends numbered BME280 readings to your phone.

This project demonstrates how to count readings and stream them over Bluetooth in a simple logging session.

The final result should let a phone start logging, stop logging, and receive numbered temperature, humidity, and pressure samples.

# Full Project Code

```python
from machine import Pin, I2C
import bluetooth
import time
import BME280
from ble_uart import BLEUART

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=100000)
bme = BME280.BME280(i2c=i2c)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Logger')

logging_on = False
sample_count = 0
last_log_time = time.ticks_ms()
log_interval_ms = 2000


def send_one_sample():
    global sample_count

    sample_count += 1

    line = '#{} Temp:{} Hum:{} Press:{}'.format(
        sample_count,
        bme.temperature,
        bme.humidity,
        bme.pressure
    )

    uart.write((line + '\n').encode())
    print(line)


def on_rx(data):
    global logging_on, sample_count

    command = data.decode('utf-8').strip().lower()

    if command == 'start':
        logging_on = True
        uart.write(b'Logging started\n')

    elif command == 'stop':
        logging_on = False
        uart.write(b'Logging stopped\n')

    elif command == 'read':
        send_one_sample()

    elif command == 'reset':
        sample_count = 0
        uart.write(b'Log counter reset\n')

    elif command == 'status':
        state = 'ON' if logging_on else 'OFF'
        uart.write(
            ('Logging: {} Count:{}\n'.format(state, sample_count)).encode()
        )

    elif command == 'help':
        uart.write(
            b'Commands: start, stop, read, reset, status, help\n'
        )

    else:
        uart.write(b'Unknown command. Send help.\n')


uart.on_rx(on_rx)

print('Bluetooth sensor data logger ready')
print('Send start, stop, read, reset, status, or help')

while True:
    if (
        logging_on and
        time.ticks_diff(time.ticks_ms(), last_log_time) >= log_interval_ms
    ):
        send_one_sample()
        last_log_time = time.ticks_ms()

    time.sleep(0.1)
```

---

# How the Code Works

| Code Section          | What It Does                                                        | Why It Matters                                                |
| --------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------- |
| `logging_on` variable | Stores whether logging is currently active                          | The Pico knows when it should send repeated samples           |
| `sample_count`        | Numbers each logged reading                                         | Students can see the order of samples                         |
| `send_one_sample()`   | Builds one data line containing temperature, humidity, and pressure | Keeps the logging format consistent                           |
| Timed main loop       | Sends a new sample every few seconds while logging is active        | Creates automatic logging without blocking Bluetooth commands |

# Expected Result

After running the code, your BLE app should find **Pico-Logger**.

Sending:

- `start`
- `stop`
- `read`
- `reset`
- `status`

should allow you to control the logging session.

### Example Behavior

- `start` -> Begins streaming numbered sensor readings.
- `stop` -> Stops logging.
- `read` -> Sends one sensor sample immediately.
- `reset` -> Resets the sample counter.
- `status` -> Shows whether logging is active and how many samples have been sent.

# Troubleshooting

| Problem                         | Possible Cause                                        | Solution                                                                                       |
| ------------------------------- | ----------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| No I2C device found             | SDA and SCL are reversed or the BME280 is not powered | Verify GPIO 8, GPIO 9, 3.3V, and GND connections                                               |
| No samples appear after `start` | Logging is off or the BLE app disconnected            | Use `status` to verify the logger state and keep the BLE connection open                       |
| Phone cannot find Pico-Logger   | BLE helper files are missing or Bluetooth is inactive | Verify `ble_uart.py` and `ble_advertising.py` are installed and rerun the BLE advertising test |
