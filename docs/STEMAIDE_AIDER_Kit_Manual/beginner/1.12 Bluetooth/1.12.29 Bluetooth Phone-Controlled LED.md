## Overview

Build a Bluetooth-controlled LED that can be switched from a phone app.

This project demonstrates Bluetooth Low Energy communication between a Pico 2 W and a smartphone.

The final result should let you send commands such as `on`, `off`, `toggle`, and `status` from a BLE app to control the LED.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import Pin
import bluetooth
import time
from ble_uart import BLEUART

led = Pin(0, Pin.OUT)
led.off()

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-LED')


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    if command in ('1', 'on'):
        led.on()
        uart.write(b'LED is ON\n')
    elif command in ('0', 'off'):
        led.off()
        uart.write(b'LED is OFF\n')
    elif command == 'toggle':
        led.value(0 if led.value() else 1)
        uart.write(b'LED toggled\n')
    elif command == 'status':
        state = 'ON' if led.value() else 'OFF'
        uart.write(('LED status: ' + state + '\n').encode())
    elif command == 'help':
        uart.write(b'Commands: on, off, toggle, status, help\n')
    else:
        uart.write(b'Unknown command. Send help.\n')


uart.on_rx(on_rx)

print('Bluetooth LED controller ready')
print('Connect with a BLE app and send commands: on, off, toggle, status, help')

while True:
    time.sleep(0.1)
```

## How the Code Works

| Code Section            | What It Does                                          | Why It Matters                                           |
| ----------------------- | ----------------------------------------------------- | -------------------------------------------------------- |
| BLEUART setup           | Creates a Bluetooth UART-style service named Pico-LED | This lets a phone app send text commands easily          |
| LED setup               | Creates the LED output on GPIO 0                      | The Bluetooth commands need a physical output to control |
| on_rx() callback        | Runs whenever the phone sends data                    | This is where incoming commands are decoded and acted on |
| status and help replies | Send text back to the phone app                       | Students get feedback that the command worked            |

## Troubleshooting

| Problem                                | Possible Cause                                               | Solution                                                 |
| -------------------------------------- | ------------------------------------------------------------ | -------------------------------------------------------- |
| Phone cannot find Pico-LED             | BLE libraries are missing or Bluetooth is not active         | Check the library files and rerun the advertising test   |
| Phone connects but LED does not change | LED wiring or command text is wrong                          | Run the LED test and send simple commands like on or off |
| Unknown command reply appears          | Text sent from the app does not match the supported commands | Send one of: on, off, toggle, status, help               |
