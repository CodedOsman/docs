# Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import Pin
import bluetooth
import time
from ble_uart import BLEUART

pir = Pin(1, Pin.IN)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Motion')

last_motion = 0
motion_count = 0


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)
    if command == 'status':
        uart.write(('Motion count: {}\n'.format(motion_count)).encode())
    elif command == 'help':
        uart.write(b'Commands: status, help\n')
    else:
        uart.write(b'Unknown command. Send help.\n')

uart.on_rx(on_rx)

print('Bluetooth motion alert ready')
print('Wait 15 seconds for PIR warm-up')
time.sleep(15)
print('Connect with a BLE app and wait for alerts or send status')

while True:
    motion = pir.value()
    if motion == 1 and last_motion == 0:
        motion_count += 1
        uart.write(b'MOTION DETECTED!\n')
        print('Motion detected! Count:', motion_count)
    last_motion = motion
    time.sleep(0.1)
```

# How the Code Works

| Code Section     | What It Does                                    | Why It Matters                                                |
| ---------------- | ----------------------------------------------- | ------------------------------------------------------------- |
| PIR input        | Reads whether motion is currently detected      | This is the event source for the Bluetooth alert              |
| Edge detection   | Sends an alert only when motion starts          | This prevents one long motion event from repeating constantly |
| `motion_count`   | Stores how many motion events have happened     | The phone can request this total later                        |
| BLE status reply | Lets the phone ask for the current motion count | This adds useful feedback beyond the automatic alert          |

# Troubleshooting

| Problem                       | Possible Cause                                       | Solution                                                       |
| ----------------------------- | ---------------------------------------------------- | -------------------------------------------------------------- |
| No motion alerts appear       | PIR is still warming up or wiring is wrong           | Wait longer and recheck the PIR OUT pin on GPIO 1              |
| Too many alerts               | One movement is causing repeated triggers            | Keep the edge detection code and reduce nearby constant motion |
| Phone cannot find Pico-Motion | BLE libraries are missing or Bluetooth is not active | Check the library files and rerun the advertising test         |
