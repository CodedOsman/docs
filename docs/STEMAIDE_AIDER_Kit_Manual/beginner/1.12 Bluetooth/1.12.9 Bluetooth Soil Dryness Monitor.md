# Overview

Build a Bluetooth soil dryness monitor that reports moisture level to your phone.

This project demonstrates how an analog sensor reading can be turned into a simple percentage.

The final result should let a phone request the current soil moisture percentage and see whether the soil is dry or okay.

# Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import ADC, Pin
import bluetooth
import time
from ble_uart import BLEUART

soil = ADC(Pin(26))

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Soil')


def get_moisture_percent():
    raw = soil.read_u16()
    return 100 - int((raw * 100) / 65535)


def moisture_status(percent):
    if percent < 30:
        return 'DRY'
    return 'OK'


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    moisture = get_moisture_percent()
    status = moisture_status(moisture)

    if command in ('read', 'status', 'moisture'):
        uart.write(('Moisture: {}%\n'.format(moisture)).encode())
        uart.write(('Status: {}\n'.format(status)).encode())
    elif command == 'help':
        uart.write(b'Commands: read, status, moisture, help\n')
    else:
        uart.write(b'Unknown command. Send help.\n')

uart.on_rx(on_rx)

last_status = moisture_status(get_moisture_percent())
print('Bluetooth soil monitor ready')
print('Connect with a BLE app and send: read, status, moisture, or help')

while True:
    moisture = get_moisture_percent()
    current_status = moisture_status(moisture)
    if current_status != last_status:
        uart.write(('Status changed: {} ({}%)\n'.format(current_status, moisture)).encode())
        print('Status changed:', current_status, moisture)
        last_status = current_status
    time.sleep(1)
```

# How the Code Works

| Code Section             | What It Does                                                 | Why It Matters                                      |
| ------------------------ | ------------------------------------------------------------ | --------------------------------------------------- |
| `get_moisture_percent()` | Converts the raw ADC value into a percentage                 | A percentage is easier for beginners to understand  |
| `moisture_status()`      | Labels the reading as DRY or OK                              | This gives a quick decision from the sensor data    |
| `on_rx()` callback       | Sends the current reading to the phone when asked            | Students can request live moisture data at any time |
| Status change check      | Sends a Bluetooth message when the dry or okay state changes | This turns the project into an automatic alert      |

---

# Expected Result

After running the code, your phone BLE app should find Pico-Soil. Sending `read`, `status`, or `moisture` should return the current moisture percentage and a status such as DRY or OK. If the soil changes enough to cross the threshold, the Pico should also send an update message.

# Troubleshooting

| Problem                      | Possible Cause                                                 | Solution                                                                                               |
| ---------------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| Moisture value never changes | Sensor wiring is wrong or the probe is not changing conditions | Check AOUT on GPIO 26 and compare dry and wet readings                                                 |
| Phone cannot find Pico-Soil  | BLE helper files are missing or Bluetooth is not active        | Check that `ble_uart.py` and `ble_advertising.py` are saved on the Pico and rerun the advertising test |
| Percentage seems inaccurate  | Different soil sensors need calibration                        | Treat the number as an estimate and adjust the dry threshold after testing your own sensor             |
