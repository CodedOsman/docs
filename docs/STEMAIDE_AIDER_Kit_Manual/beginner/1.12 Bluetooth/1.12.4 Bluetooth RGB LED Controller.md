## Bluetooth RGB LED Controller

## Overview

Build a Bluetooth-controlled RGB LED that changes color when commands are sent from a phone.

This project demonstrates using PWM on three color channels and sending color data wirelessly.

The final result should let the phone set named colors or send red, green, and blue values like 255,0,0.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import Pin, PWM
import bluetooth
import time
from ble_uart import BLEUART

red = PWM(Pin(0))
green = PWM(Pin(1))
blue = PWM(Pin(2))

for pwm in (red, green, blue):
    pwm.freq(1000)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-RGB')


def set_rgb(r, g, b):
    red.duty_u16(r * 257)
    green.duty_u16(g * 257)
    blue.duty_u16(b * 257)


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    if command == 'red':
        set_rgb(255, 0, 0)
        uart.write(b'Red set\n')
    elif command == 'green':
        set_rgb(0, 255, 0)
        uart.write(b'Green set\n')
    elif command == 'blue':
        set_rgb(0, 0, 255)
        uart.write(b'Blue set\n')
    elif command == 'white':
        set_rgb(255, 255, 255)
        uart.write(b'White set\n')
    elif command == 'off':
        set_rgb(0, 0, 0)
        uart.write(b'LED off\n')
    else:
        try:
            parts = command.split(',')
            if len(parts) == 3:
                r = max(0, min(255, int(parts[0])))
                g = max(0, min(255, int(parts[1])))
                b = max(0, min(255, int(parts[2])))
                set_rgb(r, g, b)
                uart.write(b'Custom color set\n')
            else:
                uart.write(b'Send red, green, blue, white, off, or R,G,B\n')
        except ValueError:
            uart.write(b'Send red, green, blue, white, off, or R,G,B\n')


uart.on_rx(on_rx)
set_rgb(0, 0, 0)

print('Bluetooth RGB controller ready')
print('Send red, green, blue, white, off, or values like 255,0,0')

while True:
    time.sleep(0.1)
```

## How the Code Works

| Code Section         | What It Does                                                 | Why It Matters                                   |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------ |
| PWM RGB setup        | Creates three PWM channels for red, green, and blue          | An RGB LED needs separate control for each color |
| set_rgb()            | Converts 0-255 color values into PWM duty values             | This makes color commands easier to understand   |
| Named color commands | Support quick commands like red, green, blue, white, and off | These are easy for beginners to test             |
| Custom R,G,B input   | Lets the phone send exact color values such as 255,0,0       | Students can experiment with color mixing        |

## Troubleshooting

| Problem                    | Possible Cause                                                    | Solution                                                    |
| -------------------------- | ----------------------------------------------------------------- | ----------------------------------------------------------- |
| Wrong color appears        | RGB legs are wired to different pins or the LED type is different | Recheck the leg order and confirm the LED is common cathode |
| Custom color command fails | The phone did not send the values in R,G,B format                 | Send values like 255,0,0 with commas and no extra words     |
| One color is missing       | That resistor or color leg is not connected correctly             | Test each color again with the RGB LED test code            |
