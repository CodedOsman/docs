This project demonstrates combining an analog sensor with Bluetooth messages.

The final result should let the phone request the current moisture level and show whether the plant needs water.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import ADC, Pin
import bluetooth
import time
from ble_uart import BLEUART

soil = ADC(Pin(26))

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Plant')


def get_moisture_percent():
    raw = soil.read_u16()
    return 100 - int((raw / 65535) * 100)


def plant_status(percent):
    if percent < 30:
        return 'NEEDS WATER!'
    return 'OK'


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    moisture = get_moisture_percent()
    status = plant_status(moisture)

    if command in ('read', 'status', 'moisture'):
        uart.write(('Moisture: {}%\n'.format(moisture)).encode())
        uart.write(('Status: {}\n'.format(status)).encode())
    elif command == 'help':
        uart.write(b'Commands: read, status, moisture, help\n')
    else:
        uart.write(b'Unknown command. Send help.\n')


uart.on_rx(on_rx)

print('Bluetooth plant moisture monitor ready')
print('Connect with a BLE app and send read, status, moisture, or help')

while True:
    time.sleep(0.1)
```

## How the Code Works

| Code Section      | What It Does                                                   | Why It Matters                                                        |
| ----------------- | -------------------------------------------------------------- | --------------------------------------------------------------------- |
| ADC moisture read | Reads the soil sensor and converts it into a percentage        | The BLE app needs a simple moisture value instead of a raw ADC number |
| plant_status()    | Turns the moisture percentage into a beginner-friendly message | This helps students interpret the reading quickly                     |
| BLEUART service   | Creates the Bluetooth text link to the phone                   | This is how the sensor data is requested                              |
| on_rx() callback  | Sends the moisture and status reply when the phone asks for it | The app gets live plant information on demand                         |

## Troubleshooting

| Problem                      | Possible Cause                                                   | Solution                                                        |
| ---------------------------- | ---------------------------------------------------------------- | --------------------------------------------------------------- |
| Moisture never changes       | Sensor is not connected correctly or not in different conditions | Check AOUT on GPIO 26 and test dry and wet conditions           |
| Phone cannot find Pico-Plant | BLE libraries are missing or Bluetooth is not active             | Check the library files and rerun the advertising test          |
| Status seems wrong           | The threshold is not suitable for your sensor                    | Adjust the 30% threshold after testing your own sensor readings |
