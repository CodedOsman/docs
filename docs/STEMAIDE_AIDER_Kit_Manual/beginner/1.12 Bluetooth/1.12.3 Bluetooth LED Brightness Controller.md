## Bluetooth LED Brightness Controller

## Overview

Build a Bluetooth-controlled LED brightness controller using PWM and direct GPIO output.

This project demonstrates using a phone to send brightness values to an LED controller.

The final result should let you send a brightness percentage from a BLE app and have the LED brightness change in response.

## Full Project Code

Upload and run this code after the individual tests work correctly.

```python
from machine import Pin, PWM
import bluetooth
import time
from ble_uart import BLEUART

led = PWM(Pin(0))
led.freq(1000)
current_brightness = 0

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-LED')


def set_brightness(percent):
    global current_brightness
    percent = max(0, min(100, percent))
    current_brightness = percent
    duty = int((percent / 100) * 65535)
    led.duty_u16(duty)


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    if command in ('off', 'stop'):
        set_brightness(0)
        uart.write(b'LED off\n')
    elif command == 'status':
        uart.write(('Brightness: {}%\n'.format(current_brightness)).encode())
    else:
        try:
            value = int(command)
            set_brightness(value)
            uart.write(('Brightness set to {}%\n'.format(current_brightness)).encode())
        except ValueError:
            uart.write(b'Send 0-100, stop, off, or status\n')


uart.on_rx(on_rx)
set_brightness(0)

print('Bluetooth LED brightness controller ready')
print('Connect with a BLE app and send a brightness from 0 to 100')

while True:
    time.sleep(0.1)
```

## How the Code Works

| Code Section              | What It Does                                                    | Why It Matters                                     |
| ------------------------- | --------------------------------------------------------------- | -------------------------------------------------- |
| PWM LED setup             | Creates a PWM output on GPIO 0                                  | PWM changes the average power going to the LED     |
| set_brightness()          | Converts a percentage into a PWM duty value                     | Students can think in easy 0-100% brightness steps |
| Numeric Bluetooth command | Reads a number from the phone and uses it as brightness percent | This is the core remote brightness-control feature |
| Status reply              | Sends the current brightness back to the phone app              | The user can confirm what brightness is active     |

## Troubleshooting

| Problem                        | Possible Cause                                                     | Solution                                                          |
| ------------------------------ | ------------------------------------------------------------------ | ----------------------------------------------------------------- |
| LED does not turn on           | LED polarity is reversed or the resistor path is open              | Flip the LED direction and recheck the GPIO 0 resistor connection |
| LED always stays off or bright | PWM value is not changing or the LED is connected to the wrong pin | Rerun the PWM LED test and confirm GPIO 0 is used                 |
| Phone commands do nothing      | The app is sending text that is not a number or supported command  | Send simple values like 25 or commands like stop                  |
