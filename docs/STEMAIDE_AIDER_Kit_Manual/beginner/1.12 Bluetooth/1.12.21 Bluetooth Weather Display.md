# Project 1.12.21: Bluetooth Weather Display

**Beginner Embedded Systems Project Using Raspberry Pi Pico 2 W and MicroPython**


# Overview

Build a Bluetooth weather display that sends temperature, humidity, and pressure readings to your phone.

This project demonstrates how one digital sensor can act like a simple wireless weather station.

The final result should let a phone request the current weather readings and view them in the BLE app.

# Full Project Code

```python
from machine import Pin, I2C
import bluetooth
import time
import BME280
from ble_uart import BLEUART

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=100000)
bme = BME280.BME280(i2c=i2c)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name='Pico-Weather')


def send_weather():
    uart.write(('Temperature: {}\n'.format(bme.temperature)).encode())
    uart.write(('Humidity: {}\n'.format(bme.humidity)).encode())
    uart.write(('Pressure: {}\n'.format(bme.pressure)).encode())


def on_rx(data):
    command = data.decode('utf-8').strip().lower()
    print('Received command:', command)

    if command in ('read', 'status', 'weather'):
        send_weather()

    elif command == 'help':
        uart.write(b'Commands: read, status, weather, help\n')

    else:
        uart.write(b'Unknown command. Send help.\n')


uart.on_rx(on_rx)

print('Bluetooth weather display ready')
print('Send read, status, weather, or help from the BLE app')

while True:
    time.sleep(1)
```

# How the Code Works

| Code Section              | What It Does                                     | Why It Matters                                              |
| ------------------------- | ------------------------------------------------ | ----------------------------------------------------------- |
| I2C sensor setup          | Starts the BME280 sensor on the I2C bus          | The correct pins and driver are needed for weather readings |
| `send_weather()`          | Builds and sends the current sensor readings     | The phone receives a simple weather report                  |
| Bluetooth command handler | Waits for read or status requests from the phone | Avoids sending repeated data all the time                   |
| Main loop                 | Keeps the Bluetooth project running              | The Pico stays ready for the next request                   |

# Troubleshooting

| Problem                        | Possible Cause                                          | Solution                                                             |
| ------------------------------ | ------------------------------------------------------- | -------------------------------------------------------------------- |
| No I2C device is found         | SDA and SCL are reversed or the BME280 is not powered   | Check GPIO 8, GPIO 9, 3.3V, and GND wiring                           |
| Import error for BME280        | The `BME280.py` file is missing from the Pico           | Save `BME280.py` to the Pico root folder and try again               |
| Phone cannot find Pico-Weather | BLE helper files are missing or Bluetooth is not active | Verify helper files are installed and rerun the BLE advertising test |
