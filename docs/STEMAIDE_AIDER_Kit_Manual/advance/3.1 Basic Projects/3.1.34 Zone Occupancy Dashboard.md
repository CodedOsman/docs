## Overview

The Pico reads three PIR sensors and shows how many monitored zones are currently occupied on a local OLED dashboard.

Shared spaces are easier to manage when users can see which zones are active instead of guessing from one sensor only.

A Pico 2 W prototype with three PIR sensors and an OLED that acts as a local occupancy dashboard.

Multi-sensor integration, simple zone logic, and honest explanation of what a local dashboard prototype can claim.

## Step-by-Step Assembly

1. Connect each PIR sensor output to its assigned GPIO pin for zones A, B, and C.
2. Connect the PIR power and ground rails and allow the sensors to warm up before validation.
3. Connect the OLED to Pico 3.3V, GND, GPIO 20, and GPIO 21.
4. Upload ssd1306.py and confirm that the library appears in the Pico file list.
5. Aim each PIR at a distinct test zone so you can validate overlap and blind spots separately.

## Testing Individual Components

Before running the full project, test each subsystem separately. This makes it easier to find wiring, library, or logic problems before full integration.

1. **Hardware setup**: Assemble the Pico, sensor, indicator, and load wiring exactly as shown in the connection table before applying power.
2. **Test the input sensor**: Print the state of each PIR separately and walk through each zone to confirm the mapping is correct.
3. **Test the output device**: Run a short OLED test so the display path is known to work before you add occupancy data.
4. **Test the decision logic**: Confirm that the total occupancy count changes correctly when one, two, or three zones are active.
5. **Run the full system**: Run the full dashboard while moving through each zone and compare the serial output with the OLED view.
6. **Validate the prototype**: Check whether overlapping PIR fields make the dashboard over-report activity.
7. **Save the project**: Save the validated program on the Pico as main.py and keep a copy on the computer for future edits.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import time

try:
    from ssd1306 import SSD1306_I2C
    HAS_OLED = True
except ImportError:
    HAS_OLED = False

ZONE_A_PIN = 14
ZONE_B_PIN = 15
ZONE_C_PIN = 16
I2C_SDA_PIN = 20
I2C_SCL_PIN = 21

zone_a = Pin(ZONE_A_PIN, Pin.IN, Pin.PULL_DOWN)
zone_b = Pin(ZONE_B_PIN, Pin.IN, Pin.PULL_DOWN)
zone_c = Pin(ZONE_C_PIN, Pin.IN, Pin.PULL_DOWN)
i2c = I2C(0, sda=Pin(I2C_SDA_PIN), scl=Pin(I2C_SCL_PIN), freq=400000)
if HAS_OLED:
    oled = SSD1306_I2C(128, 64, i2c)


def occupancy_label(total):
    if total >= 3:
        return "HIGH"
    if total == 2:
        return "MEDIUM"
    if total == 1:
        return "LOW"
    return "IDLE"


print("Zone occupancy dashboard ready")

while True:
    a = zone_a.value()
    b = zone_b.value()
    c = zone_c.value()
    total = a + b + c
    label = occupancy_label(total)

    print("A:{} B:{} C:{} Total:{} Label:{}".format(a, b, c, total, label))

    if HAS_OLED:
        oled.fill(0)
        oled.text("Zone Dashboard", 0, 0)
        oled.text("A:{} B:{} C:{}".format(a, b, c), 0, 20)
        oled.text("Total:{} {}".format(total, label), 0, 40)
        oled.show()

    time.sleep(1)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Multi-sensor setup | Initializes three PIR inputs and the optional OLED display | The project becomes more advanced by combining multiple zones instead of one sensor only | Confirm each PIR is mapped to the intended zone before changing any dashboard logic |
| occupancy_label | Turns the active-zone total into a simple LOW, MEDIUM, or HIGH status | A category is easier to read than only raw numbers | Change the label logic if your real use case needs different occupancy bands |
| Main loop | Reads the three zones, calculates the total, and prints the snapshot status | Students can see how multi-sensor data fusion works in a simple local dashboard | Keep the update interval slow enough that the OLED remains readable |
| OLED update | Displays per-zone activity and the total local occupancy label | This demonstrates a dashboard without overclaiming cloud analytics | If the display is blank, verify ssd1306.py and the I2C wiring before changing the code |

### Validation Checks

- **Normal condition test**: confirm the system stays in its safe or idle state under baseline conditions
- **Warning condition test**: move the input close to the chosen threshold and confirm the transition is sensible
- **Critical condition test**: trigger the strongest alert or control state and confirm the output response is correct
- **Calibration test**: compare the sensor or timing against a known reference or repeated trial
- **Limitation test**: deliberately create an awkward or noisy condition and note how the prototype behaves

### Deployment and Limitations

- This prototype works well for classroom demonstrations and local monitoring pilots
- Before deployment, it needs calibration records, protective mounting, and regular validation checks
- Display-only systems still depend on correct sensor placement and trained users

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| One zone never changes | That PIR sensor is wired wrong or aimed badly | Test the silent zone by itself and confirm its output pin wiring |
| Total occupancy seems too high | PIR coverage overlaps between zones | Reposition the sensors so each one watches a distinct area |
| OLED is blank | I2C wiring or library file is missing | Use GPIO 20 and GPIO 21 and confirm that ssd1306.py is on the Pico |
| All zones look active at once | PIR warm-up or room layout is causing false triggers | Wait for warm-up and test each zone in isolation first |
