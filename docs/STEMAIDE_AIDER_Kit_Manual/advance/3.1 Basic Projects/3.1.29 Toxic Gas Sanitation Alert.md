### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules.
- Allow the gas sensor to warm up before calibration. The exact threshold values depend heavily on your sensor, room air, and module setup.

### Project-Specific Safety Note

Gas sensor readings are for learning and prototype use only and should not be treated as certified safety measurements.

Many MQ sensor modules use a 5V heater and can output up to 5V on the analog pin. Use a resistor divider or a 3.3V-safe conditioning board before connecting the analog output to the Pico ADC.

Do not use this prototype as a certified health or safety monitor.

## Circuit Connections


|---------------|-------------|---------------------------------|-------|
| MQ sensor VCC | Module-rated supply | Often external 5V | Check the board label before powering |
| MQ sensor GND | GND | Physical pin 38 | Common ground |
| MQ sensor AO | Voltage-divider midpoint to GPIO 26 | GPIO 26 / physical pin 31 | Do not connect a 5V analog output directly |
| LED anode | 220 ohm resistor to GPIO 15 | GPIO 15 / physical pin 20 | Warning LED |
| Buzzer positive | GPIO 16 | GPIO 16 / physical pin 21 | Critical buzzer |

---

## Wiring Diagram

```
  MQ AO -> resistor divider -> GPIO 26
  GND -> MQ GND, LED cathode, buzzer negative
  GPIO 15 -> 220 ohm resistor -> LED anode
  GPIO 16 -> buzzer positive
  External supply -> MQ sensor heater power if required
```

---

## Step-by-Step Assembly

1. Power the MQ sensor according to its board label and connect its ground to Pico GND.
2. Build a resistor divider on the analog output before connecting the divider midpoint to GPIO 26.
3. Connect the LED to GPIO 15 through a 220 ohm resistor and connect the LED cathode to GND.
4. Connect the buzzer positive pin to GPIO 16 and its negative pin to GND.
5. Allow the sensor to warm up before recording any calibration values.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

GAS_PIN = 26
LED_PIN = 15
BUZZER_PIN = 16
WARNING_LEVEL = 18000
CRITICAL_LEVEL = 30000
WARMUP_SECONDS = 60

gas = ADC(GAS_PIN)
led = Pin(LED_PIN, Pin.OUT)
buzzer = Pin(BUZZER_PIN, Pin.OUT)


def read_level(samples=10):
    total = 0
    for _ in range(samples):
        total += gas.read_u16()
        time.sleep_ms(20)
    return total // samples


print("Warming gas sensor...")
for remaining in range(WARMUP_SECONDS, 0, -1):
    print("Warm-up remaining:", remaining)
    time.sleep(1)

print("Gas alert ready")

while True:
    level = read_level()

    if level >= CRITICAL_LEVEL:
        status = "CRITICAL"
        led.on()
        buzzer.on()
    elif level >= WARNING_LEVEL:
        status = "WARNING"
        led.on()
        buzzer.off()
    else:
        status = "NORMAL"
        led.off()
        buzzer.off()

    print("Gas level: {} | Status: {}".format(level, status))
    time.sleep(2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Warm-up phase | Waits before trusting the gas sensor output | MQ sensors need time to stabilize before calibration is meaningful | Increase WARMUP_SECONDS if your module needs a longer initial settle time |
| read_level | Averages several analog readings into one gas-level value | Averaging makes it easier to compare room baseline and elevated conditions | Increase the sample count if the value is too noisy |
| Threshold logic | Separates NORMAL, WARNING, and CRITICAL gas states | Clear states make it easier to validate what the prototype is doing | Set WARNING_LEVEL and CRITICAL_LEVEL from real room tests instead of guessing |
| Alert outputs | Turns on the LED for warning and both outputs for critical conditions | Students can see graded response instead of a single alarm only | Keep the buzzer reserved for the critical state if classroom testing must stay manageable |

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| Gas value is always zero or flat | Analog output or divider wiring is incorrect | Rebuild the AO divider and confirm the midpoint reaches GPIO 26 |
| System is always in alarm | Thresholds are too low or the room baseline is high | Record the stable warm baseline and raise WARNING_LEVEL and CRITICAL_LEVEL appropriately |
| Readings drift a lot | Sensor has not warmed fully or room airflow is changing rapidly | Extend warm-up time and test in a more stable airflow condition |
| Buzzer never activates | Critical threshold is too high or the buzzer wiring is wrong | Lower CRITICAL_LEVEL briefly for a safe test and verify the buzzer separately |
