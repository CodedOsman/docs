## Overview

The Pico reads an MQ-4 style methane sensor and raises graded local alerts when the gas level rises above calibrated thresholds.

Methane leaks around waste sites, pits, and enclosed service areas can create explosion and health risks if they are not noticed early.

A Pico 2 W prototype with a methane gas sensor, LED, buzzer, and analog threshold logic.

Sensor warm-up, methane threshold calibration, and honest discussion of non-certified gas safety measurements.

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules.
- Allow the MQ-4 sensor to warm up fully before collecting baseline values. Thresholds depend on your module, airflow, and room conditions.

### Project-Specific Safety Note

Gas sensor readings are for learning and prototype use only and should not be treated as certified safety measurements.

Many MQ sensor modules use a 5V heater and can output up to 5V on the analog pin. Use a resistor divider or a 3.3V-safe conditioning board before connecting the analog output to the Pico ADC.

Gas sensor readings are for prototype use only and must not replace certified methane safety instruments.

## Circuit Connections

|---------------|-------------|---------------------------------|-------|
| MQ-4 VCC | Module-rated supply | Often external 5V | Check the board label |
| MQ-4 GND | GND | Physical pin 38 | Common ground |
| MQ-4 AO | Voltage-divider midpoint to GPIO 26 | GPIO 26 / physical pin 31 | Do not connect a 5V analog output directly |
| LED anode | 220 ohm resistor to GPIO 15 | GPIO 15 / physical pin 20 | Warning LED |
| Buzzer positive | GPIO 16 | GPIO 16 / physical pin 21 | Critical alarm |

---

## Wiring Diagram

```
  MQ-4 AO -> resistor divider -> GPIO 26
  GND -> MQ-4 GND, LED cathode, buzzer negative
  GPIO 15 -> 220 ohm resistor -> LED anode
  GPIO 16 -> buzzer positive
  External supply -> MQ-4 heater power if required
```

---

## Step-by-Step Assembly

1. Power the MQ-4 sensor according to its board label and connect its ground to Pico GND.
2. Build a resistor divider on the analog output before connecting the midpoint to GPIO 26.
3. Connect the LED to GPIO 15 through a 220 ohm resistor and the LED cathode to GND.
4. Connect the buzzer positive pin to GPIO 16 and its negative pin to GND.
5. Place the sensor in a ventilated test area and allow warm-up before calibration.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

GAS_PIN = 26
LED_PIN = 15
BUZZER_PIN = 16
WARNING_LEVEL = 18000
CRITICAL_LEVEL = 30000
WARMUP_SECONDS = 60

gas = ADC(GAS_PIN)
led = Pin(LED_PIN, Pin.OUT)
buzzer = Pin(BUZZER_PIN, Pin.OUT)


def read_level(samples=10):
    total = 0
    for _ in range(samples):
        total += gas.read_u16()
        time.sleep_ms(20)
    return total // samples


print("Warming methane sensor...")
for remaining in range(WARMUP_SECONDS, 0, -1):
    print("Warm-up remaining:", remaining)
    time.sleep(1)

print("Methane leak detector ready")

while True:
    level = read_level()

    if level >= CRITICAL_LEVEL:
        status = "CRITICAL"
        led.on()
        buzzer.on()
    elif level >= WARNING_LEVEL:
        status = "WARNING"
        led.on()
        buzzer.off()
    else:
        status = "NORMAL"
        led.off()
        buzzer.off()

    print("Methane level: {} | Status: {}".format(level, status))
    time.sleep(2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Warm-up phase | Waits for the MQ-4 sensor to stabilize | Methane decisions are meaningless if the sensor heater has not settled | Increase WARMUP_SECONDS if your module needs a longer initial warm-up |
| read_level | Averages several analog samples into one gas level | Averaging reduces noise and makes threshold testing more reliable | Increase the sample count if the values still jump too much |
| Threshold logic | Separates NORMAL, WARNING, and CRITICAL methane states | Graded states are easier to validate and explain than a single alarm point | Tune WARNING_LEVEL and CRITICAL_LEVEL using real baseline data |
| Alert outputs | Uses LED for warning and both LED and buzzer for critical conditions | Students can see how one sensor can support multiple response levels | Keep the buzzer reserved for CRITICAL if classroom testing needs to stay manageable |

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| Reading is always low or flat | The analog output or divider wiring is incorrect | Rebuild the divider and confirm the AO line reaches GPIO 26 |
| System is always in alarm | Thresholds are too low for the room baseline | Record the warm baseline and raise the thresholds accordingly |
| Values drift a lot | Sensor is not fully warmed or airflow is unstable | Extend warm-up time and test in a more stable airflow condition |
| Buzzer never sounds | Critical threshold is too high or the buzzer is miswired | Temporarily lower CRITICAL_LEVEL and verify the buzzer separately |
