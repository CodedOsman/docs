## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import Pin
import time

PIR_PIN = 14
OUTPUT_PIN = 15
LED_PIN = 16
HOLD_SECONDS = 20

pir = Pin(PIR_PIN, Pin.IN, Pin.PULL_DOWN)
output = Pin(OUTPUT_PIN, Pin.OUT)
led = Pin(LED_PIN, Pin.OUT)
last_motion_time = 0

print('Occupancy light controller ready')

while True:
    motion = pir.value() == 1
    now = time.time()
    if motion:
        last_motion_time = now

    active = motion or (now - last_motion_time) < HOLD_SECONDS
    output.value(1 if active else 0)
    led.value(1 if active else 0)
    print('Motion:{} Active:{}'.format(motion, active))
    time.sleep(0.2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Motion timestamp | Remembers the last time motion was seen | The hold-time logic depends on accurate motion timing | Watch the printed motion state while walking through the area |
| Hold-time rule | Keeps the output active for a fixed time after movement stops | This avoids lights turning off the moment a person stands still | Shorten or lengthen the hold time during user testing |
| Mirrored status LED | Shows when the controller considers the area active | The LED helps validate timing even when the real load is disconnected | Use the LED during safe dry-run testing before attaching a real load |

### Validation Checks

- **Normal condition test**: confirm the system stays in its safe or idle state under baseline conditions
- **Warning condition test**: move the input close to the chosen threshold and confirm the transition is sensible
- **Critical condition test**: trigger the strongest alert or control state and confirm the output response is correct
- **Calibration test**: compare the sensor or timing against a known reference or repeated trial
- **Limitation test**: deliberately create an awkward or noisy condition and note how the prototype behaves

### Deployment and Limitations

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| The light never turns off | The PIR keeps retriggering or the hold time is too long | Aim the sensor more narrowly and reduce the hold time |
| The light turns off too soon | The hold time is too short or motion is not being detected well | Increase HOLD_SECONDS and improve PIR placement |
| The relay or light never turns on | The output wiring is wrong | Test GPIO 15 with a simple on-off script before retrying the full logic |
| The PIR seems random | The sensor is still warming up or covering too wide an area | Wait for warm-up and re-aim the PIR toward the entry zone |
