## Overview

The Pico reads an analog TDS sensor, estimates dissolved-solids concentration, and displays the result locally.

Quick water screening can help students compare water sources and discuss what a sensor can and cannot prove.

A Pico 2 W prototype with a TDS sensor and OLED that shows raw readings, estimated ppm, and a local quality category.

Calibration, honest interpretation of a proxy measurement, and a clean local display workflow.

## Step-by-Step Assembly

1. Connect the TDS module power pins to Pico 3.3V and GND.
2. Connect the TDS analog output pin to GPIO 26 on the Pico.
3. Connect the OLED VCC and GND pins to Pico 3.3V and GND.
4. Connect OLED SDA to GPIO 20 and OLED SCL to GPIO 21.
5. Upload ssd1306.py to the Pico and confirm that the file appears in os.listdir().
6. Place the TDS probe in a clean sample cup only during testing, then dry the area around the electronics.

## Testing Individual Components

Before running the full project, test each subsystem separately. This makes it easier to find wiring, library, or logic problems before full integration.

1. **Hardware setup**: Assemble the Pico, sensor, indicator, and load wiring exactly as shown in the connection table before applying power.
2. **Test the input sensor**: Read and print raw ADC values first, then test at least two different water samples to confirm the values change in a believable direction.
3. **Test the output device**: Run a short OLED hello-world script so you know the display wiring and library file are correct before mixing it with sensor code.
4. **Test the decision logic**: Check that LOW, MODERATE, and HIGH TDS labels change only when the averaged reading crosses the defined thresholds.
5. **Run the full system**: Run the full display for several minutes and compare the serial data with the OLED output to catch mismatched updates.
6. **Validate the prototype**: Rinse the probe between samples, repeat the measurement, and note whether the reading stays stable enough for comparison work.
7. **Save the project**: Save the validated program on the Pico as main.py and keep a copy on the computer for future edits.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin, I2C
import time

try:
    from ssd1306 import SSD1306_I2C
    HAS_OLED = True
except ImportError:
    HAS_OLED = False

TDS_PIN = 26
I2C_SDA_PIN = 20
I2C_SCL_PIN = 21
SAMPLE_COUNT = 10
LOW_TDS_MAX = 300
MODERATE_TDS_MAX = 600

tds_sensor = ADC(TDS_PIN)
i2c = I2C(0, sda=Pin(I2C_SDA_PIN), scl=Pin(I2C_SCL_PIN), freq=400000)
if HAS_OLED:
    oled = SSD1306_I2C(128, 64, i2c)


def read_tds_ppm():
    total = 0
    for _ in range(SAMPLE_COUNT):
        total += tds_sensor.read_u16()
        time.sleep_ms(20)

    raw = total / SAMPLE_COUNT
    voltage = raw * 3.3 / 65535
    ppm = max(0, (133.42 * voltage ** 3 - 255.86 * voltage ** 2 + 857.39 * voltage) * 0.5)
    return raw, voltage, ppm


def classify(ppm):
    if ppm < LOW_TDS_MAX:
        return "LOW TDS"
    if ppm <= MODERATE_TDS_MAX:
        return "MODERATE TDS"
    return "HIGH TDS"


print("Water purity proxy display ready")

while True:
    raw, voltage, ppm = read_tds_ppm()
    status = classify(ppm)
    print("Raw: {:.0f} | Voltage: {:.2f} V | TDS: {:.0f} ppm | Status: {}".format(raw, voltage, ppm, status))

    if HAS_OLED:
        oled.fill(0)
        oled.text("Water Quality", 0, 0)
        oled.text("TDS:{:4.0f}ppm".format(ppm), 0, 20)
        oled.text(status, 0, 40)
        oled.show()

    time.sleep(2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Import and display setup | Loads the ADC and optional OLED driver | The project can still run in serial mode if the OLED library is missing during early tests | Verify I2C pins and upload ssd1306.py if the OLED stays blank |
| read_tds_ppm | Averages analog readings and converts the result into a voltage and approximate ppm value | Averaging helps remove random noise and makes sample comparison fairer | Adjust SAMPLE_COUNT if the display flickers or responds too slowly |
| classify | Maps ppm values to LOW, MODERATE, or HIGH dissolved-solids categories | Status categories help students discuss decision thresholds instead of staring only at numbers | Modify LOW_TDS_MAX and MODERATE_TDS_MAX after comparing your own sample set |
| Main loop | Prints the live reading and updates the OLED every two seconds | This keeps the display readable and leaves time for the analog signal to settle | Change the refresh interval if the probe needs more stabilization time |

## Expected Result

The serial monitor prints raw ADC, voltage, ppm estimate, and status every two seconds.

The OLED shows the estimated ppm and the current LOW, MODERATE, or HIGH TDS label.

Different water samples should produce noticeably different ppm values if the probe is working.

### Validation Checks

- **Normal condition test**: use clean tap or filtered water and observe the baseline ppm category
- **Warning condition test**: compare a mineral-rich or salt-added sample and observe the status shift upward
- **Sensor reading validation**: repeat the same sample three times and confirm the displayed value stays reasonably close
- **Calibration test**: rinse the probe, dry it, and compare filtered water versus salty water to understand the range
- **Limitation test**: note that cloudy, contaminated, or unsafe water may still appear similar to other samples because TDS alone is not a full safety test

### Deployment and Limitations

- This prototype is useful for classroom demonstrations, filtration comparisons, and local screening discussions
- Before real deployment, the sensor needs calibration records, enclosure protection, and a defined sampling procedure
- Because TDS does not detect all contaminants, trained users still need additional tests before making public-health decisions

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| OLED stays blank | I2C pins or library file are wrong | Use GPIO 20 and GPIO 21, upload ssd1306.py, and run a simple display test first |
| TDS value does not change | Probe is disconnected or dry | Check the analog output wire and make sure the probe tip is fully immersed in the sample |
| Readings are unstable | Probe is moving or sample is noisy | Hold the probe still, shorten the analog wiring, and average more samples |
| Display says HIGH TDS for all water | Sensor module output range or threshold assumptions are wrong | Measure raw voltage in several samples and recalculate the status thresholds for your module |
