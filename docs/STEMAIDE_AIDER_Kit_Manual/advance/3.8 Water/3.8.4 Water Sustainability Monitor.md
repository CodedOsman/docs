## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import time

try:
    from ssd1306 import SSD1306_I2C
    HAS_OLED = True
except ImportError:
    HAS_OLED = False

FLOW_PIN = 14
I2C_SDA_PIN = 20
I2C_SCL_PIN = 21
PULSES_PER_LITER = 450
WINDOW_SECONDS = 5

flow_sensor = Pin(FLOW_PIN, Pin.IN, Pin.PULL_UP)
i2c = I2C(0, sda=Pin(I2C_SDA_PIN), scl=Pin(I2C_SCL_PIN), freq=400000)
if HAS_OLED:
    oled = SSD1306_I2C(128, 64, i2c)

pulse_count = 0
total_liters = 0.0

def count_pulse(pin):
    global pulse_count
    pulse_count += 1

flow_sensor.irq(trigger=Pin.IRQ_FALLING, handler=count_pulse)

print('Flow dashboard ready')

while True:
    pulse_count = 0
    time.sleep(WINDOW_SECONDS)
    pulses = pulse_count
    liters = pulses / PULSES_PER_LITER
    total_liters += liters
    flow_lpm = liters * (60 / WINDOW_SECONDS)
    print('Pulses:{} Flow:{:.2f}L/min Total:{:.2f}L'.format(pulses, flow_lpm, total_liters))

    if HAS_OLED:
        oled.fill(0)
        oled.text('Flow Dash', 0, 0)
        oled.text('Rate:{:.1f}L/m'.format(flow_lpm), 0, 20)
        oled.text('Tot:{:.2f}L'.format(total_liters), 0, 40)
        oled.show()
```

### Validation Checks

- **Normal condition test**: confirm the system stays in its safe or idle state under baseline conditions
- **Warning condition test**: move the input close to the chosen threshold and confirm the transition is sensible
- **Critical condition test**: trigger the strongest alert or control state and confirm the output response is correct
- **Calibration test**: compare the sensor or timing against a known reference or repeated trial
- **Limitation test**: deliberately create an awkward or noisy condition and note how the prototype behaves

### Deployment and Limitations

- This prototype works well for classroom demonstrations and local monitoring pilots
- Before deployment, it needs calibration records, protective mounting, and regular validation checks
- Display-only systems still depend on correct sensor placement and trained users

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| The flow rate stays at zero | No pulses are being received | Check the signal wire, pull-up input, and sensor output voltage level |
| The total volume drifts from reality | The calibration factor is wrong | Measure a known volume and adjust the pulses-per-liter constant |
| The OLED is blank | The library is missing or the I2C pins are wrong | Confirm ssd1306.py is present and use GPIO 20 and GPIO 21 |
| Values jump around | Flow is unstable or the sensor wiring is loose | Secure the wiring and test with a steadier flow source |
