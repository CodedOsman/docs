## Overview

The Pico acts as a local farm orchestrator that decides when irrigation and supplemental lighting should turn on.

Farm automation becomes hard to explain when sensors are collected without a clear local control layer that decides which action should happen next.

A Pico 2 W prototype with soil, light, and climate sensing plus relay outputs for irrigation and grow-light control.

Multi-sensor orchestration, dual-output control, and honest wording for an OS-style supervisor that is still a prototype.

### Project Story

**Advanced Project**: This advanced project is designed to help learners move beyond basic wiring and coding into complete system thinking. The learner should build the prototype, test each subsystem, validate the data, explain the design decisions, and propose improvements for real-world deployment.

Small farms and greenhouses often need one controller to coordinate several simple tasks such as irrigation and lighting. This prototype helps students build a local supervisor that decides between multiple actions, while staying honest that it is not a full operating system or industrial controller.

---

## Learning Objectives

- Combine soil, light, temperature, and humidity data in one control loop
- Control two separate relay outputs from one Pico program
- Use thresholds and simple orchestration rules to decide between multiple actions
- Explain why a local supervisor is not the same as a full IoT operating system
- Test each actuator path safely before full system integration
- Discuss how scheduling and safety would need to improve for field deployment

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules.
- This project uses a local orchestration loop only. The title uses OS in the sense of a supervisor-style control layer, not a full operating system.

### Project-Specific Safety Note

If a relay module is controlled by the Pico, make sure the Pico GND and external power supply GND are connected together unless the relay module is fully opto-isolated and wired correctly.

Keep electronics away from water and dry the work area before powering the Pico.

Use an external power supply for pumps, fans, and other inductive loads.

Test both relay channels without the real loads first, then attach the pump and lamp only after the logic is verified.

## Step-by-Step Assembly

1. Connect the soil sensor analog output to GPIO 26 and plan to calibrate dry and wet references.
2. Build the light sensor analog path and connect it to GPIO 27.
3. Connect the DHT22 to Pico 3.3V, GND, and GPIO 15.
4. Connect relay channel 1 to GPIO 16 and relay channel 2 to GPIO 17, then power the relay module correctly.
5. Test both relay channels without the real pump or lamp before attaching the final loads.

## Testing Individual Components

Before running the full project, test each subsystem separately. This makes it easier to find wiring, library, or logic problems before full integration.

1. **Hardware setup**: Assemble the Pico, sensor, indicator, and load wiring exactly as shown in the connection table before applying power.
2. **Test the input sensor**: Test each sensor separately so you trust the dryness, light, and temperature values before controlling two outputs.
3. **Test the output device**: Toggle each relay channel from a short test script before running the supervisor logic.
4. **Test the decision logic**: Create dry and dark conditions separately so you can verify irrigation and lighting decisions one subsystem at a time.
5. **Run the full system**: Run the full orchestrator and compare the measured values with both relay states.
6. **Validate the prototype**: Review whether the rules are enough for real farming or whether timing and scheduling would still be required.
7. **Save the project**: Save the validated program on the Pico as main.py and keep a copy on the computer for future edits.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

Upload and run this code after the individual tests work correctly.

```python
from machine import ADC, Pin
import dht
import time

SOIL_PIN = 26
LIGHT_PIN = 27
DHT_PIN = 15
IRRIGATION_RELAY_PIN = 16
LIGHT_RELAY_PIN = 17
SOIL_DRY_RAW = 48000
SOIL_WET_RAW = 18000
LIGHT_THRESHOLD = 22000
HOT_TEMP = 34

soil = ADC(SOIL_PIN)
light = ADC(LIGHT_PIN)
climate = dht.DHT22(Pin(DHT_PIN))
irrigation_relay = Pin(IRRIGATION_RELAY_PIN, Pin.OUT)
light_relay = Pin(LIGHT_RELAY_PIN, Pin.OUT)


def clamp(value, low, high):
    return max(low, min(high, value))


def dryness_percent(raw):
    span = SOIL_DRY_RAW - SOIL_WET_RAW
    if span <= 0:
        return 0
    pct = int((raw - SOIL_WET_RAW) * 100 / span)
    return clamp(pct, 0, 100)


print('Smart agriculture IoT OS ready')

while True:
    dry_pct = dryness_percent(soil.read_u16())
    light_raw = light.read_u16()
    try:
        climate.measure()
        temperature = climate.temperature()
        humidity = climate.humidity()
    except OSError:
        temperature = 0
        humidity = 0

    irrigation_on = dry_pct >= 65 or (dry_pct >= 55 and temperature >= HOT_TEMP)
    lighting_on = light_raw < LIGHT_THRESHOLD

    irrigation_relay.value(1 if irrigation_on else 0)
    light_relay.value(1 if lighting_on else 0)

    print('Dry:{}% Light:{} Temp:{}C Hum:{}% Irrigation:{} LightAssist:{}'.format(
        dry_pct,
        light_raw,
        temperature,
        humidity,
        irrigation_on,
        lighting_on,
    ))
    time.sleep(3)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Sensor normalization | Converts raw soil and light values into more useful decision inputs. | The supervisor should act on interpretable signals rather than raw counts only. | Calibrate the soil and light thresholds with your own hardware before field-style tests. |
| Dual-output orchestration | Decides irrigation and lighting states in the same control loop. | This is what makes the project more advanced than a single-output demo. | Test one output at a time before trusting both together. |
| Relay output update | Applies the two control decisions to the relay module. | Hardware outputs must match the documented logic for the project to be trustworthy. | Invert a relay output here if your relay module is active low. |

## Expected Result

The serial monitor shows soil, light, temperature, humidity, and the state of both outputs.

Dry conditions turn on irrigation while dark conditions turn on supplemental lighting.

Both outputs can be active at once when the farm conditions require both actions.

### Validation Checks

- **Normal condition test**: use moist soil and bright light and confirm both outputs stay off.
- **Irrigation test**: dry the soil and confirm only the irrigation relay turns on when the threshold is crossed.
- **Lighting test**: reduce the light level and confirm only the lighting relay turns on.
- **Combined condition test**: use dry and dark conditions together and confirm both relays activate.
- **Output response validation**: test the relay module without the real loads first, then attach the pump and lamp after safe switching is confirmed.

### Deployment and Limitations

- This prototype could be used in school, farm, or sanitation demonstrations with safe low-voltage loads.
- Before deployment, the load wiring, enclosure, and power design need careful review.
- Actuator systems need maintenance planning because relay wear and sensor drift affect reliability.

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| Irrigation turns on too often | The dryness threshold is too low | Collect soil baseline data and raise the irrigation threshold carefully. |
| Light assist never turns on | The light threshold is too low or the analog divider is wrong | Print the raw light value and recalibrate the threshold in bright and dark conditions. |
| One relay output works but the other does not | The relay channel or wiring is incorrect | Test each relay channel from a short script and verify the control pin mapping. |
| Climate values stay at zero | The DHT22 read failed | Test the DHT22 separately and shorten the data wire if needed. |
