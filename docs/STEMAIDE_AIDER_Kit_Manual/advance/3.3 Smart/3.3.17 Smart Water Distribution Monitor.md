## Overview

The Pico measures flow and line pressure, then shows whether a small water-distribution branch looks normal, restricted, or leak-prone.

Water systems are harder to troubleshoot when flow and pressure are checked separately instead of being interpreted together.

A Pico 2 W prototype with a pulse flow sensor, analog pressure sensor, and OLED dashboard for water-distribution status.

Multi-sensor interpretation, flow calibration, pressure mapping, and local infrastructure diagnostics.

### Project Story

**Advanced Project**: This advanced project is designed to help learners move beyond basic wiring and coding into complete system thinking. The learner should build the prototype, test each subsystem, validate the data, explain the design decisions, and propose improvements for real-world deployment.

Water distribution problems can appear as weak supply, blocked outlets, or leak-like pressure drops. This prototype helps students compare flow and pressure together so they can discuss what kind of fault may be developing before it becomes a larger service issue.

## Learning Objectives

- Read a pulse-based flow sensor with interrupt logic
- Map a pressure-sensor reading into a meaningful relative pressure value
- Interpret flow and pressure together instead of treating them as isolated measurements

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import time

try:
    from ssd1306 import SSD1306_I2C
    HAS_OLED = True
except ImportError:
    HAS_OLED = False

FLOW_PIN = 14
PRESSURE_PIN = 27
I2C_SDA_PIN = 20
I2C_SCL_PIN = 21
PRESSURE_EMPTY = 5000
PRESSURE_FULL = 54000
SAMPLE_WINDOW_S = 5

pulse_count = 0
flow = Pin(FLOW_PIN, Pin.IN, Pin.PULL_UP)
pressure = ADC(PRESSURE_PIN)
i2c = I2C(0, sda=Pin(I2C_SDA_PIN), scl=Pin(I2C_SCL_PIN), freq=400000)
if HAS_OLED:
    oled = SSD1306_I2C(128, 64, i2c)


def count_pulse(pin):
    global pulse_count
    pulse_count += 1


def clamp(value, low, high):
    return max(low, min(high, value))


def pressure_percent(raw):
    span = PRESSURE_FULL - PRESSURE_EMPTY
    if span <= 0:
        return 0
    pct = int((raw - PRESSURE_EMPTY) * 100 / span)
    return clamp(pct, 0, 100)


def flow_lpm_from_pulses(pulses, seconds):
    if seconds <= 0:
        return 0.0
    return (pulses / seconds) / 7.5


flow.irq(trigger=Pin.IRQ_FALLING, handler=count_pulse)
print('Water distribution monitor ready')

while True:
    pulse_count = 0
    start = time.ticks_ms()
    while time.ticks_diff(time.ticks_ms(), start) < SAMPLE_WINDOW_S * 1000:
        time.sleep_ms(100)

    pulses = pulse_count
    flow_lpm = flow_lpm_from_pulses(pulses, SAMPLE_WINDOW_S)
    pressure_raw = pressure.read_u16()
    pressure_pct = pressure_percent(pressure_raw)

    if pressure_pct >= 65 and flow_lpm < 0.8:
        status = 'BLOCKAGE'
    elif pressure_pct <= 20 and flow_lpm < 0.5:
        status = 'LOW SUPPLY'
    elif pressure_pct <= 25 and flow_lpm > 3.0:
        status = 'LEAK RISK'
    else:
        status = 'NORMAL'

    print('Flow:{:.2f}L/min Pressure:{}% Status:{}'.format(
        flow_lpm, pressure_pct, status))

    if HAS_OLED:
        oled.fill(0)
        oled.text('Water Branch', 0, 0)
        oled.text('Flow:{:.1f}L'.format(flow_lpm), 0, 16)
        oled.text('P:{}%'.format(pressure_pct), 0, 30)
        oled.text(status[:16], 0, 46)
        oled.show()
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Pulse counting | Counts flow-sensor pulses during a fixed sample window | The flow calculation depends on stable counting | Verify the pulse line before changing any diagnostic thresholds |
| pressure_percent | Maps the raw pressure ADC value into a relative percentage | A relative pressure value is easier to interpret than raw ADC counts | Adjust the empty and full references after measuring your real sensor |
| Diagnostic rules | Compares flow and pressure together to identify branch-health states | This is what makes the project more advanced than two isolated readings | Tune the thresholds only after you observe real test conditions |
| OLED dashboard | Displays the current hydraulic state locally | A local dashboard helps students validate the diagnosis in real time | If the display is blank, verify ssd1306.py and the I2C wiring first |

## Expected Result

The OLED and serial monitor show live flow and pressure values together with a branch-health label.

High pressure with weak flow produces a blockage-style warning.

Low pressure with very low flow produces a low-supply warning, while low pressure with strong flow suggests leak risk.

### Validation Checks

- **Normal condition test**: run a moderate stable flow and confirm the monitor reports NORMAL
- **Warning condition test**: reduce outlet flow while pressure remains high and confirm the BLOCKAGE label appears
- **Critical interpretation test**: create a low-pressure condition and confirm the monitor distinguishes LOW SUPPLY from LEAK RISK
- **Sensor reading validation**: compare the flow estimate with a measured container fill and check that pressure changes move the relative percentage
- **Calibration test**: record empty and high-pressure analog values before finalizing the pressure mapping
- **Limitation test**: explain why this prototype cannot replace a certified plumbing or utility instrument

### Deployment and Limitations

- This prototype is useful for classroom water-network demonstrations and maintenance training
- Before deployment, the pressure-sensor scaling, enclosure, and hydraulic safety setup need improvement
- Field use would also need better plumbing integration, more calibration, and stronger water protection for the electronics

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| Flow stays at zero | The pulse output is not reaching GPIO 14 | Print the pulse count during a known flow event and verify the flow-sensor signal path |
| Pressure percentage never changes | The analog pressure path is wrong or the calibration constants are unrealistic | Print the raw ADC value and re-measure the reference points |
| Status is always blockage | The flow threshold is too strict for your test hardware | Collect real baseline values and adjust the blockage threshold |
| OLED is blank | The display library or I2C wiring is wrong | Use GPIO 20 and GPIO 21 and confirm ssd1306.py is on the Pico |
