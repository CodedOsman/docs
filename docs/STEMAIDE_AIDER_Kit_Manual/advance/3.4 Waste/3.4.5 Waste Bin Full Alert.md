## Overview

The Pico measures the distance from the top of a waste bin to the trash surface and raises a local alert when the bin is almost full.

Overflowing bins create hygiene problems, attract pests, and make collection routes less efficient.

A Pico 2 W prototype with an ultrasonic sensor, LED, and buzzer that monitors fill level.

A calibrated full-bin threshold, clear warning states, and reliable local alert behavior.

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules.
- Open the Thonny Shell and confirm that the raw distance values change as you move an object inside the bin before enabling the buzzer.

### Project-Specific Safety Note

Many HC-SR04 ultrasonic modules output 5V on the Echo pin. Raspberry Pi Pico GPIO pins are 3.3V only. Use a voltage divider on the Echo line or use a 3.3V-safe ultrasonic sensor.

Mount the sensor so it cannot be hit directly by waste during testing.

## Circuit Connections

|---------------|-------------|---------------------------------|-------|
| HC-SR04 VCC | VBUS 5V | Physical pin 40 | Power the module from USB 5V |
| HC-SR04 GND | GND | Physical pin 38 | Common ground |
| HC-SR04 Trig | GPIO 14 | GPIO 14 / physical pin 19 | 3.3V trigger output |
| HC-SR04 Echo | Voltage-divider midpoint to GPIO 13 | GPIO 13 / physical pin 17 | Do not connect Echo directly |
| LED anode | 220 ohm resistor to GPIO 15 | GPIO 15 / physical pin 20 | LED turns on for WARNING and FULL |
| LED cathode | GND | Physical pin 18 or 38 | Either ground point is fine |
| Buzzer positive | GPIO 16 | GPIO 16 / physical pin 21 | Active buzzer control |
| Buzzer negative | GND | Physical pin 23 or 38 | Common ground |

---

## Wiring Diagram

```
  VBUS                   -> HC-SR04 VCC
  GND                    -> HC-SR04 GND, LED cathode, buzzer negative
  GPIO 14                -> HC-SR04 Trig
  HC-SR04 Echo           -> resistor divider -> GPIO 13
  GPIO 15                -> 220 ohm resistor -> LED anode
  GPIO 16                -> buzzer positive
```

## Step-by-Step Assembly

1. Connect the HC-SR04 VCC pin to the Pico VBUS pin and connect the sensor GND to Pico GND.
2. Connect HC-SR04 Trig to GPIO 14.
3. Build a voltage divider on the Echo line, then connect the divider midpoint to GPIO 13.
4. Connect the LED anode to GPIO 15 through a 220 ohm resistor and connect the LED cathode to GND.
5. Connect the active buzzer positive pin to GPIO 16 and its negative pin to GND.
6. Mount the sensor so it points downward into the bin and has a clear line of sight to the waste surface.

## Testing Individual Components

Before running the full project, test each subsystem separately. This makes it easier to find wiring, library, or logic problems before full integration.

1. **Hardware setup**: Assemble the Pico, sensor, indicator, and load wiring exactly as shown in the connection table before applying power.
2. **Test the input sensor**: Run a short distance-reading script and compare the output to 5 cm, 10 cm, 20 cm, and 30 cm target distances inside a test container.
3. **Test the output device**: Toggle the LED and buzzer from a simple Pin test before combining them with the sensor logic.
4. **Test the decision logic**: Confirm that the code reports NORMAL, WARNING, and FULL at the chosen threshold distances without jumping randomly between states.
5. **Run the full system**: Place objects in the bin gradually and confirm that the alert transitions match the real fill level.
6. **Validate the prototype**: Repeat the test with different waste shapes or bag positions to identify false triggers and dead zones.
7. **Save the project**: Save the validated program on the Pico as main.py and keep a copy on the computer for future edits.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import Pin
import time

TRIG_PIN = 14
ECHO_PIN = 13
LED_PIN = 15
BUZZER_PIN = 16

FULL_DISTANCE_CM = 10.0
WARNING_DISTANCE_CM = 20.0
TIMEOUT_US = 30000

trig = Pin(TRIG_PIN, Pin.OUT)
echo = Pin(ECHO_PIN, Pin.IN)
led = Pin(LED_PIN, Pin.OUT)
buzzer = Pin(BUZZER_PIN, Pin.OUT)


def measure_distance_cm():
    trig.low()
    time.sleep_us(2)
    trig.high()
    time.sleep_us(10)
    trig.low()

    start = time.ticks_us()
    while echo.value() == 0:
        if time.ticks_diff(time.ticks_us(), start) > TIMEOUT_US:
            return None

    pulse_start = time.ticks_us()
    while echo.value() == 1:
        if time.ticks_diff(time.ticks_us(), pulse_start) > TIMEOUT_US:
            return None

    pulse_end = time.ticks_us()
    duration = time.ticks_diff(pulse_end, pulse_start)
    return (duration * 0.0343) / 2


def classify(distance_cm):
    if distance_cm is None:
        return "SENSOR ERROR"
    if distance_cm <= FULL_DISTANCE_CM:
        return "FULL"
    if distance_cm <= WARNING_DISTANCE_CM:
        return "WARNING"
    return "NORMAL"


def apply_outputs(status):
    led.value(1 if status in ("WARNING", "FULL") else 0)
    buzzer.value(1 if status == "FULL" else 0)


print("Waste bin full alert ready")

while True:
    distance = measure_distance_cm()
    status = classify(distance)
    apply_outputs(status)

    if distance is None:
        print("Distance: timeout | Status: SENSOR ERROR")
    else:
        print("Distance: {:.1f} cm | Status: {}".format(distance, status))

    time.sleep(1)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Pin and threshold setup | Defines sensor and alert pins plus the warning and full distances | The project only makes sense if the wiring and distance thresholds match the real bin geometry | Adjust FULL_DISTANCE_CM and WARNING_DISTANCE_CM after measuring your own bin |
| measure_distance_cm | Sends the ultrasonic trigger pulse and times the Echo pulse length | This converts a travel-time signal into a usable fill-level measurement | Increase TIMEOUT_US if your sensor is mounted high above the waste surface |
| classify | Turns the raw distance into NORMAL, WARNING, FULL, or SENSOR ERROR | Status labels are easier to test and explain than raw numbers alone | Change the threshold values, not the status names, during calibration |
| apply_outputs and main loop | Updates the LED and buzzer and prints the current reading once per second | This makes subsystem behavior visible during classroom testing and validation | Change the sample interval or buzzer behavior if the prototype feels too noisy |

## Expected Result

The serial monitor prints the measured bin distance and the status label once per second.

The LED turns on when the bin reaches the WARNING or FULL range.

The buzzer turns on only when the ultrasonic distance is below the full-bin threshold.

### Validation Checks

- **Normal condition test**: hold a flat object 30 cm away and confirm a NORMAL result
- **Warning condition test**: hold the object at about 20 cm and confirm WARNING with LED on and buzzer off
- **Critical condition test**: hold the object at about 5 cm to 10 cm and confirm FULL with LED and buzzer on
- **Sensor reading validation**: test at 5 cm, 10 cm, 20 cm, and 30 cm and confirm the reported value is reasonably close
- **False trigger test**: tilt the object, add irregular waste shapes, and note when the beam misses the target
- **Limitation test**: place soft fabric or angled plastic in the bin and observe whether reflections become unreliable

### Deployment and Limitations

- This prototype could be used in classrooms, markets, or school compounds as a local overflow warning device
- Before real deployment, the sensor needs mechanical protection, weather resistance, and a maintenance plan
- Power design, enclosure design, and cleaning procedures are essential because dirt and moisture reduce sensor reliability

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| Distance always times out | Echo wiring or divider is incorrect | Rebuild the Echo voltage divider and confirm the sensor ground is shared with the Pico |
| Reading changes wildly | Sensor is pointed at a curved or moving surface | Aim the ultrasonic sensor straight down and average several readings during calibration |
| Buzzer never turns on | Full threshold is too small or buzzer polarity is reversed | Lower the full threshold temporarily and confirm the buzzer wiring with a direct Pin test |
| LED is always on | Warning threshold is too large for the bin depth | Measure the empty-bin distance and choose thresholds that fit your actual container |
