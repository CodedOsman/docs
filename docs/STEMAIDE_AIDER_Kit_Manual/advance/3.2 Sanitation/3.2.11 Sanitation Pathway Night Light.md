## Overview

The Pico reads ambient light and adjusts an LED pathway light so the area becomes brighter as it gets darker.

Night pathways near washrooms or sanitation blocks need enough light for safety without wasting energy all day.

A Pico 2 W prototype with an LDR, PWM-controlled LED, and analog light-based control logic.

Sensor calibration, proportional output control, and realistic discussion of power-saving lighting design.

## Testing Individual Components

Before running the full project, test each subsystem separately. This makes it easier to find wiring, library, or logic problems before full integration.

1. **Hardware setup**: Assemble the Pico, sensor, indicator, and load wiring exactly as shown in the connection table before applying power.
2. **Test the input sensor**: Print the raw light reading in bright and dark conditions so you know the real sensor range in your room.
3. **Test the output device**: Run a short PWM LED test to confirm brightness changes before linking it to the LDR.
4. **Test the decision logic**: Check that the LED brightens smoothly as the room becomes darker instead of flickering around the threshold.
5. **Run the full system**: Run the full system and move between bright and dark conditions to validate the proportional response.
6. **Validate the prototype**: Test the effect of sensor shadows, wall reflections, and enclosure shape on the control behavior.
7. **Save the project**: Save the validated program on the Pico as main.py and keep a copy on the computer for future edits.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, PWM, Pin
import time

LDR_PIN = 26
LED_PIN = 15
DARK_THRESHOLD = 18000
BRIGHT_THRESHOLD = 52000

ldr = ADC(LDR_PIN)
led = PWM(Pin(LED_PIN))
led.freq(1000)


def read_light(samples=8):
    total = 0
    for _ in range(samples):
        total += ldr.read_u16()
        time.sleep_ms(10)
    return total // samples


def brightness_for(light_level):
    if light_level >= BRIGHT_THRESHOLD:
        return 0
    if light_level <= DARK_THRESHOLD:
        return 65535

    scale = (BRIGHT_THRESHOLD - light_level) / (BRIGHT_THRESHOLD - DARK_THRESHOLD)
    return int(scale * 65535)


print("Pathway night light ready")

while True:
    light_level = read_light()
    brightness = brightness_for(light_level)
    led.duty_u16(brightness)
    print("Light: {} | Brightness: {}".format(light_level, brightness))
    time.sleep(1)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|------------------------------|
| Sensor and PWM setup | Initializes the LDR analog input and the PWM-driven LED output | PWM allows gradual brightness control instead of a harsh on or off response | Check the LED resistor path before changing the PWM settings |
| read_light | Averages several ADC samples from the LDR divider | Stable sensor input improves the feel of the light-control response | Increase the sample count if the reading flickers too much |
| brightness_for | Maps the light reading into a 0 to 65535 PWM range | This is the core proportional-control idea in the project | Adjust DARK_THRESHOLD and BRIGHT_THRESHOLD after measuring your own room |
| Main loop | Updates the LED brightness and prints the current values | Students can compare the analog reading directly with the lighting response | Change the loop rate only after the calibration is stable |

### Validation Checks

- **Normal condition test**: confirm the system stays in its safe or idle state under baseline conditions
- **Warning condition test**: move the input close to the chosen threshold and confirm the transition is sensible
- **Critical condition test**: trigger the strongest alert or control state and confirm the output response is correct
- **Calibration test**: compare the sensor or timing against a known reference or repeated trial
- **Limitation test**: deliberately create an awkward or noisy condition and note how the prototype behaves

### Deployment and Limitations

- This prototype is useful for local indicator boards and classroom control panels
- Before deployment, labeling, enclosure design, and user training need improvement
- Status tools only work well when users trust the meaning of each state and keep the system updated

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| LED is always off | Thresholds are too high or the divider is wired incorrectly | Print the raw light value and rebuild the LDR divider if the value never changes |
| LED is always bright | Dark threshold is too high or the LDR is always shaded | Lower DARK_THRESHOLD and confirm the LDR is exposed to room light |
| Brightness jumps instead of smoothing | Threshold range is too small | Increase the gap between DARK_THRESHOLD and BRIGHT_THRESHOLD |
| No ADC change occurs | The LDR or resistor is not actually in a divider | Reconnect the LDR and 10k resistor so the ADC sees the midpoint |
