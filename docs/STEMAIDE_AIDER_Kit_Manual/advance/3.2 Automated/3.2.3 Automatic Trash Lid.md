## Overview

The Pico opens a servo-driven trash lid when an IR proximity sensor detects a hand near the bin opening.

Touch-free bins reduce contamination and make waste disposal easier in shared spaces.

A Pico 2 W prototype with an IR proximity sensor, a servo motor for the lid, and a timed open-close sequence.

Proximity-based actuation, servo timing, and safe external-power use for small motors.

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules.
- Calibrate the open and closed servo angles with the lid disconnected first so the motor does not strain the hinge.
- Many IR proximity modules are active-low, so confirm the sensor output before trusting the full lid logic.

### Project-Specific Safety Note

Connect the servo ground to Pico ground, but power the servo from a separate regulated supply because startup current can reset the Pico. Keep fingers clear of the lid hinge while testing.

## Step-by-Step Assembly

1. Connect the IR sensor to Pico 3.3V, GND, and GPIO 14.
2. Connect the servo signal wire to GPIO 15.
3. Power the servo from an external 5V supply and connect the servo ground to Pico ground.
4. Test the servo angles without the lid attached so you do not jam the mechanism during calibration.
5. Mount the sensor so a hand near the bin mouth is detected reliably but the surrounding room is ignored.
6. Attach the lid only after the servo open and closed positions are safe.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE, copy and paste this code into a new file or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import Pin, PWM
import time

IR_PIN = 14
SERVO_PIN = 15
OPEN_DUTY = 5200
CLOSED_DUTY = 2500
HOLD_OPEN_MS = 2500
CYCLE_GAP_MS = 3000

ir = Pin(IR_PIN, Pin.IN, Pin.PULL_UP)
servo = PWM(Pin(SERVO_PIN))
servo.freq(50)
last_state = 1
last_cycle_ms = -CYCLE_GAP_MS


def set_servo(duty):
    servo.duty_u16(duty)
    time.sleep_ms(500)
    servo.duty_u16(0)


set_servo(CLOSED_DUTY)
print("Automatic trash lid ready")

while True:
    now = time.ticks_ms()
    current = ir.value()
    new_detection = last_state == 1 and current == 0

    if new_detection and time.ticks_diff(now, last_cycle_ms) >= CYCLE_GAP_MS:
        print("State: OPENING")
        set_servo(OPEN_DUTY)
        print("State: HOLDING")
        time.sleep_ms(HOLD_OPEN_MS)
        print("State: CLOSING")
        set_servo(CLOSED_DUTY)
        last_cycle_ms = time.ticks_ms()
        print("State: IDLE")

    last_state = current
    time.sleep_ms(50)
```

## How the Code Works

| Code Section | What It Does | Why It Matters | What to Modify During Testing |
|--------------|--------------|----------------|-------------------------------|
| Servo calibration constants | Defines the duty values for open and closed positions | Servo mechanics vary, so the lid must be calibrated to the actual hardware | Adjust `OPEN_DUTY` and `CLOSED_DUTY` with the lid disconnected first |
| `set_servo()` | Moves the servo and then releases PWM holding torque | This reduces continuous buzzing on small servos during classroom tests | Increase the move delay only if the servo needs more time to reach position |
| Edge detection | Starts a cycle only on a new proximity event | This prevents the lid from retriggering constantly while a hand stays in view | If false triggers happen, improve sensor placement before changing the code |
| Open-hold-close sequence | Runs the full lid motion and then enforces a short gap before the next cycle | This keeps the user interaction predictable and reduces motor stress | Adjust `HOLD_OPEN_MS` and `CYCLE_GAP_MS` after observing real use |

## Expected Result

The serial monitor reports the current state clearly. When a hand moves near the IR proximity sensor, the servo opens the lid, holds it open briefly, then closes it after the delay. The system ignores repeated triggering during the cycle gap.

### Validation Checks

- **Normal condition test**: Confirm the lid stays closed when no hand is near the sensor.
- **Detection test**: Move a hand into the sensor range and confirm the lid opens once.
- **Cycle test**: Keep a hand near the sensor and confirm the lid does not jitter open and closed repeatedly.
- **Calibration test**: Compare the open and closed angles with the real lid mechanism.
- **Limitation test**: Try awkward hand positions and note where the sensor fails or triggers too easily.

### Deployment and Limitations

- This prototype could be used in school or sanitation demonstrations with safe low-voltage parts.
- Before deployment, the lid mechanism, enclosure, wiring, and power supply need careful review.
- Actuator systems need maintenance planning because hinges, servo gears, and sensor alignment can drift over time.

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|----------------|----------|
| Servo twitches but does not move the lid | Angles or power supply are wrong | Calibrate the servo with the lid disconnected and confirm the external 5V supply is stable |
| Lid opens repeatedly | Sensor sees a constant detection or no cycle gap is applied | Narrow the sensor view and keep the cycle gap in place |
| Pico resets when the servo moves | Servo is drawing too much current from the wrong supply | Move the servo to a separate regulated 5V supply and keep grounds common |
| Lid closes too soon | Hold time is too short for the user | Increase `HOLD_OPEN_MS` after observing the disposal action |
