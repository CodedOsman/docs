## Overview

This project uses local humidity and temperature trends to create a simple rule-based rain alert heuristic.

Students will collect several BME280 readings over time, compare older and newer samples, and generate CLEAR, WATCH, or RAIN LIKELY states.

The final system should explain why it thinks rain is becoming more likely instead of simply flashing an unexplained alert.

### Project Story

The real-world use case is a field demo where students want an early warning to cover tools, pause irrigation, or protect crops before rain actually starts.

---

## Learning Objectives

- Collect a short environmental history instead of judging weather from one reading
- Use rule-based trend logic for an early warning heuristic
- Separate WATCH and RAIN LIKELY states instead of using only one alarm threshold
- Explain why a simple forecast model is different from a true weather service
- Use LEDs and a buzzer to show graded alert levels

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 Sensor Module
Place the BME280 module where it can sense the surrounding air. Identify VCC, GND, and DATA before wiring.

### Step 3: Connect the BME280 Sensor Module
Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 2** and BME280 SCL to **GPIO 3**. If you are using a bare BME280 sensor, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 4: Place and Connect the Alert LED
Place the alert LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the alert LED long leg through a 220Ω resistor to **GPIO 16**. Connect the alert LED short leg to **GND**.

### Step 5: Place and Connect the Buzzer
Place the buzzer on the breadboard and identify its positive (+) and negative (-) pins. Connect the buzzer positive pin to **GPIO 17**. Connect the buzzer negative pin to **GND**.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] BME280 SDA connects to GPIO 2 and SCL connects to GPIO 3
- [x] Alert LED long leg connects through a 220Ω resistor to GPIO 16
- [x] Alert LED short leg connects to GND
- [x] Buzzer positive pin connects to GPIO 17
- [x] Buzzer negative pin connects to GND
- [x] No loose jumper wires

> **Intermediate Note**
> The BME280 needs a short delay between readings. This project is a learning forecast demo, so treat the rain-likely state as a pattern indicator, not a real weather guarantee.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import bme280
import time

i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


alert_led = Pin(16, Pin.OUT)
buzzer = Pin(17, Pin.OUT)

humidity_history = []
temperature_history = []
history_size = 8

alert_repeat_s = 30
last_alert = time.time() - alert_repeat_s


def alert_burst():
    for _ in range(2):
        alert_led.on()
        buzzer.on()
        time.sleep(0.15)
        alert_led.off()
        buzzer.off()
        time.sleep(0.15)


def forecast_state(humidity_values, temperature_values):
    if len(humidity_values) < 6 or len(temperature_values) < 6:
        return "COLLECTING", []

    recent_h = sum(humidity_values[-3:]) / 3
    older_h = sum(humidity_values[-6:-3]) / 3
    recent_t = sum(temperature_values[-3:]) / 3
    older_t = sum(temperature_values[-6:-3]) / 3

    reasons = []
    if recent_h - older_h >= 6:
        reasons.append("humidity rising quickly")
    if older_t - recent_t >= 1.5:
        reasons.append("temperature falling")

    if len(reasons) >= 2:
        return "RAIN LIKELY", reasons
    if len(reasons) == 1:
        return "WATCH", reasons
    return "CLEAR", ["no strong rain pattern"]


print("Rain prediction alert system ready")
time.sleep(2)

while True:
    try:
        temp_c, humidity, pressure_hpa = read_bme280()
    except Exception as error:
        print("Sensor error:", error)
        time.sleep(2)
        continue

    humidity_history.append(humidity)
    temperature_history.append(temp_c)

    if len(humidity_history) > history_size:
        humidity_history.pop(0)
        temperature_history.pop(0)

    state, reasons = forecast_state(humidity_history, temperature_history)

    print("Humidity:{}% Temp:{}C Forecast:{} Reasons:{}".format(
        humidity,
        temp_c,
        state,
        ", ".join(reasons),
    ))

    if state == "RAIN LIKELY":
        alert_led.on()
        if time.time() - last_alert >= alert_repeat_s:
            alert_burst()
            last_alert = time.time()
    elif state == "WATCH":
        alert_led.on()
    else:
        alert_led.off()
        buzzer.off()

    time.sleep(10)
```

## Troubleshooting

| Problem | 1 | Solution |
|---------|---|----------|
| The state never leaves COLLECTING | 1 | Wait for more readings or reduce the number of samples required |
| RAIN LIKELY appears too easily | 1 | Increase the required humidity rise or temperature drop values |
| The LED is always on | The state is stuck in WATCH or RAIN LIKELY | Print the recent trend values and tune the thresholds |
| The buzzer never sounds | 1 | Check the reasons list and adjust the stronger alert rule if appropriate |
