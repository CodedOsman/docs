## Overview

This project monitors movement in a livestock area and summarizes activity over a timed observation window.

Students will build a PIR-based event logger that counts motion and classifies the space as QUIET, LOW ACTIVITY, MODERATE ACTIVITY, or HIGH ACTIVITY.

The final system should flash an LED when movement is logged, print a report at the end of each window, and prevent one continuous motion event from being counted too many times.

### Project Story

The real-world use case is a pen entrance, feeding lane, or animal shelter where a simple motion summary helps students observe behavior patterns.

---

## Learning Objectives

- Use a PIR sensor for simple activity monitoring
- Count events over a timed observation window
- Convert event totals into a meaningful activity classification
- Add debounce timing so one movement does not become many counts
- Think about how sensor placement changes behavior observations

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the PIR Motion Sensor

Place the PIR motion sensor so it can face the livestock test area without moving.
Check the printed pin labels and identify VCC, GND, and OUT before wiring.

### Step 3: Connect the PIR Sensor

Connect PIR VCC to the supply voltage required by the module label.
Connect PIR GND to GND.
Connect PIR OUT to GPIO 4.
If the PIR is powered from 5V, confirm that OUT is still safe for 3.3V Pico GPIO.

### Step 4: Place and Connect the Activity LED

Place the activity LED on the breadboard.
Identify the long leg as the anode (+) and the short leg as the cathode (-).
Connect the activity LED long leg through a 220Ω resistor to GPIO 16.
Connect the activity LED short leg to GND.

## Full Project Code

```python
from machine import Pin
import time

pir = Pin(4, Pin.IN)
activity_led = Pin(16, Pin.OUT)
motion_count = 0
window_s = 30
window_start = time.time()
last_motion = 0


def activity_state(count):
    if count >= 6:
        return "HIGH ACTIVITY"
    if count >= 3:
        return "MODERATE ACTIVITY"
    if count >= 1:
        return "LOW ACTIVITY"
    return "QUIET"


print("Livestock area motion monitor ready")
while True:
    now = time.time()
    motion = pir.value() == 1
    if motion and (now - last_motion) > 1:
        motion_count += 1
        last_motion = now
        activity_led.on()
        print("Motion event logged")
    else:
        activity_led.off()

    if (now - window_start) >= window_s:
        state = activity_state(motion_count)
        print("Window summary: {} events -> {}".format(motion_count, state))
        motion_count = 0
        window_start = now

    time.sleep(0.2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|-------------|----------------|
| `motion_count` | Stores how many events happened in the current window | A single summary is more useful than isolated motion blips |
| `last_motion` timing | Stops continuous movement from being counted too many times | Simple debounce improves activity data quality |
| `activity_state()` | Converts the count into a behavior label | Labels are easier to interpret than raw numbers alone |
| `window_start` and `window_s` | Define the reporting interval | Students can compare short and long observation windows |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The count is always zero | The PIR is not warmed up or is aimed badly | Wait for warm-up and aim the sensor at the movement path. |
| The count rises too quickly | The PIR stays high for long motion events | Increase the event gap or move the sensor farther from the movement path. |
| The LED never turns on | The LED wiring is wrong or no motion is logged | Test the LED separately and confirm the PIR output changes. |
| The window summary seems meaningless | The observation window is too short or too long | Adjust `window_s` to match the behavior you want to study. |
