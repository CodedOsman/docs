## Overview

This project manages several irrigation zones independently while still limiting the system to one active watering zone at a time.

Students will build a three-zone controller, each with its own calibration and thresholds, plus a master pause button for maintenance or emergency shutdown.

The final system should read all three zones, water only the zones that are dry enough, and avoid running more than one zone simultaneously.

### Project Story

The real-world use case is a small greenhouse or farm layout where different beds dry at different rates but the water source or power budget cannot support every pump at once.

---

## Learning Objectives

- Manage several irrigation zones with one controller
- Store per-zone calibration and threshold values in structured data
- Use sequencing so only one pump runs at a time
- Add a master pause input for safety and maintenance
- Understand the external ADC limit of the Pico and plan within it honestly

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules
- This manual implements three real analog zones because the Pico provides three external ADC inputs directly. Expanding to a fourth analog zone would need extra hardware such as an analog multiplexer

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins.

Use an external power supply for pumps, motors, relay modules, and fans when required.

If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly.

Check whether your relay module is active-low or active-high before testing the final load.

Keep electronics away from water and dry your hands before touching the circuit.

Do not let exposed jumper connections touch water.

Do not let a pump run dry.

Use safe test loads before connecting three real pumps.

Run only one zone at a time during this classroom build to reduce supply and plumbing problems.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Label the Three Irrigation Zones
Label the soil sensors and relay channels as Zone A, Zone B, and Zone C before wiring. Identify VCC, GND, and AOUT / AO / Signal on each soil sensor.

### Step 3: Connect the Soil Sensor Power Rails
Connect each soil sensor VCC pin to **3.3V**. Connect each soil sensor GND pin to **GND**.

### Step 4: Connect Zone A Soil Sensor
Connect Zone A AOUT, AO, or Signal to **GPIO 26**.

### Step 5: Connect Zone B Soil Sensor
Connect Zone B AOUT, AO, or Signal to **GPIO 27**.

### Step 6: Connect Zone C Soil Sensor
Connect Zone C AOUT, AO, or Signal to **GPIO 28**.

### Step 7: Place the Relay Hardware
Place the relay hardware where all three control inputs and load terminals are easy to reach. Identify VCC, GND, IN, COM, NO, and NC for each relay channel used.

### Step 8: Connect the Three Relay Inputs
Connect Zone A relay IN to **GPIO 15**. Connect Zone B relay IN to **GPIO 16**. Connect Zone C relay IN to **GPIO 17**.

### Step 9: Power the Relay Hardware
Power the relay module from the supply required by its label. Connect relay GND to Pico GND and to the external pump or load supply GND where shared grounding is required.

### Step 10: Place the Master Pause Button
Place the master pause push button across the breadboard center gap. Connect one side of the master pause button to **GPIO 18**. Connect the opposite side of the button to **GND**.

### Step 11: Wire Each Zone Load
For each zone, connect the external supply positive wire to that zone relay **COM**. Connect that zone relay **NO** to the pump or safe test load positive lead. Connect the pump or load negative lead to the external supply negative wire.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

RELAY_ON = 0
RELAY_OFF = 1
PUMP_RUN_S = 4
MIN_REST_S = 30

master_button = Pin(18, Pin.IN, Pin.PULL_UP)
soil = ADC(26)
relay = Pin(15, Pin.OUT)
master_enabled = True
last_button = time.time()
last_cycle = time.time() - MIN_REST_S
ready = True

SOIL_DRY = 52000
SOIL_WET = 22000
DRY_THRESHOLD = 35
RECOVERY_THRESHOLD = 45

relay.value(RELAY_OFF)


def clamp(value, low, high):
    return max(low, min(high, value))


def moisture_percent():
    raw = soil.read_u16()
    span = SOIL_DRY - SOIL_WET
    if span == 0:
        return 0
    return clamp(int(((SOIL_DRY - raw) * 100) / span), 0, 100)


def water_zone():
    global last_cycle, ready
    relay.value(RELAY_ON)
    print("Zone output ON")
    time.sleep(PUMP_RUN_S)
    relay.value(RELAY_OFF)
    last_cycle = time.time()
    ready = False
    print("Zone output OFF")


print("Single-zone irrigation system ready")

while True:
    now = time.time()

    if master_button.value() == 0 and (now - last_button) > 0.3:
        master_enabled = not master_enabled
        last_button = now
        print("Master enabled:", master_enabled)
        while master_button.value() == 0:
            time.sleep(0.05)

    if not master_enabled:
        relay.value(RELAY_OFF)
        print("System paused by master button")
        time.sleep(2)
        continue

    moisture = moisture_percent()

    if moisture >= RECOVERY_THRESHOLD:
        ready = True

    if ready and moisture <= DRY_THRESHOLD and (time.time() - last_cycle) >= MIN_REST_S:
        water_zone()
    else:
        print("Moisture:{}% Ready:{}".format(moisture, ready))

    time.sleep(5)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| `zones` list | Stores sensor, relay, thresholds, and state for each zone | Structured data keeps the multi-zone design manageable |
| `master_enabled` | Pauses or resumes all irrigation zones | A multi-zone system needs a safe global control |
| `watered_this_loop` | Allows only one active watering cycle per pass | Sequencing prevents overload on power and plumbing |
| Per-zone ready and last values | Track re-arming and rest timing independently | Each zone should behave according to its own conditions |

## Expected Result

Each zone should report its own moisture percentage. When a zone is dry enough, the controller should water that zone and then stop before moving on. If several zones are dry, only one should run at a time. Pressing the master pause button should stop all zones from running until resumed.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The wrong zone waters | The sensor or relay labels are mixed up | Label the hardware and compare each zone one at a time with the code |
| Two outputs run together | The sequencing flag was bypassed or another relay is wired incorrectly | Check `watered_this_loop` logic and the relay channels carefully |
| One zone never waters | That zone's threshold is too low or its ADC input is wrong | Print the live moisture for that zone and recalibrate it |
| Master pause does nothing | The button wiring or debounce timing is wrong | Test GPIO 18 separately and confirm the Shell message changes when pressed |
