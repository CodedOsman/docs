## Overview

This project builds a public restroom usage monitor that counts visits, tracks hourly usage patterns, and serves a local dashboard over Wi-Fi.

Students will combine a digital ultrasonic sensor, a cleaning-reset button, Wi-Fi time synchronisation, hourly usage storage, and a simple web page into one monitoring system.

The final system should connect to Wi-Fi, count restroom visits once per entry event, track the busiest hour, show when cleaning is recommended, and display the results in both the serial monitor and a browser.

### Project Story

The real-world use case is a shared public facility where staff need to know when usage is highest and when cleaning service is likely needed.

- Count visits with lockout logic to avoid duplicate events
- Store hourly usage patterns across the day

- Think about how usage data can guide maintenance decisions

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Ultrasonic Distance Sensor
Place the entry or ultrasonic sensor where one restroom-use event creates one clean trigger instead of repeated rapid triggers. Identify VCC, GND, and output before wiring.

### Step 3: Connect the Entry Sensor
Connect sensor VCC to the safe low-voltage supply required by the module label. Connect sensor GND to GND. Connect sensor output to GPIO 4.

### Step 4: Place the Cleaning Reset Button
Place the cleaning reset push button across the breadboard center gap. Connect one side of the button to GPIO 5. Connect the opposite side of the button to GND.

### Step 5: Place and Connect the Service LED
Place the service LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the service LED long leg through a 220Ω resistor to GPIO 16. Connect the service LED short leg to GND.

### Step 6: Prepare for Wi-Fi Dashboard Testing
The built-in Pico LED is used for Wi-Fi connection state, so no extra Wi-Fi LED is needed. Prepare your Wi-Fi credentials in your local code copy before running the dashboard. Open the IP address shown in the Thonny Shell from a browser on the same Wi-Fi network.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
import network
import ntptime
import socket
import time
from machine import Pin, time_pulse_us

WIFI_SSID = 'YOUR_WIFI_NAME'
WIFI_PASSWORD = 'YOUR_WIFI_PASSWORD'
UTC_OFFSET_HOURS = 0

trig = Pin(4, Pin.OUT)
echo = Pin(6, Pin.IN)
reset_button = Pin(5, Pin.IN, Pin.PULL_UP)
service_led = Pin(16, Pin.OUT)
wifi_led = Pin('LED', Pin.OUT)

PRESENCE_CM = 80
VISIT_LOCKOUT_SECONDS = 8
SERVICE_THRESHOLD = 30
TIME_SYNC_INTERVAL = 3600
DEBOUNCE_MS = 250

hourly_usage = [0] * 24
total_visits = 0
visits_since_clean = 0
last_visit_time = 0
last_sync = 0
last_button_ms = 0
previous_present = False


def distance_cm():
    trig.value(0)
    time.sleep_us(2)
    trig.value(1)
    time.sleep_us(10)
    trig.value(0)
    pulse = time_pulse_us(echo, 1, 30000)
    if pulse < 0:
        return 999
    return (pulse / 2) / 29.1


def person_present():
    return distance_cm() <= PRESENCE_CM


def button_pressed():
    global last_button_ms
    now_ms = time.ticks_ms()
    if reset_button.value() == 0 and time.ticks_diff(now_ms, last_button_ms) > DEBOUNCE_MS:
        while reset_button.value() == 0:
            time.sleep(0.02)
        last_button_ms = now_ms
        return True
    return False


def connect_wifi():
    wlan = network.WLAN(network.STA_IF)
    wlan.active(True)
    if not wlan.isconnected():
        print('Connecting to Wi-Fi...')
        wlan.connect(WIFI_SSID, WIFI_PASSWORD)
        start = time.time()
        while not wlan.isconnected():
            wifi_led.value(1 - wifi_led.value())
            if time.time() - start > 20:
                raise RuntimeError('Wi-Fi connection timeout')
            time.sleep(0.5)
    wifi_led.value(1)
    print('Wi-Fi connected:', wlan.ifconfig()[0])
    return wlan


def sync_time():
    global last_sync
    ntptime.settime()
    last_sync = time.time()
    print('Clock synchronised from NTP.')


def local_now():
    return time.localtime(time.time() + (UTC_OFFSET_HOURS * 3600))


def record_visit():
    global total_visits, visits_since_clean, last_visit_time
    now = time.time()
    if now - last_visit_time < VISIT_LOCKOUT_SECONDS:
        return
    hour = local_now()[3]
    hourly_usage[hour] += 1
    total_visits += 1
    visits_since_clean += 1
    last_visit_time = now
    print('Visit counted. Total:', total_visits)


print('=== Public Restroom Usage Monitor ===')
wlan = connect_wifi()
try:
    sync_time()
except Exception as error:
    print('Time sync failed:', error)

while True:
    present = person_present()
    if present and not previous_present:
        record_visit()
    previous_present = present

    if button_pressed():
        visits_since_clean = 0
        print('Cleaning counter reset.')

    service_led.value(1 if visits_since_clean >= SERVICE_THRESHOLD else 0)
    print('Distance:{:.1f}cm Total:{} SinceClean:{}'.format(
        distance_cm(), total_visits, visits_since_clean
    ))
    time.sleep(1)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| Wi-Fi and NTP setup | Connect the Pico 2 W to the network and synchronise time | Hourly usage analysis needs a reliable clock |
| Visit lockout logic | Counts one visit only once within the lockout period | This avoids many duplicate counts from one entry event |
| Hourly usage array | Stores visit totals for each hour of the day | Students can identify patterns such as peak-use times |
| Dashboard page | Serves the totals and service status to a local browser | This turns the monitor into a usable maintenance dashboard |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The page never loads | The Pico did not connect to Wi-Fi or the browser is on a different network | Check the Shell for the IP address and confirm both devices are on the same 2.4 GHz network |
| The hourly counts go to the wrong slot | Time sync failed or the timezone offset is wrong | Check the NTP sync messages and adjust UTC_OFFSET_HOURS |
| One visit becomes many counts | The lockout is too short or the sensor is bouncing | Increase VISIT_LOCKOUT_SECONDS or improve sensor placement |
| The service LED never turns on | The service threshold is too high or visit events are not being counted | Lower SERVICE_THRESHOLD for testing and verify the input event logic |
