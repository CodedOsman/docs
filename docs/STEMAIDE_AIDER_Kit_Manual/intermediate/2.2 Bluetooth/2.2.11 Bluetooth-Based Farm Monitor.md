## Overview

This project creates a wireless farm monitor that streams sensor data to a phone over BLE without requiring Wi-Fi.

The real-world use case is a field edge, greenhouse corner, or temporary monitoring point where local wireless access is enough and internet access may not be available.

Students will combine soil moisture, temperature, humidity, and BLE UART communication into one portable monitoring node.

The final system should advertise over BLE, stream live readings to a phone, and respond to basic monitoring commands.

### Project Story

This project creates a wireless farm monitor that streams sensor data to a phone over BLE without requiring Wi-Fi.

## Before You Begin

Before starting this project, make sure you have completed the foundational sections at the beginning of the manual:

- Software Installation and Setup
- Safety Guidelines

### Project-Specific Setup Notes

- These Bluetooth projects use the built-in `bluetooth` module plus two helper files: `ble_uart.py` and `ble_advertising.py`.
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`.
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`.
- Copy `bme280.py` to the Pico before running this project.
- In Thonny Shell, run `import os; print(os.listdir())` and confirm the BLE helper files and `bme280.py` are visible on the Pico.
- Install a BLE app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone.
- Scan for the device name `AIDER_FarmMonitor` after the Pico starts advertising.
- Do not pair from the normal phone Bluetooth settings menu. Open the BLE app, scan, connect to the UART-style service, then send text commands from inside the app.
- If Bluetooth does not work, restart the Pico and rescan from the BLE app before changing the wiring.

### Project-Specific Safety Note

- Do not try to pair this project from the normal phone Bluetooth settings page. Use a BLE app instead.
- Keep the wet soil sample away from the Pico board and USB cable.
- Do not pour water onto the Pico, breadboard, USB cable, or jumper wires. Add water to the soil sample away from the electronics.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap.

Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Soil Moisture Sensor

Place the soil moisture probe so the sensing end can enter the soil sample.

Identify VCC, GND, and AOUT / AO / Signal on the sensor module.

### Step 3: Connect the Soil Sensor

Connect soil sensor VCC to 3.3V.

Connect soil sensor GND to GND.

Connect soil sensor AOUT, AO, or Signal to GPIO 26 (ADC0).

### Step 4: Place the BME280 Sensor Module

Place the BME280 module where it can sense the farm monitor air conditions.

Identify VCC, GND, SDA, and SCL before wiring.

### Step 5: Connect the BME280 Sensor Module

Connect BME280 VCC to 3.3V.

Connect BME280 GND to GND.

Connect BME280 SDA to GPIO 2 and BME280 SCL to GPIO 3.

If you are using a bare BME280 sensor, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 6: Load the BLE Helper Files

Upload `ble_uart.py`, `ble_advertising.py`, and `bme280.py` to the Pico root folder before running the main code.

In the Thonny Shell, use `os.listdir()` to confirm the helper files are on the Pico.

## Full Project Code

```python
from machine import ADC, I2C, Pin
import bluetooth
import bme280
import time
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_FarmMonitor"
REPORT_INTERVAL_S = 5

soil = ADC(26)
i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)

soil_dry = 52000
soil_wet = 22000
command_queue = []
stream_enabled = True
last_report = 0

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)


def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


def soil_percent():
    raw = soil.read_u16()
    span = soil_dry - soil_wet
    if span == 0:
        return 0
    percent = int(((soil_dry - raw) * 100) / span)
    return clamp(percent, 0, 100)


def climate_values():
    temp_c, humidity_pct, pressure_hpa = read_bme280()
    return temp_c, humidity_pct


def send_line(message):
    print(message)
    uart.write((message + "\n").encode())


def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)


def send_sensor_report(prefix=""):
    moisture = soil_percent()
    temp_c, humidity_pct = climate_values()
    send_line("{}Soil:{}% Temp:{}C Hum:{}%".format(prefix, moisture, temp_c, humidity_pct))


uart.on_rx(on_rx)
send_line("Bluetooth-based farm monitor ready")

while True:
    now = time.time()

    if stream_enabled and (now - last_report) >= REPORT_INTERVAL_S:
        try:
            send_sensor_report("")
        except Exception as error:
            send_line("Sensor error: {}".format(error))
        last_report = now

    if command_queue:
        command = command_queue.pop(0)
        if command == "status":
            try:
                send_sensor_report("Status ")
            except Exception as error:
                send_line("Sensor error: {}".format(error))
        elif command == "stream on":
            stream_enabled = True
            send_line("Streaming enabled")
        elif command == "stream off":
            stream_enabled = False
            send_line("Streaming disabled")
        elif command == "help":
            send_line("Commands: status, stream on, stream off, help")
        else:
            send_line("Unknown command")

    time.sleep(0.2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| `BLEUART` setup | Creates the BLE UART service named `AIDER_FarmMonitor` | This is the main communication path of the project |
| `soil_percent()` | Converts the ADC reading into a soil moisture percentage | Makes the moisture reading easier to compare and tune |
| `read_bme280()` | Reads temperature, humidity, and pressure from the BME280 | Adds environmental context to the farm report |
| `stream_enabled` | Controls whether the monitor sends periodic reports | Students can compare streaming and on-demand behaviour |
| `command_queue` | Stores received BLE commands until the main loop can process them | Queued commands are easier to parse and debug |
| `status` command | Sends one fresh measurement on request | On-demand reporting is useful when continuous streaming is disabled |
| `stream on` / `stream off` commands | Enables or disables automatic reports | Gives the user control over BLE traffic and battery use |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The phone cannot find the device | `ble_uart.py` or `ble_advertising.py` is missing, or the Pico is not advertising | Confirm both helper files are on the Pico and restart the board. |
| The phone tries to pair but nothing works | The normal phone Bluetooth settings page is being used | Open a BLE app and connect to the BLE UART-style service instead of pairing in settings. |
| The soil data looks wrong | The soil sensor is not calibrated or the ADC reading is unstable | Recalibrate `soil_dry` and `soil_wet`, then test the sensor in dry and wet soil. |
| The BME280 values are missing | The BME280 is not connected, `bme280.py` is missing, or the I2C address is wrong | Check SDA/SCL wiring, confirm `bme280.py` is on the Pico, and test the BME280 locally. |
| Commands do nothing | The command text does not match the supported commands | Send the exact commands shown in the `help` response. |
| Streaming stops unexpectedly | The BLE connection dropped or the app is disconnected | Reconnect to `AIDER_FarmMonitor` in the BLE app and send `stream on`. |
