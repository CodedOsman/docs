## Overview

This project builds a motion-activated light that can be armed or disarmed from a phone over BLE. A PIR sensor detects movement, the Pico applies decision logic, and an LED provides local lighting.

The system uses edge detection and a timed hold so the light reacts to new motion events and avoids constant retriggering.

---

## Learning Objectives

- Integrate a PIR sensor, LED output, and BLE command channel in one system
- Use edge detection so the system reacts only to new motion events
- Tune motion hold time and sensor sensitivity for a real environment
- Validate BLE communication without confusing BLE app connection with normal pairing
- Test sensor and output stages independently before running the full system

### Project-Specific Setup Notes

- This Bluetooth project uses the built-in `bluetooth` module plus two helper files: `ble_uart.py` and `ble_advertising.py`.
- Save `ble_uart.py` and `ble_advertising.py` to the Pico root folder and confirm with `import os; print(os.listdir())` in Thonny.
- Install a BLE UART app (nRF Connect, LightBlue) and scan for `AIDER_MotionLight` after the Pico starts advertising.
- Do not pair from the phone's normal Bluetooth settings — connect from the BLE app to the UART service instead.

### Project-Specific Safety Note

Many PIR modules use 5V. If you power the PIR from VBUS/5V, confirm its OUT pin is 3.3V safe before connecting to the Pico. Allow the PIR about 30 seconds to warm up before judging detection behaviour.

## Step-by-Step Assembly

1. Place the Raspberry Pi Pico 2W on the breadboard across the center gap.
2. Position the PIR module so the dome faces the detection area and identify VCC/GND/OUT.
3. Connect PIR VCC to VBUS/5V only if the module requires it and its OUT is 3.3V safe.
4. Connect PIR GND to Pico GND.
5. Connect PIR OUT to GPIO 4.
6. Place the LED, connect its anode through a 220Ω resistor to GPIO 0 and its cathode to GND.
7. Verify wiring and give the PIR time to warm up before testing.

## Full Project Code

```python
from machine import Pin
import bluetooth
import time
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_MotionLight"
LED_ON_TIME_MS = 5000
PIR_WARMUP_MS = 30000

led = Pin(0, Pin.OUT)
pir = Pin(4, Pin.IN)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

system_armed = True
led_until = 0
last_motion = 0
command_queue = []

def send_line(message):
    print(message)
    uart.write((message + "\n").encode())

def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)

def report_status():
    armed_text = "ARMED" if system_armed else "DISARMED"
    led_text = "ON" if led.value() else "OFF"
    send_line("System: {} | LED: {}".format(armed_text, led_text))

uart.on_rx(on_rx)
send_line("Bluetooth motion light ready")
send_line("Wait {} seconds for PIR warm-up.".format(PIR_WARMUP_MS // 1000))
time.sleep_ms(PIR_WARMUP_MS)

while True:
    now = time.ticks_ms()
    motion = pir.value()

    # Edge detection: trigger only on rising edge
    if system_armed and motion == 1 and last_motion == 0:
        led_until = time.ticks_add(now, LED_ON_TIME_MS)
        send_line("Motion detected. Light timer restarted.")

    # LED on while timer active
    if system_armed and time.ticks_diff(led_until, now) > 0:
        led.on()
    else:
        led.off()

    # Handle incoming BLE commands
    if command_queue:
        command = command_queue.pop(0)
        if command in ("arm", "on"):
            system_armed = True
            send_line("System armed.")
        elif command in ("disarm", "off"):
            system_armed = False
            led.off()
            send_line("System disarmed.")
        elif command == "status":
            report_status()
        elif command == "help":
            send_line("Commands: arm, disarm, on, off, status, help")
        else:
            send_line("Unknown command. Send help.")

    last_motion = motion
    time.sleep_ms(100)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| BLEUART setup | Starts BLE UART service and sets device name | Lets students connect from a BLE UART app |
| PIR warm-up | Waits ~30s for sensor stabilization | Prevents false triggers during startup |
| Edge detection | Triggers on PIR rising edge only | Avoids repeated retriggers while PIR is high |
| `led_until` timer | Keeps LED on for a fixed hold time after motion | Creates expected motion-light behaviour |
| Command queue | Queues BLE text commands for main loop handling | Keeps wireless input non-blocking |

---

## Expected Result

After warm-up, the LED turns on for the configured hold time when new motion is detected while the system is armed. A phone connected via the BLE app can arm, disarm, and request status.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| LED never turns on | LED polarity reversed or resistor path broken | Reconnect LED anode to GPIO 0 via 220Ω and cathode to GND |
| Motion is detected constantly | PIR still warming up or facing heat source | Wait longer, move PIR away from warm airflow, retest |
| Motion does not trigger | PIR OUT on wrong pin or module power incorrect | Check PIR OUT wiring to GPIO 4 and module supply voltage |
| Phone cannot find BLE device | Pico not advertising or helper files missing | Confirm `ble_uart.py` and `ble_advertising.py` on Pico and restart |
| Commands ignored | Phone connected to wrong service or text differs | Reconnect to BLE UART service and send exact commands |
