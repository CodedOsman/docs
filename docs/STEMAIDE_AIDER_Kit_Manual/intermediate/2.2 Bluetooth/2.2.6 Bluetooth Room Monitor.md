## Overview

This project builds a BLE room monitor that reads temperature and humidity from a BME280 sensor and classifies the room condition.

The Pico uses threshold logic to decide whether the room is too cold, too hot, too humid, or comfortable.

A similar system could support classroom comfort checks, server room monitoring, or basic greenhouse observation.

### Project Story

The final system should show live environmental data, send state changes over BLE, and allow the student to tune the comfort thresholds.

---

## Learning Objectives

- Read room data from a BME280 sensor over I2C
- Convert environmental readings into comfort decisions
- Tune threshold logic for cold, hot, and humid conditions
- Send state-change messages over BLE instead of only raw data
- Test sensor behaviour and communication behaviour separately

### Project-Specific Setup Notes

- These Bluetooth projects use the built-in `bluetooth` module plus two helper files: `ble_uart.py` and `ble_advertising.py`
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`
- In Thonny Shell, run `import os; print(os.listdir())` and confirm both BLE helper files are visible on the Pico
- Save `BME280.py` to the Pico root folder as `BME280.py`
- Run `import os; print(os.listdir())` again and confirm the additional project library files are also visible on the Pico
- **Communication Setup**: install a BLE app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone
- Scan for the device name `AIDER_RoomMonitor` after the Pico starts advertising
- **Do not** pair from the normal phone Bluetooth settings menu. Open the BLE app, scan, connect to the UART-style service, then send text commands from inside the app
- If Bluetooth does not work, restart the Pico and rescan from the BLE app before changing the wiring

### Project-Specific Safety Note

Power the BME280 from 3.3V and keep the I2C wiring short so readings stay stable.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 Sensor
Place the BME280 module on the breadboard or hold it beside the breadboard with jumper wires. Check the printed labels and identify VCC, GND, SDA, and SCL before wiring.

### Step 3: Connect BME280 VCC
Connect BME280 VCC to **3.3V**.

### Step 4: Connect BME280 GND
Connect BME280 GND to **GND**.

### Step 5: Connect SDA to GPIO 8
Connect BME280 SDA to **GPIO 8**.

### Step 6: Connect SCL to GPIO 9
Connect BME280 SCL to **GPIO 9**.

### Step 7: Check the I2C Bus
Make sure SDA and SCL are not crossed before powering the Pico.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] BME280 VCC connects to 3.3V
- [x] BME280 GND connects to GND
- [x] BME280 SDA connects to GPIO 8
- [x] BME280 SCL connects to GPIO 9
- [x] No loose jumper wires

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import bluetooth
import time
import BME280
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_RoomMonitor"

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=100000)
bme = BME280.BME280(i2c=i2c)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

cold_limit = 20.0
hot_limit = 28.0
high_humidity_limit = 70.0

last_state = ""
command_queue = []


def send_line(message):
    print(message)
    uart.write((message + "\n").encode())


def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)


def first_number(text):
    digits = ""
    for char in text:
        if char in "-0123456789.":
            digits += char
        elif digits:
            break
    return float(digits)


def room_state(temp_c, humidity_pct):
    if temp_c < cold_limit:
        return "TOO COLD"
    if temp_c > hot_limit:
        return "TOO HOT"
    if humidity_pct > high_humidity_limit:
        return "TOO HUMID"
    return "COMFORTABLE"


def current_report():
    temp_text = bme.temperature
    humidity_text = bme.humidity
    pressure_text = bme.pressure
    temp_c = first_number(temp_text)
    humidity_pct = first_number(humidity_text)
    state = room_state(temp_c, humidity_pct)
    return temp_text, humidity_text, pressure_text, state


uart.on_rx(on_rx)
send_line("Bluetooth room monitor ready")

while True:
    temp_text, humidity_text, pressure_text, state = current_report()

    if state != last_state:
        send_line("Room state changed: {}".format(state))
        last_state = state

    if command_queue:
        command = command_queue.pop(0)
        if command in ("status", "read"):
            send_line("Temp:{} Hum:{} Press:{}".format(temp_text, humidity_text, pressure_text))
            send_line("State:{}".format(state))
        elif command.startswith("cold "):
            cold_limit = float(command.split()[1])
            send_line("Cold limit set to {}C".format(cold_limit))
        elif command.startswith("hot "):
            hot_limit = float(command.split()[1])
            send_line("Hot limit set to {}C".format(hot_limit))
        elif command.startswith("hum "):
            high_humidity_limit = float(command.split()[1])
            send_line("Humidity limit set to {}%".format(high_humidity_limit))
        elif command == "help":
            send_line("Commands: status, read, cold <value>, hot <value>, hum <value>, help")
        else:
            send_line("Unknown command. Send help.")

    time.sleep_ms(500)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| BME280 reading block | Collects temperature, humidity, and pressure text from the sensor | Provides the room data for every decision |
| `room_state()` | Classifies the room as too cold, too hot, too humid, or comfortable | Turns numbers into an engineering decision |
| State-change check | Sends a message only when the room condition changes | Makes the monitoring output easier to follow |
| Threshold commands | Let the user tune cold, hot, and humidity limits over BLE | Supports calibration for different rooms or climates |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| Readings are missing | BME280 wiring or library file is incorrect | Check GPIO 8, GPIO 9, 3.3V, GND, and confirm BME280.py is on the Pico |
| Room state looks wrong | The threshold values are unrealistic for the test room | Review the cold, hot, and humidity limits and tune them again |
| No state-change messages appear | The room never crossed a threshold or the limits are too wide | Temporarily tighten the limits to verify the change-detection logic |
| Phone cannot find the BLE device | The Pico is not advertising or the BLE helper files are missing | Confirm ble_uart.py and ble_advertising.py are saved on the Pico, restart the board, and scan again from nRF Connect or LightBlue |
| Commands do not change the system state | The phone is connected to the wrong service or the command text is different from the expected command | Reconnect to the BLE UART service and send the exact command shown in the test section |
