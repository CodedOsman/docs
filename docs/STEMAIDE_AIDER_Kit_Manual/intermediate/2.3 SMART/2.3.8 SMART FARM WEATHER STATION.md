## Overview

This project builds a small farm weather station that combines temperature, humidity, rainfall state, and brightness into one field-style summary.

Students will wire a BME280 climate sensor, a digital rain sensor, and an LDR light sensor, then classify the environment with rule-based weather states.

The final system should print live weather readings and label conditions such as CLEAR, HOT, COLD, RAIN WATCH, or RAINING.

### Project Story

The real-world use case is a planting area or small farm corner where a quick local weather snapshot helps decide whether to irrigate, cover crops, or delay outdoor work.

## Learning Objectives

- Use a digital rain sensor together with analog brightness and BME280 climate data

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 Sensor Module
Place the BME280 module where it can sense the air around the weather station. Identify VCC, GND, and DATA before wiring.

### Step 3: Connect the BME280 Sensor Module
Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 2** and BME280 SCL to **GPIO 3**. If you are using a bare BME280 sensor, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 4: Position the Rain Sensor
Place the rain sensor area where test drops can reach it without dripping onto the Pico or breadboard. Identify VCC, GND, and DOUT before wiring.

### Step 5: Connect the Rain Sensor
Connect rain sensor VCC to **3.3V** when the module supports 3.3V operation. Connect rain sensor GND to **GND**. Connect rain sensor DOUT to **GPIO 5**. Confirm the DOUT signal is safe for 3.3V Pico GPIO before powering the circuit.

### Step 6: Build the LDR Divider
Place the LDR and 10kΩ resistor on the breadboard. Connect one LDR leg to **3.3V**. Connect the other LDR leg to **GPIO 26**. Connect one end of the 10kΩ resistor to **GPIO 26**. Connect the other end of the 10kΩ resistor to **GND**.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] DHT22 DATA connects to GPIO 4
- [x] Rain sensor DOUT connects to GPIO 5
- [x] Rain sensor output is confirmed 3.3V-safe before connecting to the Pico
- [x] LDR divider node connects to GPIO 26
- [x] 10kΩ resistor connects from GPIO 26 to GND
- [x] No loose jumper wires

> **Intermediate Note**
> The BME280 needs a short delay between readings. Test the rain sensor dry and with a few drops, and test the LDR in bright light and shade before trusting the weather state.

> **Safety Note**
> Keep water drops away from the Pico, breadboard, USB cable, and jumper wires. If the rain sensor board is designed for 5V modules, confirm its digital output is 3.3V-safe before connecting it to GPIO 5.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import bme280
import time

i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


rain_sensor = Pin(5, Pin.IN, Pin.PULL_UP)
light_sensor = ADC(26)

temp_min = 10
temp_max = 35
humidity_high = 90
rain_light_threshold = 35


def light_percent():
    raw = light_sensor.read_u16()
    return int((raw / 65535) * 100)


def read_weather():
    temp_c, humidity_pct, pressure_hpa = read_bme280()
    raining = rain_sensor.value() == 0
    brightness = light_percent()
    return temp_c, humidity_pct, raining, brightness


def weather_state(temp_c, humidity_pct, raining, brightness):
    if raining:
        return "RAINING"

    if humidity_pct >= humidity_high and brightness <= rain_light_threshold:
        return "RAIN WATCH"

    if temp_c > temp_max:
        return "HOT"
    if temp_c < temp_min:
        return "COLD"

    return "CLEAR"


print("Smart farm weather station ready")
time.sleep(2)

while True:
    try:
        temp_c, humidity_pct, raining, brightness = read_weather()
    except Exception as error:
        print("Sensor error:", error)
        time.sleep(2)
        continue

    state = weather_state(temp_c, humidity_pct, raining, brightness)
    print("Temp:{}C Hum:{}% Light:{}% Rain:{} State:{}".format(
        temp_c,
        humidity_pct,
        brightness,
        raining,
        state,
    ))
    time.sleep(5)
```

## Troubleshooting

| Problem | 1 | Solution |
|---------|---|----------|
| The station always says CLEAR | 1 | Test humidity, brightness, and rain inputs separately to find which sensor is stuck |
| RAINING appears all the time | 1 | Dry the board and confirm whether active-low logic matches your sensor |
| Brightness never changes | 1 | Rebuild the divider and check the ADC node at GPIO 26 |
| Humidity is always zero | 1 | Check wiring, power, and read timing for the BME280 sensor |
