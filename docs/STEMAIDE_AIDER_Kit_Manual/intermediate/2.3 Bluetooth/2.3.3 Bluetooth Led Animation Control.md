# Project 2.3.3: Bluetooth Led Animation Control

**Intermediate Embedded Systems Project Using Raspberry Pi Pico 2 W and     MicroPython**

---
## Overview

This project builds a four-LED animation controller that receives pattern commands over BLE.

A similar approach could be used for indicator panels, wearable art, small status displays, or prototype user feedback systems.

The final system should allow a phone to trigger animations, turn all LEDs on or off, and tune the animation speed.

### Project Story

The Pico runs different timing patterns such as blink, chase, wave, and pong, so students can see how timing logic affects output behaviour.

---

## Learning Objectives

- Control multiple digital outputs from one command interface
- Use loops and timing to create repeatable animation patterns
- Tune animation speed to change how a system behaves visually
- Structure commands so one controller can manage several behaviours
- Test each LED channel before testing the full animation logic

### Project-Specific Setup Notes

- These Bluetooth projects use the built-in `bluetooth` module plus two helper files: `ble_uart.py` and `ble_advertising.py`
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`
- In Thonny Shell, run `import os; print(os.listdir())` and confirm both BLE helper files are visible on the Pico
- **Communication Setup**: install a BLE app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone
- Scan for the device name `AIDER_LEDAnim` after the Pico starts advertising
- **Do not** pair from the normal phone Bluetooth settings menu. Open the BLE app, scan, connect to the UART-style service, then send text commands from inside the app
- If Bluetooth does not work, restart the Pico and rescan from the BLE app before changing the wiring

### Project-Specific Safety Note

Each LED must have its own series resistor. Do not connect four LEDs in parallel to one resistor.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the Four LEDs
Place four LEDs in a clear row on the breadboard. For each LED, identify the long leg as the anode (+) and the short leg as the cathode (-).

### Step 3: Connect LED 1 to GPIO 16
Connect LED 1 long leg to one end of a 220Ω resistor. Connect the other end of the resistor to **GPIO 16**.

### Step 4: Connect LED 2 to GPIO 17
Connect LED 2 long leg to one end of a 220Ω resistor. Connect the other end of the resistor to **GPIO 17**.

### Step 5: Connect LED 3 to GPIO 18
Connect LED 3 long leg to one end of a 220Ω resistor. Connect the other end of the resistor to **GPIO 18**.

### Step 6: Connect LED 4 to GPIO 19
Connect LED 4 long leg to one end of a 220Ω resistor. Connect the other end of the resistor to **GPIO 19**.

### Step 7: Connect the LED Short Legs
Connect the short leg of each LED to **GND**.

### Step 8: Check the LED Order
Confirm that the left-to-right LED order matches GPIO 16, GPIO 17, GPIO 18, and GPIO 19.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import Pin
import bluetooth
import time
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_LEDAnim"

leds = [Pin(16, Pin.OUT), Pin(17, Pin.OUT), Pin(18, Pin.OUT), Pin(19, Pin.OUT)]

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

step_delay_ms = 120
command_queue = []


def send_line(message):
    print(message)
    uart.write((message + "\n").encode())


def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)


def all_off():
    for led in leds:
        led.off()


def all_on():
    for led in leds:
        led.on()


def blink_pattern():
    for _ in range(3):
        all_on()
        time.sleep_ms(step_delay_ms)
        all_off()
        time.sleep_ms(step_delay_ms)


def chase_pattern():
    for _ in range(3):
        for led in leds:
            all_off()
            led.on()
            time.sleep_ms(step_delay_ms)


def wave_pattern():
    for _ in range(3):
        for index in range(len(leds)):
            leds[index].on()
            time.sleep_ms(step_delay_ms)
        for index in range(len(leds)):
            leds[index].off()
            time.sleep_ms(step_delay_ms)


def pong_pattern():
    order = [0, 1, 2, 3, 2, 1]
    for _ in range(3):
        for index in order:
            all_off()
            leds[index].on()
            time.sleep_ms(step_delay_ms)


uart.on_rx(on_rx)
all_off()
send_line("Bluetooth LED animation controller ready")

while True:
    if command_queue:
        command = command_queue.pop(0)
        if command == "blink":
            blink_pattern()
            send_line("Blink pattern complete")
        elif command == "chase":
            chase_pattern()
            send_line("Chase pattern complete")
        elif command == "wave":
            wave_pattern()
            send_line("Wave pattern complete")
        elif command == "pong":
            pong_pattern()
            send_line("Pong pattern complete")
        elif command == "all on":
            all_on()
            send_line("All LEDs ON")
        elif command == "all off":
            all_off()
            send_line("All LEDs OFF")
        elif command.startswith("speed "):
            step_delay_ms = int(command.split()[1])
            send_line("Animation speed set to {} ms".format(step_delay_ms))
        elif command == "status":
            send_line("Animation step delay: {} ms".format(step_delay_ms))
        elif command == "help":
            send_line("Commands: blink, chase, wave, pong, all on, all off, speed <ms>, status, help")
        else:
            send_line("Unknown command. Send help.")

    time.sleep_ms(80)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| LED list | Stores all four output channels in one structure | Makes repeated animations easier to write |
| Pattern functions | Group the blink, chase, wave, and pong logic | Keeps the command handler simple and readable |
| `step_delay_ms` | Controls the timing speed for every pattern | Lets students tune the visual behaviour without rewriting the logic |
| BLE command handler | Chooses which pattern or control action to run | Turns the phone into a remote animation controller |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| One LED never lights | The LED is reversed or the resistor path is broken | Recheck the polarity and the resistor connection for that channel |
| Pattern order looks wrong | The LEDs are wired in a different physical order from the code | Trace each GPIO wire and relabel the LEDs from left to right |
| Animation is too fast or too slow | The step delay is not tuned for the test setup | Use `speed <ms>` to adjust the timing until the pattern is clear |
| Phone cannot find the BLE device | The Pico is not advertising or the BLE helper files are missing | Confirm ble_uart.py and ble_advertising.py are saved on the Pico, restart the board, and scan again from nRF Connect or LightBlue |
| Commands do not change the system state | The phone is connected to the wrong service or the command text is different from the expected command | Reconnect to the BLE UART service and send the exact command shown in the test section |
