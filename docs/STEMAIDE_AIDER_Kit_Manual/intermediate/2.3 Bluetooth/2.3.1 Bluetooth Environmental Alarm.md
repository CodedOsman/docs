# Project 2.3.1: Bluetooth Environmental Alarm

**Intermediate Embedded Systems Project Using Raspberry Pi Pico 2 W and MicroPython**

---
## Overview

This project combines a BME280 environmental sensor, a buzzer, and BLE monitoring to create a simple environment alarm node.

The Pico reads temperature, humidity, and pressure, then uses threshold logic to decide when to raise an alert.

A similar design could be used in a greenhouse, small storeroom, or equipment cabinet where unsafe climate conditions should be reported quickly.

The final system should show live environmental data, trigger a buzzer when limits are exceeded, and allow the phone to adjust the temperature and humidity thresholds.

### Project Story

This project connects classroom electronics to a practical embedded-system problem. Learners will see how sensors, logic, outputs, and communication can work together in a real application.

---

## Learning Objectives

- Wire and test a BME280 sensor on the I2C bus
- Parse sensor readings and compare them against decision thresholds
- Use a buzzer as a local alarm output
- Tune temperature and humidity limits through BLE commands
- Separate sensor verification, output verification, and full-system testing

### Project-Specific Setup Notes

- These Bluetooth projects use the built-in `bluetooth` module plus two helper files: `ble_uart.py` and `ble_advertising.py`
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`
- In Thonny Shell, run `import os; print(os.listdir())` and confirm both BLE helper files are visible on the Pico
- Save `BME280.py` to the Pico root folder as `BME280.py`
- Run `import os; print(os.listdir())` again and confirm the additional project library files are also visible on the Pico
- **Communication Setup**: install a BLE app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone
- Scan for the device name `AIDER_EnvAlarm` after the Pico starts advertising
- **Do not** pair from the normal phone Bluetooth settings menu. Open the BLE app, scan, connect to the UART-style service, then send text commands from inside the app
- If Bluetooth does not work, restart the Pico and rescan from the BLE app before changing the wiring

### Project-Specific Safety Note

Power the BME280 from **3.3V**, not 5V, unless your exact breakout board documentation clearly states that it is level-shifted and 3.3V safe on SDA and SCL.

Do not leave the buzzer on continuously for long tests. Short bursts are enough to confirm the alarm works.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 Sensor
Place the BME280 module on the breadboard or hold it beside the breadboard with jumper wires. Check the printed labels and identify VCC, GND, SDA, and SCL before wiring.

### Step 3: Connect BME280 VCC
Connect BME280 VCC to **3.3V**.

### Step 4: Connect BME280 GND
Connect BME280 GND to **GND**.

### Step 5: Connect SDA to GPIO 8
Connect BME280 SDA to **GPIO 8**.

### Step 6: Connect SCL to GPIO 9
Connect BME280 SCL to **GPIO 9**.

### Step 7: Place the Buzzer
Place the buzzer on the breadboard and identify its positive (+) and negative (-) pins.

### Step 8: Connect the Buzzer Positive Pin
Connect the buzzer positive (+) pin to **GPIO 18**.

### Step 9: Connect the Buzzer Negative Pin
Connect the buzzer negative (-) pin to **GND**.

### Step 10: Check the I2C Wires
Make sure SDA and SCL are not crossed before powering the Pico.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import bluetooth
import time
import BME280
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_EnvAlarm"
ALARM_COOLDOWN_MS = 15000

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=100000)
bme = BME280.BME280(i2c=i2c)
buzzer = Pin(18, Pin.OUT)

ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

temp_limit = 32.0
humidity_limit = 80.0
alarm_enabled = True
last_alarm_ms = time.ticks_add(time.ticks_ms(), -ALARM_COOLDOWN_MS)

command_queue = []


def send_line(message):
    print(message)
    uart.write((message + "\n").encode())


def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)


def first_number(text):
    digits = ""
    for char in text:
        if char in "-0123456789.":
            digits += char
        elif digits:
            break
    return float(digits)


def alarm_beep():
    for _ in range(2):
        buzzer.on()
        time.sleep_ms(200)
        buzzer.off()
        time.sleep_ms(200)


def read_environment():
    temp_text = bme.temperature
    humidity_text = bme.humidity
    pressure_text = bme.pressure
    temp_c = first_number(temp_text)
    humidity_pct = first_number(humidity_text)
    return temp_text, humidity_text, pressure_text, temp_c, humidity_pct


uart.on_rx(on_rx)
send_line("Bluetooth environmental alarm ready")

while True:
    now = time.ticks_ms()
    temp_text, humidity_text, pressure_text, temp_c, humidity_pct = read_environment()

    alerts = []
    if temp_c > temp_limit:
        alerts.append("HIGH TEMP")
    if humidity_pct > humidity_limit:
        alerts.append("HIGH HUMIDITY")

    if alerts and alarm_enabled and time.ticks_diff(now, last_alarm_ms) >= ALARM_COOLDOWN_MS:
        alarm_beep()
        send_line("ALERT: {}".format(" | ".join(alerts)))
        last_alarm_ms = now

    if command_queue:
        command = command_queue.pop(0)
        if command in ("status", "read"):
            send_line("Temp:{} Hum:{} Press:{}".format(temp_text, humidity_text, pressure_text))
            send_line("Limits: Temp>{}C Hum>{}%".format(temp_limit, humidity_limit))
        elif command == "arm":
            alarm_enabled = True
            send_line("Alarm armed")
        elif command in ("silence", "disarm"):
            alarm_enabled = False
            buzzer.off()
            send_line("Alarm silenced")
        elif command.startswith("temp "):
            temp_limit = float(command.split()[1])
            send_line("Temperature limit set to {}C".format(temp_limit))
        elif command.startswith("hum "):
            humidity_limit = float(command.split()[1])
            send_line("Humidity limit set to {}%".format(humidity_limit))
        elif command == "help":
            send_line("Commands: status, read, arm, silence, disarm, temp <value>, hum <value>, help")
        else:
            send_line("Unknown command. Send help.")

    time.sleep_ms(300)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| BME280 setup | Starts the I2C sensor and captures temperature, humidity, and pressure | The project needs reliable climate data before it can make decisions |
| `first_number()` | Converts BME280 text readings into numeric values | The code must compare real numbers against threshold limits |
| alerts list | Collects every threshold that has been crossed | Allows one message to explain exactly what is wrong |
| Alarm cooldown | Prevents the buzzer from sounding continuously every loop cycle | Makes the alarm more realistic and less noisy |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| No environmental reading appears | SDA and SCL are reversed or BME280.py is missing | Check GPIO 8 and GPIO 9 wiring and confirm BME280.py is saved on the Pico root folder |
| Alarm sounds too often | The threshold is set too close to normal room conditions | Increase the limit values or add more cooldown time after testing |
| Buzzer is silent | The buzzer wiring is wrong or the buzzer needs a driver | Verify buzzer polarity and use a transistor driver if the module is not 3.3V-safe |
| Phone cannot find the BLE device | The Pico is not advertising or the BLE helper files are missing | Confirm ble_uart.py and ble_advertising.py are saved on the Pico, restart the board, and scan again from nRF Connect or LightBlue |
| Commands do not change the system state | The phone is connected to the wrong service or the command text is different from the expected command | Reconnect to the BLE UART service and send the exact command shown in the test section |
