## Overview

This project builds a humidity-control demo that runs a misting output when the air becomes too dry.

Students will use a BME280 sensor to measure humidity, apply threshold and recovery logic, and drive a relay-controlled mister or small pump.

The final system should monitor humidity continuously, start a timed misting cycle when humidity is too low, and wait for recovery before allowing another cycle.

### Project Story

The real-world use case is a seedling tray or small humidity-sensitive plant area where dry air can stress young plants.

---

## Learning Objectives

- Read humidity values from a BME280 sensor in MicroPython
- Use a relay-controlled humidifying output safely
- Apply threshold and recovery logic to reduce short repeated cycles
- Understand why humidity control needs both sensing and rest timing
- Test low-humidity response before connecting the real misting device

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules
- This project uses the BME280 MicroPython library provided by MicroPython, so no extra library file is normally needed

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins.

Use an external power supply for pumps, motors, relay modules, and fans when required.

If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly.

Check whether your relay module is active-low or active-high before testing the final load.

Keep electronics away from water and dry your hands before touching the circuit.

Do not let exposed jumper connections touch water.

Do not let a pump run dry.

Do not point mist or spray directly at the Pico, breadboard, or USB cable.

If you replace the mister with a pump for testing, keep the tubing secure and supervise the water path.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 Sensor Module
Place the BME280 module where it can sense the greenhouse air without getting wet. Identify VCC, GND, SDA, and SCL before wiring.

### Step 3: Connect the BME280 Sensor Module
Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 2** and BME280 SCL to **GPIO 3**.

### Step 4: Place and Wire the Relay Module
Identify relay VCC, GND, IN, COM, NO, and NC. Connect relay IN to **GPIO 15**. Power the relay module from the supply required by its label. Connect relay GND to Pico GND and to the external mister or pump supply GND where shared grounding is required.

### Step 5: Place the Status LED
Place the status LED on the breadboard. Connect its long leg through a 220Ω resistor to **GPIO 16**. Connect its short leg to **GND**.

### Step 6: Wire the Mister Through the Relay
Connect the external mister or pump supply positive wire to relay **COM**. Connect relay **NO** to the mister or pump positive lead. Connect the mister or pump negative lead to the external supply negative wire.

### Step 7: Aim the Moisture Output Safely
Point mist, spray, or tubing away from the Pico, breadboard, USB cable, and jumper wires.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] BME280 VCC connects to 3.3V
- [x] BME280 GND connects to GND
- [x] BME280 SDA connects to GPIO 2
- [x] BME280 SCL connects to GPIO 3
- [x] Relay IN connects to GPIO 15
- [x] Relay module uses the correct external supply and shared ground where required
- [x] Status LED long leg connects through a 220Ω resistor to GPIO 16
- [x] Status LED short leg connects to GND
- [x] Mister or pump positive path uses relay COM and NO
- [x] No loose jumper wires

> **Intermediate Note**
> The BME280 needs a short delay between readings. If humidity readings fail, wait a moment, check the SDA and SCL wires, and confirm whether your sensor module needs a pull-up resistor.

> **Safety Note**
> Use an external supply for the mister or pump. Keep mist and water away from the Pico, breadboard, USB cable, and jumper wires, and never let a pump run dry.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import bme280
import time

RELAY_ON = 0
RELAY_OFF = 1
MIST_RUN_S = 4
MIN_REST_S = 20

i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)

humidity_low = 45
humidity_recovery = 55
last_cycle = time.time() - MIN_REST_S
ready_to_mist = True

relay.value(RELAY_OFF)
status_led.value(0)


def read_humidity():
    temp_c, humidity, pressure_hpa = read_bme280()
    return humidity


def mist_cycle():
    relay.value(RELAY_ON)
    status_led.value(1)
    print("Mister ON: humidity below low threshold")
    time.sleep(MIST_RUN_S)
    relay.value(RELAY_OFF)
    status_led.value(0)
    print("Mister OFF: timed cycle complete")


print("Greenhouse humidity control ready")
time.sleep(2)

while True:
    try:
        humidity = read_humidity()
    except Exception as error:
        print("Sensor error:", error)
        time.sleep(2)
        continue

    now = time.time()

    if humidity >= humidity_recovery:
        ready_to_mist = True

    if ready_to_mist and humidity <= humidity_low and (now - last_cycle) >= MIN_REST_S:
        mist_cycle()
        last_cycle = time.time()
        ready_to_mist = False
    else:
        print("Humidity:{}% Ready:{} Rest:{}s".format(
            humidity,
            ready_to_mist,
            int(now - last_cycle),
        ))

    time.sleep(3)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| `humidity_low` and `humidity_recovery` | Define the start and recovery thresholds for misting | Two thresholds are safer than one for actuator control |
| `MIN_REST_S` | Enforces a minimum gap between misting cycles | The output should not pulse rapidly if humidity changes slowly |
| `mist_cycle()` | Runs a timed misting burst | Timed control makes the output easy to test safely |
| Exception handling around `read_humidity()` | Catches sensor read errors gracefully | BME280 sensors can be noisy and should fail safely |

## Expected Result

The system should report live humidity values. When the humidity falls below the low threshold, the relay should run the misting output for a short cycle and the LED should light. The system should then wait until humidity rises above the recovery threshold before allowing another automatic cycle.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| Humidity always reads the same | Wrong I2C wiring or inadequate delay between reads | Recheck SDA/SCL wiring and keep at least a two-second gap between reads |
| The mister never turns on | Thresholds are too low or relay logic is reversed | Raise the low threshold for testing and verify `RELAY_ON` / `RELAY_OFF` |
| The output keeps pulsing | Rest interval is too short or recovery threshold not reached | Increase `MIN_REST_S` or raise `humidity_recovery` |
| Water reaches the electronics | Mister is pointed at the board | Stop the test immediately, dry the workspace, and reposition the hardware |
