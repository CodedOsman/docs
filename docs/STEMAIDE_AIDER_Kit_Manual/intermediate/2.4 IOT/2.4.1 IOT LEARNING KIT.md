## Overview

This project builds a small BLE learning station that combines an analog input, a PWM-controlled LED, and a threshold-based buzzer alert.

A potentiometer acts as the input, the LED brightness follows the input level, and the buzzer warns when the level crosses a threshold.

The final system should display the input level, adjust LED brightness, sound an alert above a chosen threshold, and let the threshold be tuned from a phone.

### Project Story

The real-world lesson is that IoT systems often combine sensing, visual feedback, alerts, and remote monitoring rather than doing only one thing.

---

## Learning Objectives

- Integrate analog input, PWM output, alert logic, and BLE communication
- Use a potentiometer to simulate a changing sensor level
- Drive an LED with PWM so brightness follows the input
- Tune an alert threshold for rule-based decision making
- Compare local feedback with remote status reporting

### Project-Specific Setup Notes

- This project uses the Pico's built-in wireless module plus two helper files: `ble_uart.py` and `ble_advertising.py`
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`
- In Thonny Shell, run `import os; print(os.listdir())` and confirm both helper files are visible on the Pico
- Communication Setup: install a BLE app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone
- Scan for the device name `AIDER_LearnKit` after the Pico starts advertising
- **Do not** pair from the normal phone Bluetooth settings menu. Open the terminal app, scan, connect to the UART-style service, then send text commands from inside the app
- If the wireless connection does not work, restart the Pico and rescan from the terminal app before changing the wiring

### Project-Specific Safety Note

Use 3.3V only for the potentiometer. If the buzzer needs more current than a GPIO pin can supply, add a transistor driver stage.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the Potentiometer

Insert the potentiometer onto the breadboard. Make sure each of the three potentiometer pins sits on a separate breadboard row.

### Step 3: Connect One Outer Pin

Connect one outer potentiometer pin to 3.3V.

### Step 4: Connect the Middle Pin

Connect the potentiometer middle pin to GPIO 26 (ADC0).

### Step 5: Connect the Other Outer Pin

Connect the other outer potentiometer pin to GND.

### Step 6: Place the LED

Place the LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-).

### Step 7: Connect the LED Long Leg

Connect the LED long leg to one end of a 220Ω resistor.

### Step 8: Connect the Resistor to GPIO 16

Connect the other end of the resistor to GPIO 16.

### Step 9: Connect the LED Short Leg

Connect the LED short leg to GND.

### Step 10: Place the Buzzer

Place the buzzer on the breadboard and identify its positive (+) and negative (-) pins.

### Step 11: Connect the Buzzer Positive Pin

Connect the buzzer positive (+) pin to GPIO 17.

### Step 12: Connect the Buzzer Negative Pin

Connect the buzzer negative (-) pin to GND.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, PWM, Pin
import bluetooth
import time
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_LearnKit"
ALERT_COOLDOWN_MS = 1500
sensor = ADC(26)
led = PWM(Pin(16))
led.freq(1000)
buzzer = Pin(17, Pin.OUT)
ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

threshold_percent = 80
alerts_enabled = True
last_alert_ms = time.ticks_add(time.ticks_ms(), -ALERT_COOLDOWN_MS)
command_queue = []


def send_line(message):
    print(message)
    uart.write((message + "\n").encode())


def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)


def current_percent():
    raw_value = sensor.read_u16()
    return int((raw_value * 100) / 65535)


def set_brightness(percent):
    duty = int((percent * 65535) / 100)
    led.duty_u16(duty)


def beep_alert():
    buzzer.on()
    time.sleep_ms(120)
    buzzer.off()


uart.on_rx(on_rx)
send_line("Bluetooth IoT learning kit ready")

while True:
    now = time.ticks_ms()
    percent = current_percent()
    set_brightness(percent)

    if alerts_enabled and percent >= threshold_percent and time.ticks_diff(now, last_alert_ms) >= ALERT_COOLDOWN_MS:
        beep_alert()
        send_line("ALERT: Input level {}% reached the threshold".format(percent))
        last_alert_ms = now

    if command_queue:
        command = command_queue.pop(0)
        if command == "status":
            send_line("Level:{}% Threshold:{}% Alerts:{}".format(percent, threshold_percent, alerts_enabled))
        elif command.startswith("threshold "):
            threshold_percent = int(command.split()[1])
            send_line("Threshold set to {}%".format(threshold_percent))
        elif command == "alerts on":
            alerts_enabled = True
            send_line("Alerts enabled")
        elif command in ("alerts off", "mute"):
            alerts_enabled = False
            buzzer.off()
            send_line("Alerts disabled")
        elif command == "help":
            send_line("Commands: status, threshold <value>, alerts on, alerts off, mute, help")
        else:
            send_line("Unknown command. Send help.")

    time.sleep_ms(100)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| current_percent() | Reads the analog level and scales it to 0 to 100% | One normalised value can drive several outputs |
| PWM LED control | Maps the percentage directly to LED brightness | Creates clear visual feedback |
| Threshold comparison | Checks whether the input level is above the alert limit | Adds rule-based decision logic to the kit |
| BLE threshold commands | Allow the student to tune the alert limit remotely | Encourage calibration and experimentation |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| LED brightness does not change | The LED is not on the PWM pin path or the potentiometer is wired incorrectly | Check GPIO 16, the resistor, and the potentiometer middle-pin connection on GPIO 26. |
| Alert never triggers | The threshold is too high or alerts are disabled | Lower the threshold temporarily and confirm alerts are enabled. |
| Buzzer stays silent | The buzzer wiring is wrong or the buzzer needs a driver | Verify GPIO 17 wiring and use a driver if the buzzer needs more current. |
| Phone cannot find the BLE device | The Pico is not advertising or the BLE helper files are missing | Confirm ble_uart.py and ble_advertising.py are saved on the Pico, restart the board, and scan again from nRF Connect or LightBlue. |
| Commands do not change the system state | The phone is connected to the wrong service or the command text is different from the expected command | Reconnect to the BLE UART service and send the exact command shown in the test section. |
