# Project 2.4.3: Greenhouse Temperature Control

**Intermediate Embedded Systems Project Using Raspberry Pi Pico 2 W and MicroPython**

---
## Overview

This project builds a greenhouse temperature controller that switches a cooling fan when the air becomes too hot.

Students will build a shared I2C sensor-and-display bus using a BME280 and OLED, then connect a relay-controlled fan output.

The final system should show live climate values, turn the fan on above the high temperature threshold, and turn it off again only after the temperature drops below the low threshold.

### Project Story

The real-world use case is a seedling enclosure or covered plant shelf where temperature needs active control to prevent heat stress.

---

## Learning Objectives

- Wire and test two I2C devices on the same bus
- Read BME280 temperature, humidity, and pressure data in MicroPython
- Use hysteresis to control a relay-driven cooling fan
- Display live operating status on an OLED screen
- Test the sensor path, display path, and output path separately before full-system control

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins.

Use an external power supply for pumps, motors, relay modules, and fans when required.

If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly.

Check whether your relay module is active-low or active-high before testing the final load.

Power the BME280 and OLED from 3.3V, not 5V, unless the exact breakout boards are documented as level-shifted and 3.3V safe.

Do not power the fan directly from the Pico.

Check the fan polarity before the first live test.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 and OLED
Place the BME280 and OLED display so their I2C pins are easy to reach. Identify VCC, GND, SDA, and SCL on both modules before wiring.

### Step 3: Connect BME280 Power and I2C
Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 8**. Connect BME280 SCL to **GPIO 9**.

### Step 4: Connect OLED Power and I2C
Connect OLED VCC to **3.3V**. Connect OLED GND to **GND**. Connect OLED SDA to the same **GPIO 8** I2C bus. Connect OLED SCL to the same **GPIO 9** I2C bus.

### Step 5: Place and Wire the Relay Module
Identify relay VCC, GND, IN, COM, NO, and NC. Connect relay IN to **GPIO 15**. Power the relay module from the supply required by the module label. Connect relay GND to Pico GND and to the external fan supply GND where shared grounding is required.

### Step 6: Place the Status LED
Place the status LED on the breadboard. Connect its long leg through a 220Ω resistor to **GPIO 16**. Connect its short leg to **GND**.

### Step 7: Wire the Fan Through the Relay
Connect the external fan supply positive wire to relay **COM**. Connect relay **NO** to the fan positive lead. Connect the fan negative lead to the external fan supply negative wire.

### Step 8: Check the Fan
Make sure the fan blades can spin freely and no wire can touch the blades.

### Wiring Check

- [x] BME280 GND and OLED GND connect to GND
- [x] BME280 and OLED SDA share GPIO 8
- [x] BME280 and OLED SCL share GPIO 9
- [x] Relay IN connects to GPIO 15

- [x] Fan positive path uses relay COM and NO
- [x] Fan negative connects to external fan supply negative

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import time
import BME280
from ssd1306 import SSD1306_I2C

RELAY_ON = 0
RELAY_OFF = 1

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=100000)
bme = BME280.BME280(i2c=i2c)
oled = SSD1306_I2C(128, 64, i2c)

fan_relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)

temp_high = 30.0
temp_low = 27.0
fan_running = False

fan_relay.value(RELAY_OFF)
status_led.value(0)


def first_number(text):
    digits = ""
    for char in text:
        if char in "-0123456789.":
            digits += char
        elif digits:
            break
    return float(digits)


def read_environment():
    temp_text, pressure_text, humidity_text = bme.values
    temp_c = first_number(temp_text)
    humidity_pct = first_number(humidity_text)
    return temp_c, humidity_pct, pressure_text


def update_display(temp_c, humidity_pct, pressure_text, fan_state):
    oled.fill(0)
    oled.text("Greenhouse", 18, 0)
    oled.text("Temp:{:.1f}C".format(temp_c), 0, 18)
    oled.text("Hum :{:.1f}%".format(humidity_pct), 0, 30)
    oled.text("Fan :{}".format("ON" if fan_state else "OFF"), 0, 42)
    oled.text(pressure_text[:16], 0, 54)
    oled.show()


print("Greenhouse temperature control ready")

while True:
    temp_c, humidity_pct, pressure_text = read_environment()

    if temp_c >= temp_high and not fan_running:
        fan_relay.value(RELAY_ON)
        status_led.value(1)
        fan_running = True
        print("Fan ON: {:.1f}C is above the high threshold".format(temp_c))
    elif temp_c <= temp_low and fan_running:
        fan_relay.value(RELAY_OFF)
        status_led.value(0)
        fan_running = False
        print("Fan OFF: {:.1f}C is below the low threshold".format(temp_c))

    print("Temp:{:.1f}C Hum:{:.1f}% Fan:{}".format(
        temp_c,
        humidity_pct,
        "ON" if fan_running else "OFF",
    ))

    update_display(temp_c, humidity_pct, pressure_text, fan_running)
    time.sleep(3)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| Shared I2C bus | Lets the BME280 and OLED use the same SDA and SCL pins | Students learn multi-device bus wiring rather than one-device-only wiring |
| `first_number()` | Extracts numeric values from the BME280 text strings | MicroPython sensor libraries often return formatted strings that must be parsed |
| `temp_high` and `temp_low` | Create hysteresis for the fan control | A single threshold would cause faster relay chatter near the set point |
| `update_display()` | Draws the climate data and fan state on the OLED | Local feedback makes the system easier to test without a computer |

## Expected Result

The OLED should show live greenhouse readings. When the temperature rises above the high threshold, the relay should switch the fan on and the status LED should light. The fan should stay on until the temperature drops below the low threshold.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The OLED is blank | Wrong I2C wiring or the display address is not on the bus | Run `i2c.scan()`, confirm power, and recheck SDA/SCL |
| The sensor values do not change | The BME280 is not responding correctly | Check the power, I2C bus, and the `BME280.py` file on the Pico |
| The fan runs at boot | Relay logic polarity is reversed | Swap `RELAY_ON` and `RELAY_OFF` or confirm the relay module type |
| The fan toggles too often | The high and low thresholds are too close together | Increase the hysteresis gap between `temp_high` and `temp_low` |
