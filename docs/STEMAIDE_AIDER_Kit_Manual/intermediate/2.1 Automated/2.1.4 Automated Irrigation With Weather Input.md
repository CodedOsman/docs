## Overview

This project adds a weather-aware rule to irrigation so the system can skip watering when humidity suggests rain or very damp conditions are likely.

Students will combine soil moisture, humidity trend logic, a relay-controlled pump, and a timed watering cycle.

The final system should water only when the soil is dry enough and the weather rule does not suggest that watering should be postponed.

### Project Story

The real-world use case is an outdoor or ventilated growing area where watering just before rain would waste water and may over-saturate the soil.

---

## Learning Objectives

- Combine environmental sensing with irrigation decision logic
- Use humidity history as a simple weather input heuristic
- Prevent watering when damp or rain-like conditions are detected
- Keep pump control safe with timed cycles and recovery thresholds
- Understand why weather-aware control is still rule-based, not a true forecast service

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules
- This project uses a humidity-based weather heuristic. It does not use online forecast data or machine learning

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins.
Use an external power supply for pumps, motors, relay modules, and fans when required.
If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly.
Check whether your relay module is active-low or active-high before testing the final load.
Keep electronics away from water and dry your hands before touching the circuit.
Do not let exposed jumper connections touch water.
Do not let a pump run dry.
Because the system can skip watering based on humidity, always verify the weather rule during testing before leaving the system unattended.
Use a short pump run time during the first live test.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Soil Moisture Sensor

Place the soil moisture probe so the sensing end can enter the soil sample. Identify VCC, GND, and AOUT / AO / Signal before wiring.

### Step 3: Connect the Soil Sensor

Connect soil sensor VCC to 3.3V. Connect soil sensor GND to GND. Connect soil sensor AOUT, AO, or Signal to GPIO 26 (ADC0).

### Step 4: Place the BME280 Sensor Module

Identify BME280 VCC, GND, SDA, and SCL before wiring. Connect BME280 VCC to 3.3V. Connect BME280 GND to GND. Connect BME280 SDA to GPIO 2 and BME280 SCL to GPIO 3. If you are using a bare BME280 sensor, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 5: Place and Wire the Relay Module

Identify relay VCC, GND, IN, COM, NO, and NC before wiring. Connect relay IN to GPIO 15. Power the relay module from the supply required by its label. Connect relay GND to Pico GND and to the external pump supply GND where shared grounding is required.

### Step 6: Place and Connect the Status LED

Place the status LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the status LED long leg through a 220Ω resistor to GPIO 16. Connect the status LED short leg to GND.

### Step 7: Wire the Pump Through the Relay

Connect the external pump supply positive wire to relay COM. Connect relay NO to the pump positive lead. Connect the pump negative lead to the external pump supply negative wire.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import bme280
import time

RELAY_ON = 0
RELAY_OFF = 1

PUMP_RUN_S = 5
MIN_REST_S = 60

soil = ADC(26)
i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)

def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)

def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)

relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)

soil_dry = 52000
soil_wet = 22000

dry_threshold = 35
recovery_threshold = 45

humidity_skip = 85
humidity_history = []
history_size = 6

last_cycle = time.time() - MIN_REST_S
ready_to_water = True

relay.value(RELAY_OFF)
status_led.off()

def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value

def soil_percent():
    raw = soil.read_u16()
    span = soil_dry - soil_wet
    if span == 0:
        return 0
    percent = int(((soil_dry - raw) * 100) / span)
    return clamp(percent, 0, 100)

def humidity_value():
    temp_c, humidity, pressure_hpa = read_bme280()
    return humidity

def rain_watch(values):
    if len(values) < 6:
        return False
    recent = sum(values[-3:]) / 3
    older = sum(values[-6:-3]) / 3
    return (recent - older) >= 5

def run_cycle():
    global last_cycle, ready_to_water
    relay.value(RELAY_ON)
    status_led.on()
    print("Pump ON: soil dry and weather rule allowed watering")
    time.sleep(PUMP_RUN_S)
    relay.value(RELAY_OFF)
    status_led.off()
    print("Pump OFF: timed cycle complete")
    last_cycle = time.time()
    ready_to_water = False

print("Automated irrigation with weather input ready")

while True:
    moisture = soil_percent()
    humidity = humidity_value()

    humidity_history.append(humidity)
    if len(humidity_history) > history_size:
        humidity_history.pop(0)

    if moisture >= recovery_threshold:
        ready_to_water = True

    skip_for_weather = humidity >= humidity_skip or rain_watch(humidity_history)

    if ready_to_water and moisture <= dry_threshold and not skip_for_weather and (time.time() - last_cycle) >= MIN_REST_S:
        run_cycle()
    else:
        print("Moisture:{}% Humidity:{}% WeatherSkip:{}".format(moisture, humidity, skip_for_weather))
    time.sleep(10)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|---|---|---|
| rain_watch() | Looks at a short humidity trend instead of one reading | A single humidity value is weaker than a short trend |
| skip_for_weather | Combines current humidity and trend into one decision | Weather-aware systems should explain when watering is intentionally skipped |
| Recovery threshold | Prevents repeated watering from one dry event | Prevents repeated watering from one dry event |
| Timed pump cycle | Keeps each watering burst short and predictable | Short cycles are safer and easier to tune |

## Expected Result

If the soil is dry and the weather rule does not block irrigation, the pump should run one timed cycle and the LED should light. If humidity is already very high or the humidity trend suggests possible rain, the system should skip irrigation and explain that choice in the Shell.

## Troubleshooting

| Problem | Possible cause | Solution |
|---|---|---|
| The pump never waters | The weather rule is blocking all attempts | Print the humidity history and compare it with the skip thresholds. |
| The system waters even when humidity is high | The skip rule or trend logic is too permissive | Lower the humidity_skip threshold or inspect the rain_watch() calculation. |
| The pump cycles too often | The rest time or recovery threshold is too low | Increase recovery_threshold or MIN_REST_S. |
| Humidity readings look unstable | The BME280 is not seated or the read interval is too short | Check the sensor wiring and keep the read interval reasonable. |
