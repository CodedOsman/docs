## Overview

This project builds a two-level water tank refill controller that starts filling when the water drops too low and stops before overflow.

Students will use two level sensors, a relay-controlled pump, a status LED, and a buzzer for fault alerts.

The final system should start the pump when the tank falls below the lower sensor, stop the pump when the upper sensor is reached, and shut down safely if the sensors report an impossible state or filling takes too long.

### Project Story

The real-world use case is a demonstration reservoir, livestock trough, or greenhouse supply tank that needs automatic refill behaviour.

---

## Learning Objectives

- Use dual-sensor logic to control one actuator safely
- Distinguish between normal states and impossible sensor combinations
- Add a timeout fail-safe to protect a pump if the tank never fills
- Drive a buzzer only for true fault conditions, not for every state change
- Think about how level-sensor placement changes the logic of the system

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins.

Use an external power supply for pumps, motors, relay modules, and fans when required.

If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly.

Check whether your relay module is active-low or active-high before testing the final load.

Keep electronics away from water and dry your hands before touching the circuit.

Do not let exposed jumper connections touch water.

Do not let a pump run dry.

Mount the upper sensor physically above the lower sensor so the logic matches the code.

Never test a refill pump without confirming where the incoming water will go.

If the pump runs but the tank does not fill, stop immediately and inspect the tubing and water source.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Tank Level Sensors
Place the low-level sensor at the lower refill point in the test tank. Place the high-level sensor above it at the stop-fill point. Identify each sensor output, VCC, and GND before wiring.

### Step 3: Connect the Low-Level Sensor
Connect the low-level sensor output to **GPIO 4**. Connect the low-level sensor GND to Pico GND. Wire its power pin according to the sensor type used in your kit.

### Step 4: Connect the High-Level Sensor
Connect the high-level sensor output to **GPIO 5**. Connect the high-level sensor GND to Pico GND. Wire its power pin according to the sensor type used in your kit.

### Step 5: Place and Wire the Relay Module
Identify relay VCC, GND, IN, COM, NO, and NC. Connect relay IN to **GPIO 15**. Power the relay module from the supply required by its label. Connect relay GND to Pico GND and to the external pump supply GND where shared grounding is required.

### Step 6: Place the Status LED
Place the status LED on the breadboard. Connect its long leg through a 220Ω resistor to **GPIO 16**. Connect its short leg to **GND**.

### Step 7: Place the Buzzer
Place the buzzer on the breadboard and identify its positive (+) and negative (-) pins. Connect the buzzer positive pin to **GPIO 17**. Connect the buzzer negative pin to **GND**.

### Step 8: Wire the Pump Through the Relay
Connect the external pump supply positive wire to relay **COM**. Connect relay **NO** to the pump positive lead. Connect the pump negative lead to the external pump supply negative wire.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

RELAY_ON = 0
RELAY_OFF = 1
MAX_FILL_S = 60
LOW_LEVEL = 35
FULL_LEVEL = 75

water_sensor = ADC(26)
relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)
buzzer = Pin(17, Pin.OUT)

filling = False
fill_started = 0

relay.value(RELAY_OFF)
status_led.value(0)
buzzer.value(0)


def water_level_percent():
    return int((water_sensor.read_u16() / 65535) * 100)


def alarm_burst():
    for _ in range(2):
        buzzer.on()
        time.sleep(0.15)
        buzzer.off()
        time.sleep(0.15)


def start_fill(reason):
    global filling, fill_started
    relay.value(RELAY_ON)
    status_led.value(1)
    filling = True
    fill_started = time.time()
    print("Pump ON:", reason)


def stop_fill(reason):
    global filling
    relay.value(RELAY_OFF)
    status_led.value(0)
    filling = False
    print("Pump OFF:", reason)


print("Smart water tank refill system ready")

while True:
    level = water_level_percent()
    now = time.time()

    if filling and level >= FULL_LEVEL:
        stop_fill("tank reached full threshold")
    elif (not filling) and level <= LOW_LEVEL:
        start_fill("water level below low threshold")
    elif filling and (now - fill_started) >= MAX_FILL_S:
        stop_fill("maximum fill time exceeded")
        alarm_burst()
    else:
        print("WaterLevel:{}% Status:{}".format(level, "REFILLING" if filling else "MONITORING"))

    time.sleep(1)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| `water_present()` | Normalises the sensor reading into water-present logic | Different sensor modules are easier to handle when the code uses one meaning |
| Sensor conflict check | Looks for the impossible state of high wet while low dry | That state usually means a wiring or mounting problem |
| `MAX_FILL_S` | Stops the pump if filling takes too long | A timeout can prevent dry running or overflow from a failed sensor |
| `alarm_burst()` | Uses the buzzer only for fault conditions | Students learn that alarms should signal problems, not every normal event |

## Expected Result

When the tank is below the low sensor, the pump should start and the LED should turn on. When the water reaches the high sensor, the pump should stop. If the upper sensor reads wet while the lower sensor reads dry, or if filling takes too long, the system should stop the pump and beep the buzzer.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The pump never starts | The low sensor logic is inverted or the relay is miswired | Print the low sensor state and verify `RELAY_ON` / `RELAY_OFF` |
| The pump never stops | The high sensor is not mounted high enough or never changes state | Test the upper sensor manually and confirm its physical placement |
| The buzzer alarms immediately | The sensors are swapped or the code sees an impossible state | Check that the high sensor is above the low sensor and wired to the correct pins |
| The timeout always trips | The normal fill time is longer than `MAX_FILL_S` or the water path is blocked | Increase `MAX_FILL_S` after measurement and inspect the tubing and pump flow |
