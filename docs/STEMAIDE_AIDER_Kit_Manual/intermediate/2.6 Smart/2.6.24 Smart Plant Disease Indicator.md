# Project 2.6.24: Smart Plant Disease Indicator

**Intermediate Embedded Systems Project Using Raspberry Pi Pico 2 W and MicroPython**

---
## Overview

This project builds a rule-based plant disease risk indicator using soil moisture, temperature, and humidity readings.

Students will build a multi-sensor monitoring circuit, calibrate the soil sensor, and combine several readings into one disease-risk score.

The final system should classify plant conditions as LOW RISK, MODERATE RISK, or HIGH RISK and show the result with LEDs and serial output.

### Project Story

The real-world use case is greenhouse or nursery monitoring, where warm, humid, and overly wet conditions can increase the chance of fungal or leaf diseases.

---

## Learning Objectives

- Integrate multiple sensors into one monitoring system
- Calibrate an analog soil moisture sensor
- Use threshold logic to build a rule-based risk score
- Translate sensor values into a practical farm decision
- Test indicator outputs before running the full program
- Use serial monitoring to explain why a decision was made

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Soil Moisture Sensor
Place the soil moisture probe so the sensing end can enter the soil sample. Check the module labels and identify VCC, GND, and AOUT / AO / Signal.

### Step 3: Connect the Soil Sensor
Connect soil sensor VCC to **3.3V**. Connect soil sensor GND to **GND**. Connect soil sensor AOUT, AO, or Signal to **GPIO 26** (ADC0).

### Step 4: Place the BME280 Sensor Module
Place the BME280 module where it can sense the plant-area air. Identify VCC, GND, and DATA before wiring. Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 2** and BME280 SCL to **GPIO 3**. If you are using a bare BME280 sensor instead of a module, use the module pins labeled VCC, GND, SDA, and SCL to 3.3V.

### Step 5: Place and Connect the Green LED
Place the green LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the green LED long leg through a 220Ω resistor to **GPIO 16**. Connect the green LED short leg to **GND**.

### Step 6: Place and Connect the Red LED
Place the red LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the red LED long leg through a 220Ω resistor to **GPIO 17**. Connect the red LED short leg to **GND**.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] Soil sensor VCC connects to 3.3V
- [x] Soil sensor GND connects to GND
- [x] Soil sensor AOUT / AO / Signal connects to GPIO 26
- [x] BME280 SDA connects to GPIO 2 and SCL connects to GPIO 3
- [x] Green LED long leg connects through a 220Ω resistor to GPIO 16
- [x] Red LED long leg connects through a 220Ω resistor to GPIO 17
- [x] Both LED short legs connect to GND
- [x] No loose jumper wires

> **Intermediate Note**
> Calibrate `SOIL_DRY` and `SOIL_WET` before final testing. The BME280 needs a short delay between readings, and disease-risk logic is only as good as the local thresholds.

> **Safety Note**
> Do not pour water onto the Pico, breadboard, USB cable, or jumper wires. Add water to the soil sample away from the electronics.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import bme280
import time

soil_sensor = ADC(26)
i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


green_led = Pin(16, Pin.OUT)
red_led = Pin(17, Pin.OUT)

SOIL_DRY = 52000
SOIL_WET = 22000
TEMP_HIGH = 30
HUMIDITY_HIGH = 80
SOIL_WET_RISK = 75
MODERATE_RISK = 30
HIGH_RISK = 70
SAMPLE_DELAY = 10


def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value


def soil_percent():
    raw = soil_sensor.read_u16()
    span = SOIL_DRY - SOIL_WET
    if span <= 0:
        return 0
    percent = int(((SOIL_DRY - raw) * 100) / span)
    return clamp(percent, 0, 100)


def read_environment():
    temp_c, humidity, pressure_hpa = read_bme280()
    return temp_c, humidity


def disease_risk(temp_c, humidity, soil_moisture):
    score = 0
    reasons = []

    if humidity >= HUMIDITY_HIGH:
        score += 40
        reasons.append('high humidity')
    elif humidity >= 70:
        score += 20
        reasons.append('rising humidity')

    if temp_c >= TEMP_HIGH:
        score += 30
        reasons.append('high temperature')
    elif temp_c >= 26:
        score += 15
        reasons.append('warm temperature')

    if soil_moisture >= SOIL_WET_RISK:
        score += 30
        reasons.append('very wet soil')
    elif soil_moisture >= 60:
        score += 15
        reasons.append('wet soil')

    return score, reasons


print('=== Smart Plant Disease Indicator ===')
print('Calibrate SOIL_DRY and SOIL_WET before final testing.\n')

while True:
    try:
        temp_c, humidity = read_environment()
        soil_moisture = soil_percent()
        risk, reasons = disease_risk(temp_c, humidity, soil_moisture)

        if risk >= HIGH_RISK:
            status = 'HIGH RISK'
            red_led.value(1)
            green_led.value(0)
        elif risk >= MODERATE_RISK:
            status = 'MODERATE RISK'
            red_led.value(1)
            green_led.value(1)
        else:
            status = 'LOW RISK'
            red_led.value(0)
            green_led.value(1)

        reason_text = ', '.join(reasons) if reasons else 'conditions stable'
        print('Temp: {} C | Humidity: {} % | Soil: {} % | Risk: {} % | {} | {}'.format(
            temp_c, humidity, soil_moisture, risk, status, reason_text
        ))

    except Exception as error:
        red_led.value(0)
        green_led.value(0)
        print('Sensor read error:', error)

    time.sleep(SAMPLE_DELAY)
```

## How the Code Works

| Code Section | 1 | Why It Matters |
|--------------|---|----------------|
| `SOIL_DRY` and `SOIL_WET` | 1 | The soil percentage is only meaningful after calibration |
| `disease_risk()` | 1 | This is rule-based decision logic instead of full machine learning |
| LED status logic | Maps LOW, MODERATE, and HIGH risk to visible LED states | Students can understand system decisions without reading code every time |
| Serial print output | 1 | Troubleshooting is much easier when the decision process is visible |

---

## Expected Result

When the environment is cool and not too wet, the green LED should stay on and the Shell should report LOW RISK.

When humidity, temperature, or soil moisture rises, the printed risk score should increase and the system should move to MODERATE RISK or HIGH RISK.

At HIGH RISK, the red LED should turn on while the Shell explains which conditions caused the alert.

## Troubleshooting

| Problem | 1 | Solution |
|---------|---|----------|
| BME280 sensor module | 1 | Temperature, humidity, and pressure input |
| The soil percentage is always 0% or 100% | 1 | Measure real raw values first and update `SOIL_DRY` and `SOIL_WET` |
| The red LED never turns on | The `HIGH_RISK` threshold is too high for your room conditions | Temporarily lower `HIGH_RISK` to confirm the LED wiring, then retune the project |
| The risk score seems unrealistic | 1 | Adjust humidity, temperature, or soil thresholds after observing several readings |
