## Overview

This project builds a BLE timer controller that turns an output on for a chosen duration and then turns it off automatically.

The example output in this manual is an LED so the timing logic can be tested safely before replacing the output with a relay or another actuator.

The final system should let the phone set a duration, start the timer, stop it early, and read the remaining time.

### Project Story

Real-world use cases include timed indicators, timed irrigation relay triggers, timed ventilation bursts, or scheduled classroom demos.

---

## Learning Objectives

- Control an output using timed logic rather than a fixed manual state
- Track elapsed time and remaining time in software
- Tune duration values from a BLE command interface
- Use a safe LED output first before upgrading to a relay-controlled load
- Separate output testing from timer testing

### Project-Specific Setup Notes

- This project uses the Pico's built-in wireless module plus two helper files: `ble_uart.py` and `ble_advertising.py`
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`
- In Thonny Shell, run `import os; print(os.listdir())` and confirm both helper files are visible on the Pico
- Communication Setup: install a wireless terminal app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone
- Scan for the device name `AIDER_SmartTimer` after the Pico starts advertising
- **Do not** pair from the normal phone settings menu. Open the terminal app, scan, connect to the UART-style service, then send text commands from inside the app
- If the wireless connection does not work, restart the Pico and rescan from the terminal app before changing the wiring

### Project-Specific Safety Note

Start with the LED version first. If you later replace the LED with a relay, revisit the relay safety rules before switching a real load.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the LED

Place the LED on the breadboard.

Identify the long leg as the anode (+) and the short leg as the cathode (-).

### Step 3: Connect the LED Long Leg

Connect the LED long leg to one end of a 220Ω resistor.

### Step 4: Connect the Resistor to GPIO 16

Connect the other end of the 220Ω resistor to GPIO 16.

### Step 5: Connect the LED Short Leg

Connect the LED short leg to GND.

### Step 6: Check the LED Polarity

Make sure the long leg goes through the resistor to GPIO 16 and the short leg goes to GND.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import Pin
import bluetooth
import time
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_SmartTimer"

output = Pin(16, Pin.OUT)
ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

duration_seconds = 10
timer_running = False
timer_deadline = 0
command_queue = []

def send_line(message):
    print(message)
    uart.write((message + "\n").encode())

def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)

def remaining_seconds(now_ms):
    if not timer_running:
        return 0
    return max(0, time.ticks_diff(timer_deadline, now_ms) // 1000)

uart.on_rx(on_rx)
output.off()
send_line("Bluetooth smart timer ready")

while True:
    now = time.ticks_ms()
    
    if timer_running and time.ticks_diff(now, timer_deadline) >= 0:
        timer_running = False
        output.off()
        send_line("Timer complete. Output OFF")
    
    if command_queue:
        command = command_queue.pop(0)
        if command.startswith("set "):
            duration_seconds = int(command.split()[1])
            send_line("Duration set to {} seconds".format(duration_seconds))
        elif command == "start":
            timer_running = True
            timer_deadline = time.ticks_add(now, duration_seconds * 1000)
            output.on()
            send_line("Timer started for {} seconds".format(duration_seconds))
        elif command == "stop":
            timer_running = False
            output.off()
            send_line("Timer stopped")
        elif command == "status":
            send_line("Running:{} Duration:{}s Remaining:{}s".format(
                timer_running, duration_seconds, remaining_seconds(now)))
        elif command == "help":
            send_line("Commands: set <seconds>, start, stop, status, help")
        else:
            send_line("Unknown command. Send help.")
    
    time.sleep_ms(100)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|---|---|---|
| duration_seconds | Stores the chosen timer length | Lets the user tune the project for different tasks |
| timer_deadline | Marks when the timer should end | The system compares the current time against this value |
| start and stop commands | Turn the output on or off and control the timer state | Provide clear remote control behaviour |
| remaining_seconds() | Calculates how much time is left | Makes status reports useful during testing |

## Troubleshooting

| Problem | Possible cause | Solution |
|---|---|---|
| LED never turns on | The LED polarity or resistor path is wrong | Check GPIO 16, the resistor, and the LED orientation. |
| Timer never ends | The start command may not have updated the deadline correctly | Send status and verify the remaining time changes while the timer is running. |
| Remaining time looks odd | The timer was restarted or the duration was changed unexpectedly | Set the duration again and run a fresh timer cycle. |
| Phone cannot find the BLE device | The Pico is not advertising or the BLE helper files are missing | Confirm ble_uart.py and ble_advertising.py are saved on the Pico, restart the board, and scan again from nRF Connect or LightBlue. |
| Commands do not change the system state | The phone is connected to the wrong service or the command text is different from the expected command | Reconnect to the BLE UART service and send the exact command shown in the test section. |
