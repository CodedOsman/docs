## Overview

This project builds an irrigation controller that listens to both soil moisture and a rain input before starting a watering cycle.

Students will combine soil sensing, rain detection, relay control, and a rain lockout timer into one practical decision-logic system.

The final system should detect rain, block watering during the rain period and for a lockout time afterwards, then allow irrigation again only when the soil is still dry.

### Project Story

The real-world use case is an outdoor garden or farm demo where watering should be skipped during rain and delayed for some time after rain stops.

---

## Learning Objectives

- Combine multiple sensor inputs into one irrigation decision
- Use a rain lockout timer to delay watering after rain events
- Test a relay output safely before connecting a real pump

- Explain why weather-aware irrigation saves water compared with simple threshold control
- Use serial output to understand why the controller skipped or allowed watering

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position and Connect the Soil Sensor
Place the soil moisture probe so the sensing end can enter the soil sample. Connect soil sensor VCC to 3.3V. Connect soil sensor GND to GND. Connect soil sensor AOUT, AO, or Signal to GPIO 26.

### Step 3: Position and Connect the Rain Sensor
Place the rain sensor where it can detect water without dripping onto the Pico or breadboard. Connect rain sensor VCC to 3.3V. Connect rain sensor GND to GND. Connect rain sensor DOUT to GPIO 4.

### Step 4: Place and Wire the Relay Module
Identify relay VCC, GND, IN, COM, NO, and NC before wiring. Connect relay IN to GPIO 15. Power the relay module according to its label. Connect relay GND to Pico GND and to the external pump or valve supply GND where shared grounding is required.

### Step 5: Place and Connect the Ready LED
Place the ready LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the ready LED long leg through a 220Ω resistor to GPIO 16. Connect the ready LED short leg to GND.

### Step 6: Place and Connect the Rain LED
Place the rain-lockout LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the rain-lockout LED long leg through a 220Ω resistor to GPIO 17. Connect the rain-lockout LED short leg to GND.

### Step 7: Keep the Irrigation Load on the Relay Side
If you connect a pump or valve, route the external supply positive wire through relay COM and NO. Connect the pump or valve negative lead to the external supply negative wire.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] Soil sensor AOUT / AO / Signal connects to GPIO 26
- [x] Rain sensor DOUT connects to GPIO 4
- [x] Relay IN connects to GPIO 15
- [x] Ready LED long leg connects through a 220Ω resistor to GPIO 16
- [x] Rain-lockout LED long leg connects through a 220Ω resistor to GPIO 17
- [x] Both LED short legs connect to GND
- [x] Pump or valve positive path uses relay COM and NO if a real load is connected
- [x] No loose jumper wires

> **Intermediate Note**
> The rain sensor uses an internal pull-up and reads LOW when rain is detected. Test the rain sensor dry and with a few drops before trusting the lockout timer.

> **Safety Note**
> Keep water away from the Pico, breadboard, USB cable, and jumper wires. Use external load power for any pump or valve, and never let a pump run dry.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

soil_sensor = ADC(26)
rain_sensor = Pin(4, Pin.IN, Pin.PULL_UP)
relay = Pin(15, Pin.OUT)
ready_led = Pin(16, Pin.OUT)
rain_led = Pin(17, Pin.OUT)

RELAY_ON = 1
RELAY_OFF = 0
SOIL_DRY = 52000
SOIL_WET = 22000
DRY_THRESHOLD = 35
RECOVERY_THRESHOLD = 50
WATER_SECONDS = 4
RAIN_LOCKOUT_SECONDS = 300
CHECK_INTERVAL = 20

last_rain_time = time.time() - RAIN_LOCKOUT_SECONDS
last_watered = time.time() - CHECK_INTERVAL


def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value


def soil_percent():
    raw = soil_sensor.read_u16()
    span = SOIL_DRY - SOIL_WET
    if span <= 0:
        return 0
    percent = int(((SOIL_DRY - raw) * 100) / span)
    return clamp(percent, 0, 100)


def is_raining():
    return rain_sensor.value() == 0


def relay_on():
    relay.value(RELAY_ON)


def relay_off():
    relay.value(RELAY_OFF)


def water_once(reason):
    global last_watered
    relay_on()
    print(reason)
    time.sleep(WATER_SECONDS)
    relay_off()
    last_watered = time.time()


relay_off()
print('=== Smart Irrigation Rain Integration ===')
print('The controller skips irrigation while rain lockout is active.\n')

while True:
    moisture = soil_percent()
    now = time.time()

    if is_raining():
        last_rain_time = now
        rain_led.value(1)
        ready_led.value(0)
        relay_off()
        print('Rain detected. Irrigation skipped immediately.')
    elif (now - last_rain_time) < RAIN_LOCKOUT_SECONDS:
        remaining = RAIN_LOCKOUT_SECONDS - int(now - last_rain_time)
        rain_led.value(1)
        ready_led.value(0)
        relay_off()
        print('Rain lockout active for {} more second(s).'.format(remaining))
    else:
        rain_led.value(0)
        ready_led.value(1)

        if moisture <= DRY_THRESHOLD and (now - last_watered) >= CHECK_INTERVAL:
            water_once('Dry soil with no rain lockout. Watering started at {}%.'.format(moisture))
        elif moisture >= RECOVERY_THRESHOLD:
            relay_off()
            print('Soil recovered above the recovery threshold. No watering needed.')
        else:
            print('Ready state | Soil: {}% | Waiting for a drier condition.'.format(moisture))

    time.sleep(CHECK_INTERVAL)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| Rain input | Detects whether water is currently on the rain sensor | The system should never ignore active rain when deciding to irrigate |
| RAIN_LOCKOUT_SECONDS | Delays irrigation for a period after the rain event | This gives natural moisture time to reach the soil and avoids wasting water |
| Dry and recovery thresholds | Define when watering should begin and when the soil is considered recovered | These thresholds keep the logic stable |
| Status LEDs | Show whether the controller is ready to irrigate or blocked by rain lockout | Students can understand the logic without reading the code constantly |

## Expected Result

When the rain sensor is active, the rain-lockout LED should turn on and the relay should stay off.

After rain stops, the controller should continue to block watering until the lockout time expires.

When the lockout ends and the soil is still dry, the relay should run one timed watering cycle and the serial monitor should explain why.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The system never leaves rain lockout | The rain sensor is always active or too sensitive | Adjust the sensor module threshold and confirm the DOUT signal changes between dry and wet states |
| The relay waters even while it is raining | The rain sensor logic is reversed | Check whether your module outputs LOW or HIGH when wet and adjust is_raining() if needed |
| The moisture percentage does not match the soil condition | The soil calibration values are incorrect | Re-measure SOIL_DRY and SOIL_WET using your actual sensor and soil |
| The lockout is too short for real outdoor use | The demo value is still set for classroom testing | Increase RAIN_LOCKOUT_SECONDS to match your real irrigation environment |
