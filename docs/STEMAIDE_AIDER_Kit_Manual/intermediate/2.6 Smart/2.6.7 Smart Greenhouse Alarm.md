## Overview

This project builds a greenhouse alarm that reacts when temperature or humidity moves outside safe limits.

Students will use a BME280 sensor, a buzzer, and status LEDs to create a clear environmental alarm system.

The final system should identify unsafe temperature or humidity conditions, show a green OK state when conditions are acceptable, and sound a repeating alarm only when unsafe conditions persist.

### Project Story

The real-world use case is a protected grow space where extreme climate conditions should be noticed quickly before plants are damaged.

---

## Learning Objectives

- Read and interpret BME280 climate data
- Create an alarm that can respond to more than one unsafe condition
- Use alarm repeat timing so a warning is noticeable but not continuous noise
- Display both normal and alarm states clearly with LEDs
- Tune environmental thresholds for different greenhouse scenarios

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place the BME280 Sensor Module
Place the BME280 module where it can sense the greenhouse air. Identify VCC, GND, and DATA before wiring.

### Step 3: Connect the BME280 Sensor Module
Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 2** and BME280 SCL to **GPIO 3**. If you are using a bare BME280 sensor instead of a module, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 4: Place the Buzzer
Place the buzzer on the breadboard and identify its positive (+) and negative (-) pins. Connect the buzzer positive pin to **GPIO 15**. Connect the buzzer negative pin to **GND**.

### Step 5: Place the Red LED
Place the red LED on the breadboard. Connect its long leg through a 220Ω resistor to **GPIO 16**. Connect its short leg to **GND**.

### Step 6: Place the Green LED
Place the green LED on the breadboard. Connect its long leg through a 220Ω resistor to **GPIO 17**. Connect its short leg to **GND**.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] BME280 VCC connects to 3.3V
- [x] BME280 GND connects to GND
- [x] BME280 SDA connects to GPIO 2
- [x] BME280 SCL connects to GPIO 3
- [x] Buzzer positive pin connects to GPIO 15
- [x] Buzzer negative pin connects to GND
- [x] Red LED long leg connects through a 220Ω resistor to GPIO 16
- [x] Green LED long leg connects through a 220Ω resistor to GPIO 17
- [x] Both LED short legs connect to GND
- [x] No loose jumper wires

> **Intermediate Note**
> The BME280 needs a short delay between readings. Keep the BME280 away from direct heaters or mist during normal testing so the alarm is based on room conditions.

### Additional Testing and Calibration Checks

**Calibration and tuning notes**

- Use temporary thresholds for classroom testing instead of forcing the environment into extreme conditions
- If the BME280 reads erratically, recheck the data wire and power stability before changing the code
- If the alarm feels too frequent, increase `ALARM_REPEAT_S` rather than removing the warning completely
- Test one unsafe condition at a time first, then try combined conditions

- [ ] BME280 readings appear correctly
- [ ] Green LED shows the safe state
- [ ] Unsafe thresholds trigger the alarm logic
- [ ] The Shell lists the actual alarm reason

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import I2C, Pin
import bme280
import time

i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


buzzer = Pin(15, Pin.OUT)
red_led = Pin(16, Pin.OUT)
green_led = Pin(17, Pin.OUT)

TEMP_HIGH = 35
TEMP_LOW = 10
HUMIDITY_HIGH = 85
HUMIDITY_LOW = 30
ALARM_REPEAT_S = 15
last_alarm = time.time() - ALARM_REPEAT_S

buzzer.off()
red_led.off()
green_led.on()


def alarm_burst():
    for _ in range(3):
        buzzer.on()
        red_led.on()
        time.sleep(0.15)
        buzzer.off()
        red_led.off()
        time.sleep(0.15)


def read_environment():
    temp_c, humidity, pressure_hpa = read_bme280()
    return temp_c, humidity


def alert_reasons(temp_c, humidity_pct):
    reasons = []
    if temp_c > TEMP_HIGH:
        reasons.append("temperature too high")
    if temp_c < TEMP_LOW:
        reasons.append("temperature too low")
    if humidity_pct > HUMIDITY_HIGH:
        reasons.append("humidity too high")
    if humidity_pct < HUMIDITY_LOW:
        reasons.append("humidity too low")
    return reasons


print("Smart greenhouse alarm ready")
time.sleep(2)

while True:
    try:
        temp_c, humidity_pct = read_environment()
    except Exception as error:
        print("Sensor error:", error)
        time.sleep(2)
        continue

    reasons = alert_reasons(temp_c, humidity_pct)
    now = time.time()

    if reasons:
        green_led.off()
        print("ALARM:", ", ".join(reasons))
        if now - last_alarm >= ALARM_REPEAT_S:
            alarm_burst()
            last_alarm = now
    else:
        red_led.off()
        buzzer.off()
        green_led.on()
        print("Status OK | Temp:{}C Hum:{}%".format(temp_c, humidity_pct))

    time.sleep(2)
```

## How the Code Works

| Code Section | 1 | Why It Matters |
|--------------|---|----------------|
| `alert_reasons()` | 1 | A good alarm should explain what is wrong |
| `ALARM_REPEAT_S` | 1 | Constant alarm noise is less useful than timed repeats |
| `alarm_burst()` | 1 | Short bursts are safer and easier to recognise |
| Green versus red LED logic | Shows safe state separately from alarm state | Status should be obvious even if the buzzer is muted |
