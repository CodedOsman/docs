## Overview

This project builds an automatic irrigation controller that reads soil moisture and waters only when the soil is genuinely dry.

Students will build a calibrated soil-sensing system, a relay-controlled pump stage, and a simple re-arm rule to prevent rapid pump cycling.

The final system should measure moisture, decide when watering is needed, run the pump for a timed burst, and wait for the soil to recover before allowing another cycle.

### Project Story

The real-world use case is a planter box or small greenhouse bed where watering should happen automatically but not continuously.

---

## Learning Objectives

- Integrate an analog soil sensor with a relay-controlled actuator
- Calibrate dry and wet reference values before trusting percentage readings
- Use threshold logic and hysteresis-style re-arming to reduce pump chatter
- Apply safe external-power wiring for a low-voltage pump
- Test sensor and pump stages separately before running the complete system

### Project-Specific Setup Notes

- No external library is required. This project uses only built-in MicroPython modules

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins. Use an external power supply for pumps, motors, relay modules, and fans when required. If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly. Check whether your relay module is active-low or active-high before testing the final load. Keep electronics away from water and dry your hands before touching the circuit. Do not let exposed jumper connections touch water. Do not let a pump run dry. Place the pump intake in water before any live pump test. Test the relay with an LED load first if you are not yet confident in the pump wiring.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Soil Moisture Sensor

Place the soil moisture probe so the sensing end can enter the soil sample.

Check the module labels and identify VCC, GND, and AOUT / AO / Signal.

### Step 3: Connect Soil Sensor VCC

Connect the soil sensor VCC pin to 3.3V.

### Step 4: Connect Soil Sensor GND

Connect the soil sensor GND pin to GND.

### Step 5: Connect Soil Sensor AOUT

Connect the soil sensor AOUT, AO, or Signal pin to GPIO 26 (ADC0).

### Step 6: Place the Relay Module

Place the relay module where its control pins and screw terminals are easy to reach.

Identify VCC, GND, IN, COM, NO, and NC before wiring.

### Step 7: Connect Relay IN

Connect relay IN to GPIO 15.

### Step 8: Connect Relay VCC

Connect relay VCC to the relay supply voltage required by the module label.

### Step 9: Connect Relay GND

Connect relay GND to Pico GND and to the external pump supply GND unless the relay module is fully isolated.

### Step 10: Place the Status LED

Place the status LED on the breadboard.

Identify the long leg as the anode (+) and the short leg as the cathode (-).

### Step 11: Connect the Status LED Long Leg

Connect the status LED long leg to one end of a 220Ω resistor.

Connect the other end of the resistor to GPIO 16.

### Step 12: Connect the Status LED Short Leg

Connect the status LED short leg to GND.

### Step 13: Wire the Pump Positive Path

Connect the external pump supply positive wire to relay COM.

Connect relay NO to the pump positive lead.

### Step 14: Connect the Pump Negative Lead

Connect the pump negative lead directly to the external pump supply negative wire.

### Step 15: Arrange the Water Area

Place the pump intake in the water container before testing.

Keep the Pico, breadboard, relay control wiring, USB cable, and jumper wires away from water.

## Full Project Code

```python
from machine import ADC, Pin
import time

RELAY_ON = 0
RELAY_OFF = 1
PUMP_RUN_S = 5
MIN_REST_S = 30

soil = ADC(26)
relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)

dry_reading = 52000
wet_reading = 22000
dry_threshold = 35
recovery_threshold = 45
last_cycle = time.time() - MIN_REST_S
ready_to_water = True

relay.value(RELAY_OFF)
status_led.value(0)


def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value


def moisture_percent(raw_value):
    span = dry_reading - wet_reading
    if span == 0:
        return 0
    percent = int(((dry_reading - raw_value) * 100) / span)
    return clamp(percent, 0, 100)


def water_cycle(reason):
    relay.value(RELAY_ON)
    status_led.value(1)
    print("Pump ON:", reason)
    time.sleep(PUMP_RUN_S)
    relay.value(RELAY_OFF)
    status_led.value(0)
    print("Pump OFF: cycle complete")


print("Smart irrigation system ready")

while True:
    now = time.time()
    raw_value = soil.read_u16()
    moisture = moisture_percent(raw_value)
    
    if moisture >= recovery_threshold:
        ready_to_water = True
    
    if ready_to_water and moisture <= dry_threshold and (now - last_cycle) >= MIN_REST_S:
        water_cycle("soil below dry threshold")
        last_cycle = time.time()
        ready_to_water = False
    else:
        print("Moisture:{}% Ready:{} Rest:{}s".format(
            moisture,
            ready_to_water,
            int(now - last_cycle),
        ))
    
    time.sleep(5)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| dry_reading and wet_reading | Store the two calibration reference points | The moisture percentage is only meaningful after calibration |
| dry_threshold and recovery_threshold | Set the watering trigger and re-arm level | These values reduce rapid on/off pump cycling |
| water_cycle() | Turns the pump on for a safe fixed time | Timed output is easier to test and troubleshoot than a continuous run |
| ready_to_water and last_cycle | Control when another watering cycle is allowed | These variables add decision logic, not just a single threshold |

---

## Expected Result

After calibration, the pump should remain off while the soil is adequately moist. When the moisture falls below the dry threshold, the relay should switch the pump on for a short watering cycle, then wait until the soil reading rises above the recovery level before another automatic run is allowed.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| Pump runs all the time | Relay logic is reversed or the threshold is set too high | Swap RELAY_ON / RELAY_OFF if needed and confirm the calibrated moisture values |
| Pump never starts | Relay wiring, external power, or calibration values are wrong | Test the relay separately, confirm external pump power, and recalibrate the sensor |
| Moisture percentage looks unrealistic | Dry and wet references do not match your sensor | Record new dry and wet readings and update the calibration values in the code |
| Pump starts too often | The rest time is too short or the recovery threshold is too low | Increase MIN_REST_S or raise the recovery threshold after observing real readings |
