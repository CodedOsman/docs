## Overview

This project builds a sanitation facility monitor that combines user activity and an air-quality proxy into one maintenance score.

Students will combine a motion or entry sensor, an analog air-quality proxy sensor, a cleaning-reset button, and warning outputs into one rule-based maintenance system.

The final system should count visits, estimate a service score, raise ATTENTION and SERVICE NEEDED states, and let staff reset the service score after cleaning.

### Project Story

The real-world use case is a shared sanitation space where cleaning need is not determined by one sensor alone but by both usage level and changing conditions inside the facility.

---

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Motion or Entry Sensor

Place the motion or entry sensor where it can represent facility activity without being splashed. Identify VCC, GND, and output before wiring.

### Step 3: Connect the Motion or Entry Sensor

Connect sensor VCC to the safe low-voltage supply required by the module label. Connect sensor GND to GND. Connect sensor output to GPIO 4.

### Step 4: Place and Connect the Air-Quality Proxy Sensor

Place the air-quality proxy sensor where it can sense a safe test condition. Identify VCC, GND, and analog output before wiring. Connect sensor VCC to the safe supply required by the module label. Connect sensor GND to GND. Connect sensor analog output to GPIO 26. Confirm the analog signal reaching GPIO 26 stays within 0V to 3.3V.

### Step 5: Place the Cleaning Reset Button

Place the cleaning reset push button across the breadboard center gap. Connect one side of the button to GPIO 5. Connect the opposite side of the button to GND.

### Step 6: Place and Connect the Service Outputs

Place the yellow attention LED and red service LED on the breadboard. Connect the yellow LED long leg through a 220Ω resistor to GPIO 16. Connect the red LED long leg through a 220Ω resistor to GPIO 17. Connect both LED short legs to GND. Connect the buzzer positive pin to GPIO 18. Connect the buzzer negative pin to GND.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

visit_sensor = Pin(4, Pin.IN, Pin.PULL_UP)
proxy_sensor = ADC(26)
reset_button = Pin(5, Pin.IN, Pin.PULL_UP)

attention_led = Pin(16, Pin.OUT)
service_led = Pin(17, Pin.OUT)
buzzer = Pin(18, Pin.OUT)

VISIT_ACTIVE_LEVEL = 0
VISIT_LOCKOUT_SECONDS = 8

ATTENTION_SCORE = 40
SERVICE_SCORE = 70

PROXY_HIGH = 65
DEBOUNCE_MS = 250

visit_count = 0
last_visit_time = 0
last_button_ms = 0
odor_hits = 0


def button_pressed():
    global last_button_ms
    now_ms = time.ticks_ms()
    if reset_button.value() == 0 and time.ticks_diff(now_ms, last_button_ms) > DEBOUNCE_MS:
        while reset_button.value() == 0:
            time.sleep(0.02)
        last_button_ms = now_ms
        return True
    return False


def proxy_percent():
    return int((proxy_sensor.read_u16() / 65535) * 100)


def service_score(visits, proxy):
    score = min(100, (visits * 3) + max(0, proxy - 30))
    return score


def beep():
    for _ in range(2):
        buzzer.value(1)
        time.sleep(0.12)
        buzzer.value(0)
        time.sleep(0.12)


print('=== Smart Sanitation Facility Monitor ===')
print('Press the reset button after cleaning to clear the service score.\n')

while True:
    now = time.time()
    proxy = proxy_percent()

    if visit_sensor.value() == VISIT_ACTIVE_LEVEL and (now - last_visit_time) >= VISIT_LOCKOUT_SECONDS:
        visit_count += 1
        last_visit_time = now
        print('Visit detected. Count = {}'.format(visit_count))

    if button_pressed():
        visit_count = 0
        odor_hits = 0
        print('Cleaning reset completed. Service score cleared.')

    if proxy >= PROXY_HIGH:
        odor_hits += 1
    else:
        odor_hits = max(0, odor_hits - 1)

    score = service_score(visit_count, proxy)

    if score >= SERVICE_SCORE or odor_hits >= 3:
        state = 'SERVICE NEEDED'
        attention_led.value(0)
        service_led.value(1)
        beep()
    elif score >= ATTENTION_SCORE:
        state = 'ATTENTION'
        attention_led.value(1)
        service_led.value(0)
        buzzer.value(0)
    else:
        state = 'OK'
        attention_led.value(0)
        service_led.value(0)
        buzzer.value(0)

    print('Visits: {} | Proxy: {}% | Odor hits: {} | Score: {} | State: {}'.format(
        visit_count, proxy, odor_hits, score, state
    ))
    time.sleep(2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|---|---|---|
| Visit lockout | Prevents one person from being counted many times in a few seconds | This makes the usage counter more realistic |
| Proxy percentage | Turns the analog service-condition sensor into a simple 0 to 100 scale | The service score can combine different inputs more easily |
| Service score | Combines visit count and condition change into one maintenance value | This is the rule-based logic that makes the monitor more useful than a single threshold |
| Cleaning reset | Lets staff clear the score after maintenance is performed | Real facility monitoring needs a clear maintenance workflow |

## Expected Result

As visits increase, the service score should rise even if the proxy value stays modest. If the proxy sensor also reports a high condition value, the system should reach ATTENTION or SERVICE NEEDED more quickly. Pressing the reset button after cleaning should clear the service score and return the system to normal monitoring.

## Troubleshooting

| Problem | Possible cause | Solution |
|---|---|---|
| The visit count rises too fast | The lockout is too short or the motion sensor is retriggering too often | Increase VISIT_LOCKOUT_SECONDS or reposition the sensor. |
| The score is always high | The proxy value is too large all the time or the score formula is too aggressive | Check the analog sensor wiring and reduce the score contribution if needed. |
| The reset button does not clear the state | The button wiring is wrong or the state is immediately re-triggered by a high proxy value | Check the button wiring and verify the proxy sensor is actually back in a lower range. |
| The buzzer sounds constantly | The system is always in SERVICE NEEDED or the buzzer output is stuck high | Check the service score logic and verify the GPIO 18 wiring. |
