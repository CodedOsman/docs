## Overview

This project builds an on-device analytics display that summarises farm sensor readings instead of showing only one raw value at a time.

Students will combine a soil moisture sensor, a BME280 sensor, an OLED display, and a page button to create a compact analytics dashboard.

The final system should show current readings, recent averages, minimum and maximum values, and a simple trend summary directly on the OLED display.

### Project Story

The real-world use case is a greenhouse bench or classroom farm station where students want to see trends and recent statistics without needing a laptop open all the time.

---

## Learning Objectives

- Use an OLED display with the Pico over I2C
- Collect and store a rolling window of sensor readings
- Calculate simple statistics such as average, minimum, and maximum
- Switch between display pages with a button
- Interpret sensor trends instead of looking at only one reading
- Test display, sensor, and button subsystems separately before combining them

### Project-Specific Setup Notes

- This project uses one external library file: `ssd1306.py`
- Save `ssd1306.py` to the Pico root folder as `ssd1306.py`
- Run `import os` and `print(os.listdir())` in the Thonny Shell and confirm `ssd1306.py` is visible on the Pico
- After wiring the OLED, run `i2c.scan()` to confirm the display is detected on the I2C bus
- If you are using a bare BME280 sensor instead of a ready-made module, add a 10 kΩ I2C pull-up resistors already included on most BME280 modules
- The BME280 uses the BME280 MicroPython library, so no extra DHT library file is required

### Project-Specific Safety Note

Power the OLED from 3.3V unless your exact module is clearly documented as safe with 3.3V logic and supply.

Keep the Pico and OLED away from wet soil and water spills.

Turn off power before rewiring the I2C bus or moving the button wires.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Place and Connect the OLED Display
Place the OLED display so its I2C pins are easy to reach. Identify VCC, GND, SDA, and SCL before wiring. Connect OLED VCC to 3.3V. Connect OLED GND to GND. Connect OLED SDA to GPIO 8. Connect OLED SCL to GPIO 9.

### Step 3: Place the BME280 Sensor Module
Place the BME280 module where it can sense the farm-area air. Connect BME280 VCC to 3.3V. Connect BME280 GND to GND. Connect BME280 SDA to GPIO 2 and BME280 SCL to GPIO 3. If your BME280 is bare, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 4: Position and Connect the Soil Sensor
Place the soil moisture probe so the sensing end can enter the soil sample. Connect soil sensor VCC to 3.3V. Connect soil sensor GND to GND. Connect soil sensor AOUT, AO, or Signal to GPIO 26.

### Step 5: Place the Page Button
Place the page push button across the breadboard center gap. Connect one side of the page button to GPIO 5. Connect the opposite side of the page button to GND.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] OLED VCC connects to 3.3V
- [x] OLED GND connects to GND
- [x] OLED SDA connects to GPIO 8
- [x] OLED SCL connects to GPIO 9
- [x] BME280 SDA connects to GPIO 2 and SCL connects to GPIO 3
- [x] Soil sensor AOUT / AO / Signal connects to GPIO 26
- [x] Page button connects between GPIO 5 and GND
- [x] No loose jumper wires

> **Intermediate Note**
> Run an I2C scanner before the full analytics code to confirm the OLED is detected. The BME280 needs a short delay between readings, and the button uses the Pico internal pull-up.

> **Safety Note**
> Power the OLED from 3.3V unless your exact module is clearly documented as 3.3V-safe at another voltage. Keep the Pico and OLED away from wet soil and water spills.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import bme280
import time
from ssd1306 import SSD1306_I2C

i2c = I2C(0, sda=Pin(8), scl=Pin(9), freq=100000)
oled = SSD1306_I2C(128, 64, i2c)
i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


soil_sensor = ADC(26)
page_button = Pin(5, Pin.IN, Pin.PULL_UP)
oled_addresses = i2c.scan()

SOIL_DRY = 52000
SOIL_WET = 22000
WINDOW_SIZE = 20
SAMPLE_DELAY = 5

soil_history = []
temp_history = []
humidity_history = []
page = 0
last_button_ms = time.ticks_ms()


def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value


def average(values):
    if not values:
        return 0
    return sum(values) / len(values)


def soil_percent():
    raw = soil_sensor.read_u16()
    span = SOIL_DRY - SOIL_WET
    if span <= 0:
        return 0
    percent = int(((SOIL_DRY - raw) * 100) / span)
    return clamp(percent, 0, 100)


def read_climate():
    try:
        temp_c, humidity, pressure_hpa = read_bme280()
        return temp_c, humidity
    except Exception as error:
        print('BME28022 read error:', error)
        return None, None


def update_history(target_list, value):
    if value is None:
        return
    target_list.append(value)
    if len(target_list) > WINDOW_SIZE:
        target_list.pop(0)


def trend_label():
    if len(soil_history) < 2:
        return 'Collecting'
    if soil_history[-1] < average(soil_history) - 5:
        return 'Drying'
    if soil_history[-1] > average(soil_history) + 5:
        return 'Recovering'
    return 'Stable'


def draw_page(current_soil, current_temp, current_humidity):
    oled.fill(0)

    if page == 0:
        oled.text('Current Data', 0, 0)
        oled.text('Soil: {}%'.format(current_soil), 0, 16)
        oled.text('Temp: {}'.format('--' if current_temp is None else str(current_temp) + 'C'), 0, 28)
        oled.text('Hum : {}'.format('--' if current_humidity is None else str(current_humidity) + '%'), 0, 40)
    elif page == 1:
        oled.text('Soil Stats', 0, 0)
        oled.text('Avg: {:>5.1f}%'.format(average(soil_history)), 0, 16)
        oled.text('Min: {:>5}%'.format(min(soil_history) if soil_history else 0), 0, 28)
        oled.text('Max: {:>5}%'.format(max(soil_history) if soil_history else 0), 0, 40)
        dry_events = sum(1 for value in soil_history if value <= 35)
        oled.text('DryEv: {}'.format(dry_events), 0, 52)
    else:
        oled.text('Trend Summary', 0, 0)
        oled.text('Trend: {}'.format(trend_label()), 0, 16)
        oled.text('Tavg: {:>5.1f}C'.format(average(temp_history)), 0, 28)
        oled.text('Havg: {:>5.1f}%'.format(average(humidity_history)), 0, 40)
        state = 'DRY' if average(soil_history) <= 40 else 'STABLE'
        oled.text('State: {}'.format(state), 0, 52)

    oled.show()


print('=== Smart Farm Analytics Display ===')
if oled_addresses:
    print('I2C devices found:', [hex(address) for address in oled_addresses])
else:
    print('Warning: no I2C devices found. Check the OLED wiring and power.')
print('Press the page button to cycle through the OLED analytics pages.\n')

while True:
    current_soil = soil_percent()
    current_temp, current_humidity = read_climate()

    update_history(soil_history, current_soil)
    update_history(temp_history, current_temp)
    update_history(humidity_history, current_humidity)

    if page_button.value() == 0 and time.ticks_diff(time.ticks_ms(), last_button_ms) > 300:
        page = (page + 1) % 3
        last_button_ms = time.ticks_ms()
        while page_button.value() == 0:
            time.sleep(0.05)

    draw_page(current_soil, current_temp, current_humidity)

    print('Page: {} | Soil: {}% | Temp: {} | Humidity: {} | Trend: {}'.format(
        page,
        current_soil,
        current_temp,
        current_humidity,
        trend_label()
    ))

    time.sleep(SAMPLE_DELAY)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| I2C scan check | Lists detected I2C devices at startup | This gives a fast clue if the display wiring is wrong before the analytics loop runs |
| History lists | Store a rolling window of recent sensor values | Analytics need several samples, not just the newest value |
| Page button logic | Cycles the OLED between display pages | This lets one small screen show several views clearly |
| draw_page() | Renders the current OLED page based on analytics values | Students can turn statistics into a readable dashboard |
| trend_label() | Compares the latest soil value against the rolling average | A simple trend label helps students think beyond one instantaneous reading |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The OLED stays blank | The display is not detected on the I2C bus or ssd1306.py is missing | Run i2c.scan(), check GPIO 8 and GPIO 9, and confirm ssd1306.py is on the Pico |
| The button changes pages randomly | Debounce timing is too short or button wiring is loose | Check the GPIO 5 wiring and keep the debounce timing in the code |
| The statistics never change | Sensor read errors or incorrect wiring | Watch the Shell for sensor errors and check the BME280 and soil wiring |
| The trend label seems wrong | Soil calibration values do not match the sensor | Collect more readings and retune SOIL_DRY and SOIL_WET |
