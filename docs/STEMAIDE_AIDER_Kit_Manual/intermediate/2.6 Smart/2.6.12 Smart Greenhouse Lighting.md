## Overview

This project builds a supplemental greenhouse lighting controller that turns a grow light on only when natural light is too low.

Students will use an LDR sensor, relay-controlled grow light, and runtime tracking to see how much artificial lighting was needed during a session.

The final system should switch the grow light on in low light, switch it off when brightness recovers, and keep a running estimate of how long the supplemental lighting has been used.

### Project Story

The real-world use case is a greenhouse shelf or indoor grow space where plants need extra light during cloudy or shaded periods.

---

## Learning Objectives

- Use an LDR to decide when supplemental plant lighting is needed
- Apply hysteresis to keep a grow-light relay from chattering
- Track how long supplemental lighting was used during a session
- Test lighting control with safe low-voltage loads before using a real grow light
- Think about how greenhouse lighting differs from general farm-area lighting

Use an external power supply for pumps, motors, relay modules, and fans when required.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Connect the LDR Module
Place the LDR module where it can sense the light level. Connect LDR module VCC to **3.3V**. Connect LDR module AOUT to **GPIO 26**. Connect LDR module GND to **GND**.

### Step 3: Place and Wire the Relay Module
Identify relay VCC, GND, IN, COM, NO, and NC before wiring. Connect relay IN to **GPIO 15**. Power the relay module from the supply required by its label. Connect relay GND to Pico GND and to the external grow-light supply GND where shared grounding is required.

### Step 4: Place and Connect the Status LED
Place the status LED on the breadboard. Identify the long leg as the anode (+) and the short leg as the cathode (-). Connect the status LED long leg through a 220Ω resistor to **GPIO 16**. Connect the status LED short leg to **GND**.

### Step 5: Wire the Grow-Light Through the Relay
Connect the external grow-light supply positive wire to relay **COM**. Connect relay **NO** to the positive side of the low-voltage grow-light load. Connect the grow-light negative wire to the external supply negative wire.

### Wiring Check

- [x] Pico 2W is placed correctly across the breadboard center gap
- [x] LDR divider node connects to GPIO 26
- [x] 10kΩ resistor connects from GPIO 26 to GND
- [x] Relay IN connects to GPIO 15
- [x] Status LED long leg connects through a 220Ω resistor to GPIO 16
- [x] Status LED short leg connects to GND
- [x] Grow-light positive path uses relay COM and NO
- [x] No load current is routed through the Pico
- [x] No loose jumper wires

> **Intermediate Note**
> Test the LDR in bright light and shade before setting the lighting threshold. Stable lighting behavior is more useful than rapid switching near one threshold.

> **Safety Note**
> Use safe low-voltage DC lighting only. Do not wire mains AC in this student project, and use a relay module that can be triggered by a 3.3V GPIO signal or a proper driver circuit.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, Pin
import time

ldr = ADC(26)
relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)

light_on_threshold = 35
light_off_threshold = 50
min_switch_gap_s = 20

light_on = False
last_switch = time.time() - min_switch_gap_s
supplemental_runtime_s = 0

relay.off()
status_led.off()


def light_percent():
    raw = ldr.read_u16()
    return int((raw / 65535) * 100)


print("Smart greenhouse lighting ready")

while True:
    level = light_percent()
    now = time.time()

    if (not light_on) and level <= light_on_threshold and (now - last_switch) >= min_switch_gap_s:
        relay.on()
        status_led.on()
        light_on = True
        last_switch = now
        print("Grow light ON: ambient light too low")

    elif light_on and level >= light_off_threshold and (now - last_switch) >= min_switch_gap_s:
        relay.off()
        status_led.off()
        light_on = False
        last_switch = now
        print("Grow light OFF: enough natural light")

    if light_on:
        supplemental_runtime_s += 3

    print("Light:{}% GrowLight:{} Supplemental:{}s".format(
        level,
        "ON" if light_on else "OFF",
        supplemental_runtime_s,
    ))
    time.sleep(3)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| Light thresholds | Decide when supplemental lighting starts and stops | Plants need stable lighting behaviour, not flicker near one threshold |
| `min_switch_gap_s` | Adds a minimum time between relay changes | Relay chatter shortens hardware life and confuses testing |
| `supplemental_runtime_s` | Tracks how long the grow light has been used | Runtime helps students think about energy use and photoperiod |
| Status LED | Shows when the grow light output is active | Visual feedback helps confirm the control logic |

## Expected Result

When the ambient light falls below the ON threshold, the relay should switch the grow light on and the status LED should light. When the light level rises above the OFF threshold, the grow light should turn off. The Shell should also show how long the supplemental light has been used during the session.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The grow light is always on | Thresholds are too high or the LDR module is miswired | Recheck the divider and lower the ON threshold after testing |
| The light never turns on | The ADC reading never reaches the low-light threshold or the relay logic is reversed | Cover the LDR fully and test the relay output separately |
| Runtime increases when the light is off | The state-tracking logic is wrong | Check that `supplemental_runtime_s` increases only inside the `light_on` branch |
| The grow light output does not respond | The external supply or relay load path is wrong | Verify the relay COM/NO wiring and the load polarity |
