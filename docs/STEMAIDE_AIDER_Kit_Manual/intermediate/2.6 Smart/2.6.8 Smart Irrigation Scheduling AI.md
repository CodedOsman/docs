## Overview

This project uses rule-based decision logic to choose when irrigation should happen based on more than one factor.

Students will build a rule-based AI-style scoring system, not a machine-learning model. The Pico combines several weighted rules and makes one watering decision from the total score.

The final system should calculate a decision score, explain the reasons behind it, and allow watering immediately after startup if the conditions already justify it.

### Project Story

The real-world use case is a smart planter or irrigation demo where time since the last watering, soil moisture, and temperature all matter together.

---

## Learning Objectives

- Combine several conditions into one rule-based decision score
- Use time since last watering as part of irrigation scheduling
- Avoid overclaiming AI when a system is really weighted rule logic
- Understand why startup state matters in automation projects
- Tune decision weights to match different environmental scenarios

### Project-Specific Safety Note

Do not power motors, pumps, fans, valves, or relay coils directly from Pico GPIO pins.

Use an external power supply for pumps, motors, relay modules, and fans when required.

If the relay module is controlled by the Pico, connect Pico GND to the external supply GND unless the relay module is fully opto-isolated and wired correctly.

Check whether your relay module is active-low or active-high before testing the final load.

Keep electronics away from water and dry your hands before touching the circuit.

Do not let exposed jumper connections touch water.

Do not let a pump run dry.

Test the relay with an LED or safe load before using the real pump.

Because this project can water immediately after startup, make sure the pump is placed safely before powering the system.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W
Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Soil Moisture Sensor
Place the soil moisture probe so the sensing end can enter the soil sample. Identify VCC, GND, and AOUT / AO / Signal before wiring.

### Step 3: Connect the Soil Sensor
Connect soil sensor VCC to **3.3V**. Connect soil sensor GND to **GND**. Connect soil sensor AOUT, AO, or Signal to **GPIO 26** (ADC0).

### Step 4: Place the BME280 Sensor Module
Identify BME280 VCC, GND, SDA, and SCL before wiring. Connect BME280 VCC to **3.3V**. Connect BME280 GND to **GND**. Connect BME280 SDA to **GPIO 2** and BME280 SCL to **GPIO 3**. If you are using a bare BME280 sensor, use the module pins labeled VCC, GND, SDA, and SCL.

### Step 5: Place and Wire the Relay Module
Identify relay VCC, GND, IN, COM, NO, and NC. Connect relay IN to **GPIO 15**. Power the relay module from the supply required by its label. Connect relay GND to Pico GND and to the external pump supply GND where shared grounding is required.

### Step 6: Place the Status LED
Place the status LED on the breadboard. Connect its long leg through a 220Ω resistor to **GPIO 16**. Connect its short leg to **GND**.

### Step 7: Wire the Pump Through the Relay
Connect the external pump supply positive wire to relay **COM**. Connect relay **NO** to the pump positive lead. Connect the pump negative lead to the external pump supply negative wire.

### Step 8: Confirm the Pump Position
Place the pump intake safely before powering the system, because the code can decide to water after startup.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import bme280
import time

RELAY_ON = 0
RELAY_OFF = 1
PUMP_RUN_S = 5
HOURS_BETWEEN = 12

soil = ADC(26)
i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


relay = Pin(15, Pin.OUT)
status_led = Pin(16, Pin.OUT)

soil_dry = 52000
soil_wet = 22000
moisture_critical = 20
moisture_low = 35
moisture_recovery = 45
temp_hot = 32
temp_cold = 15

last_watered = time.time() - (HOURS_BETWEEN * 3600)

relay.value(RELAY_OFF)
status_led.value(0)


def clamp(value, low, high):
    if value < low:
        return low
    if value > high:
        return high
    return value


def moisture_percent():
    raw = soil.read_u16()
    span = soil_dry - soil_wet
    if span == 0:
        return 0
    percent = int(((soil_dry - raw) * 100) / span)
    return clamp(percent, 0, 100)


def temperature_value():
    temp_c, humidity, pressure_hpa = read_bme280()
    return temp_c


def decision_score(moisture, temp_c, hours_since):
    score = 0
    reasons = []

    if moisture <= moisture_critical:
        score += 55
        reasons.append("critical dry soil")
    elif moisture <= moisture_low:
        score += 30
        reasons.append("low soil moisture")
    elif moisture >= moisture_recovery:
        score -= 20
        reasons.append("soil recovered")

    if hours_since < HOURS_BETWEEN:
        score -= 25
        reasons.append("recent watering")

    if temp_c >= temp_hot:
        score += 20
        reasons.append("hot weather")
    elif temp_c <= temp_cold:
        score -= 10
        reasons.append("cold weather")

    return clamp(score, 0, 100), reasons


def water_once():
    global last_watered
    relay.value(RELAY_ON)
    status_led.value(1)
    print("Pump ON: decision score requested watering")
    time.sleep(PUMP_RUN_S)
    relay.value(RELAY_OFF)
    status_led.value(0)
    last_watered = time.time()
    print("Pump OFF: timed cycle complete")


print("Smart irrigation scheduling AI ready")

while True:
    moisture = moisture_percent()
    try:
        temp_c = temperature_value()
    except Exception:
        temp_c = 25

    hours_since = (time.time() - last_watered) / 3600
    score, reasons = decision_score(moisture, temp_c, hours_since)

    print("Moisture:{}% Temp:{}C HoursSince:{:.1f}".format(moisture, temp_c, hours_since))
    print("Decision score:{} Reasons:{}".format(score, ", ".join(reasons) if reasons else "none"))

    if score >= 60:
        water_once()
    else:
        print("Decision: wait")

    time.sleep(60)
```

## Expected Result

The system should print a moisture reading, a temperature reading, a decision score, and the reasons behind that score. If the score is high enough, the pump should run for one timed cycle. Because `last_watered` is initialised with a startup offset, the project can water immediately after boot if the rules support that decision.

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| The system never waters on startup | 1 | Use `last_watered = time.time() - (HOURS_BETWEEN * 3600)` |
| The score is always high | 1 | Review the scoring weights and compare the live sensor values with the thresholds |
| The score is always low | 1 | Adjust `HOURS_BETWEEN` or reduce the negative weight for recent watering |
| The reasons list is confusing | 1 | Test one condition at a time and print the sensor values beside the reasons |
