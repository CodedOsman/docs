## Overview

This project combines soil moisture and temperature monitoring to create a simple greenhouse alert node.

The Pico reads two inputs, applies threshold logic, and raises a buzzer warning when the plant environment becomes unsafe.

A similar system could be used to warn about dry soil, overheating, or a combined hot-and-dry condition in a small greenhouse or seedling station.

The final system should report both sensor values, let the user tune the soil and temperature thresholds, and warn when the greenhouse environment needs attention.

### Project Story

This project connects classroom electronics to a practical embedded-system problem. Learners will see how sensors, logic, outputs, and communication can work together in a real application.

### Project-Specific Setup Notes

- This project uses the Pico's built-in wireless module plus two helper files: `ble_uart.py` and `ble_advertising.py`
- Save `ble_uart.py` to the Pico root folder as `ble_uart.py`
- Save `ble_advertising.py` to the Pico root folder as `ble_advertising.py`
- In Thonny Shell, run `import os; print(os.listdir())` and confirm both helper files are visible on the Pico
- Communication Setup: install a BLE app such as nRF Connect, LightBlue, or another BLE UART terminal app on your phone
- Scan for the device name `AIDER_GreenhouseAlert` after the Pico starts advertising
- **Do not** pair from the normal phone Bluetooth settings menu. Open the terminal app, scan, connect to the UART-style service, then send text commands from inside the app
- If Bluetooth does not work, restart the Pico and rescan from the terminal app before changing the wiring

### Project-Specific Safety Note

Keep electronics away from water and dry your hands before touching the circuit. Place sensors in water only after the Pico is stable and the wiring has been checked. Do not let exposed jumper connections touch water. Do not let the BME280 sensor module or its connections get wet.

## Step-by-Step Assembly

### Step 1: Place the Raspberry Pi Pico 2W

Place the Raspberry Pi Pico 2W on the breadboard so it sits across the center gap. Keep the USB port facing outward so you can easily connect it to your computer.

### Step 2: Position the Soil Moisture Sensor

Place the soil moisture probe so the sensing end can enter the soil sample. Identify VCC, GND, and AOUT / AO / Signal on the sensor module.

### Step 3: Connect Soil Sensor VCC

Connect the soil sensor VCC pin to 3.3V.

### Step 4: Connect Soil Sensor GND

Connect the soil sensor GND pin to GND.

### Step 5: Connect Soil Sensor AOUT

Connect the soil sensor AOUT, AO, or Signal pin to GPIO 26 (ADC0).

### Step 6: Place the BME280 Temperature Sensor

Place the BME280 so its leads are easy to identify and do not touch each other. Identify VCC, GND, SDA, and SCL using the printed label or the datasheet for your exact BME280 package.

### Step 7: Connect BME280 VCC

Connect BME280 VCC to 3.3V.

### Step 8: Connect BME280 GND

Connect BME280 GND to GND.

### Step 9: Connect BME280 SDA/SCL

Connect BME280 SDA/SCL to GPIO 2 (SDA) and GPIO 3 (SCL).

### Step 10: Place the Buzzer

Place the buzzer on the breadboard and identify its positive (+) and negative (-) pins.

### Step 11: Connect the Buzzer Positive Pin

Connect the buzzer positive (+) pin to GPIO 18.

### Step 12: Connect the Buzzer Negative Pin

Connect the buzzer negative (-) pin to GND.

### Step 13: Separate Soil and Electronics

Keep wet soil away from the Pico, BME280 leads, breadboard, USB cable, and jumper wires.

## Full Project Code

After completing and checking the circuit connections, open Thonny IDE. Copy and paste the code below into a new file, or upload the project file to the Raspberry Pi Pico 2 W, then run it from Thonny.

```python
from machine import ADC, I2C, Pin
import bme280
import time
import bluetooth
from ble_uart import BLEUART

DEVICE_NAME = "AIDER_GreenhouseAlert"
ALERT_COOLDOWN_MS = 12000

soil = ADC(26)
buzzer = Pin(18, Pin.OUT)
i2c_bme = I2C(1, sda=Pin(2), scl=Pin(3), freq=400000)
bme = bme280.BME280(i2c=i2c_bme)


def number_from_bme(text_value):
    cleaned = "".join(ch for ch in str(text_value) if ch.isdigit() or ch in ".-")
    return float(cleaned)


def read_bme280():
    temp_text, pressure_text, humidity_text = bme.values
    return number_from_bme(temp_text), number_from_bme(humidity_text), number_from_bme(pressure_text)


ble = bluetooth.BLE()
ble.active(True)
uart = BLEUART(ble, name=DEVICE_NAME)

dry_reading = 52000
wet_reading = 22000
soil_limit = 35
temp_limit = 30.0
alarm_enabled = True
last_alert_ms = time.ticks_add(time.ticks_ms(), -ALERT_COOLDOWN_MS)
command_queue = []


def send_line(message):
    print(message)
    uart.write((message + "\n").encode())


def on_rx(data):
    command = data.decode("utf-8").strip().lower()
    if command:
        command_queue.append(command)


def clamp(value, low, high):
    return max(low, min(high, value))


def soil_percent(raw_value):
    span = dry_reading - wet_reading
    if span == 0:
        return 0
    percent = int(((dry_reading - raw_value) * 100) / span)
    return clamp(percent, 0, 100)


def greenhouse_state(soil_pct, temp_c):
    if soil_pct < soil_limit and temp_c > temp_limit:
        return "HOT AND DRY"
    if soil_pct < soil_limit:
        return "LOW SOIL MOISTURE"
    if temp_c > temp_limit:
        return "HIGH TEMPERATURE"
    return "OK"


def beep_alert():
    for _ in range(2):
        buzzer.on()
        time.sleep_ms(200)
        buzzer.off()
        time.sleep_ms(200)


uart.on_rx(on_rx)
send_line("Bluetooth greenhouse alert ready")

while True:
    now = time.ticks_ms()
    soil_raw = soil.read_u16()
    soil_pct = soil_percent(soil_raw)
    temp_c, humidity_pct, pressure_hpa = read_bme280()
    temp_c = round(temp_c, 1)
    state = greenhouse_state(soil_pct, temp_c)

    if state != "OK" and alarm_enabled and time.ticks_diff(now, last_alert_ms) >= ALERT_COOLDOWN_MS:
        beep_alert()
        send_line("ALERT: {} | Soil:{}% Temp:{}C Hum:{:.1f}%".format(state, soil_pct, temp_c, humidity_pct))
        last_alert_ms = now

    if command_queue:
        command = command_queue.pop(0)
        if command in ("status", "read"):
            send_line("Soil:{}% Temp:{}C Hum:{:.1f}% Pressure:{:.1f}hPa State:{}".format(soil_pct, temp_c, humidity_pct, pressure_hpa, state))
        elif command == "dry":
            dry_reading = soil_raw
            send_line("Dry reference captured: {}".format(dry_reading))
        elif command == "wet":
            wet_reading = soil_raw
            send_line("Wet reference captured: {}".format(wet_reading))
        elif command == "alarm off":
            alarm_enabled = False
            buzzer.off()
            send_line("Alarm disabled")
        elif command == "alarm on":
            alarm_enabled = True
            send_line("Alarm enabled")
        elif command == "help":
            send_line("Commands: status, dry, wet, alarm on, alarm off, help")
        else:
            send_line("Unknown command. Send help.")

    time.sleep(2)
```

## How the Code Works

| Code Section | What It Does | Why It Matters |
|--------------|--------------|----------------|
| Two ADC channels | Read soil moisture on GPIO 26 and BME280 temperature on GPIO 2 and GPIO 3 | The system needs both measurements to judge greenhouse risk |
| Soil calibration | Turns the raw soil value into a percentage | Makes the moisture threshold easier to tune |
| greenhouse_state() | Summarises the environment into OK, LOW SOIL MOISTURE, HIGH TEMPERATURE, or HOT AND DRY | Shows how combined decision logic works |
| Threshold commands | Allow both moisture and temperature tuning from BLE | Make the system adaptable to different crops or conditions |

## Troubleshooting

| Problem | Possible cause | Solution |
|---------|----------------|----------|
| Temperature reading is unrealistic | The BME280 wiring is wrong or the conversion formula is not matching the sensor type | Check the BME280 pin orientation and confirm you are using the correct analog temperature sensor. |
| Soil alert behaves badly | The soil sensor is not calibrated or the threshold is too aggressive | Capture dry and wet references again and review the soil threshold. |
| The alert message is too frequent | The cooldown is too short for the environment | Increase ALERT_COOLDOWN_MS after confirming the system logic works. |
| Phone cannot find the BLE device | The Pico is not advertising or the BLE helper files are missing | Confirm ble_uart.py and ble_advertising.py are saved on the Pico, restart the board, and scan again from nRF Connect or LightBlue. |
| Commands do not change the system state | The phone is connected to the wrong service or the command text is different from the expected command | Reconnect to the BLE UART service and send the exact command shown in the test section. |
